## 13. Low-Level Design (LLD) / Object-Oriented Design

> Product-based companies typically run a **separate LLD round** from HLD. LLD tests SOLID, design patterns, class relationships, and class-level design of a small problem (rate limiter, parking lot, elevator). Don't merge it with HLD prep.

**SOLID Principles:**

- 🟡 What does SOLID stand for? Why does it matter?
- 🟡 **S** — Single Responsibility Principle (one reason to change)
- 🟡 **O** — Open/Closed Principle (open for extension, closed for modification)
- 🟡 **L** — Liskov Substitution Principle (example of violation)
- 🟡 **I** — Interface Segregation Principle (fat-interface smell)
- 🟡 **D** — Dependency Inversion Principle (DI connection)
- 💡 Which SOLID principle have you applied in your project? Give a concrete case.

**OOP Fundamentals:**

- 🟡 Tight vs loose coupling — what's the difference and why does it matter?
- 🟡 What is cohesion? Why is high cohesion desirable?
- 🔴 Why prefer composition over inheritance? Give a real example.

**Design Patterns:**

- 🟡 What is MVC architecture?
- 🟡 What is Singleton pattern? When to use it?
- 🟡 What is Factory pattern?
- 🟡 What is Repository pattern?
- 🟡 What is Strategy pattern?
- 🟡 What is Observer pattern? (pub/sub at class level)
- 🔴 What is CQRS? (Command Query Responsibility Segregation — conceptual)
- 🔴 What is Event Sourcing? (conceptual)
- 💡 Which design patterns have you used in your projects and why?

**LLD Practice Prompts (3+ yr favorites):**

- 💡 Design a **Rate Limiter** — classes, strategy pattern (fixed window / sliding window / token bucket), thread safety
- 💡 Design a **Parking Lot** — vehicle types, spot allocation, pricing, ticketing
- 💡 Design an **Elevator System** — state machine, scheduling, multi-elevator coordination
- 💡 Design a **Library Management System** — user, book, loan, reservation
- 💡 Design a **Vending Machine** — state pattern (idle / selecting / dispensing / paying)
- 💡 Design a **Logger** — levels, appenders, singleton vs DI, async logging
- 💡 Design a **Cache with LRU eviction** — `LinkedHashMap` or doubly-linked list + hashmap

**LLD Framework (how to approach any prompt):**

1. **Clarify requirements** — functional + non-functional (thread safety, scale, extensibility)
2. **Identify entities** — nouns become classes; list their responsibilities
3. **Define relationships** — inheritance, composition, association
4. **Apply patterns** — pick one that fits; justify why (don't force a pattern)
5. **Show class interactions** — sequence of method calls for the main flow
6. **Call out SOLID trade-offs** — "I put X here to keep SRP; could extract if it grows"

## 14. High-Level Design (HLD) Practice Problems

**For 3+ years, focus on these. You should be able to design each at a basic-to-intermediate level.**

### Tier 1: Must Prepare (very commonly asked)

- 💡 Design a URL Shortener (like bit.ly)
    - Hashing, base62 encoding, redirect flow, analytics
- 💡 Design an API Rate Limiter
    - Algorithms, storage (Redis), distributed rate limiting
- 💡 Design an E-commerce Order System
    - Order flow, payment, inventory, async notifications
- 💡 Design a Notification System
    - Email, SMS, push — queueing, retry, user preferences

### Tier 2: Good to Prepare (frequently asked)

- 💡 Design a Chat Application (like WhatsApp/Slack)
    - WebSockets, message storage, online status, group chat
- 💡 Design a Payment System (like Stripe/Razorpay)
    - Idempotency, double-payment prevention, webhooks, reconciliation
- 💡 Design a File Upload/Storage Service (like S3)
    - Chunked upload, presigned URLs, metadata, CDN
- 💡 Design a File-Processing / Async Job Pipeline
    - Upload → queue → worker → DLQ → status API (close to real backend work)
- 💡 Design a Link Preview / Unfurling Service
    - Fetching, parsing, caching, rate limiting per-host, SSRF protection

### Tier 3: Bonus (impressive if you know)

- 💡 Design a Ride-Hailing App (like Uber)
    - Location matching, real-time tracking, surge pricing
- 💡 Design a Video Streaming Platform (like YouTube)
    - Transcoding, adaptive bitrate, CDN, recommendations
- 💡 Design a Social Media Feed (like Twitter/Instagram)
    - Fan-out on write vs read, ranking, caching

## 15. HLD Framework (How to Approach Any HLD Problem)

**Use this framework in interviews:**

1. **Clarify Requirements** — Ask questions. Functional + non-functional.
2. **Estimate Scale** — Users, requests/sec, data size. (back-of-envelope)
3. **High-Level Design** — Draw boxes: client, API, services, DB, cache, queue.
4. **Deep Dive** — Pick 1-2 components and go deep. (DB schema, API design, caching strategy)
5. **Trade-offs** — Explain WHY you chose X over Y. This is what matters most.
6. **Address Bottlenecks** — Where could this fail? How would you scale it?

> **The #1 rule:** Interviewers care more about your _thinking process_ and _trade-off reasoning_ than the "perfect" answer. Talk through your decisions.
