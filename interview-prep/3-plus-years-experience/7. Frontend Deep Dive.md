### 30. Frontend System Design

**How to Approach:**

- 🟡 How do you approach a frontend system design question?
- 🟡 What are functional vs non-functional requirements for a UI?
- 🟡 How do you make tech choices? (library, framework, build tools)
- 🟡 How do you design component architecture and hierarchy?

**Non-Functional Requirements to Address:**

- 🟡 Responsiveness (mobile/desktop)
- 🟡 Accessibility (keyboard nav, screen readers, ARIA)
- 🟡 SEO considerations
- 🟡 Caching strategies (HTTP cache, in-memory, CDN)
- 🟡 Internationalization / Localization (i18n / l10n)
- 🔴 Offline support (Service Workers, PWA — conceptual)
- 🔴 Real-time updates (WebSocket vs SSE vs polling)
- 🔴 Logging & monitoring (Sentry, error tracking)

**Practice Problems:**

Tier 1 — Must Prepare:

- 🔴 Design an autocomplete / search bar component
- 🔴 Design an infinite scroll feed (like Twitter/Instagram)
- 🔴 Design a chat interface (real-time messaging UI)

Tier 2 — Good to Prepare:

- 🔴 Design a file upload component with progress tracking
- 🔴 Design a dashboard with real-time data (polling vs WebSocket vs SSE)
- 🔴 Design an image carousel / gallery with lazy loading

Tier 3 — Bonus:

- 🔴 Design a notification system UI (toast, badge, dropdown)
- 🔴 Design an e-commerce product listing page (filters, sort, pagination)

**Framework for Answering:**

1. **Clarify Requirements** — functional + non-functional
2. **Component Hierarchy** — break UI into components, define data flow
3. **State Management** — local state, global state, server state
4. **API Design** — what endpoints, data shape, caching
5. **Performance** — lazy loading, virtualization, memoization
6. **Edge Cases** — error states, empty states, loading states, offline
7. **Trade-offs** — explain WHY you chose X over Y
