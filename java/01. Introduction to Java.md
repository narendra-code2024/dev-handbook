## What is Java?

A general-purpose, **object-oriented programming language** plus a platform (runtime + standard libraries).

- **Created** — James Gosling at Sun Microsystems, 1995
- **Now owned** — Oracle Corporation (acquired Sun in 2010)
- **Design goals** — write once run anywhere, memory safety, automatic garbage collection, built-in concurrency
- **Used for** — enterprise backends, Android, big data tools (Hadoop, Spark), build tools, IDEs

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

## Java Editions

Java has grown into multiple editions targeted at different environments:

### Java SE (Standard Edition)

The **core Java platform** — the language, JVM, and standard libraries (`java.lang`, `java.util`, `java.io`, `java.time`, `java.nio`, etc.).

- What most developers think of as "Java"
- Includes everything needed for desktop apps, command-line tools, server apps (though servers typically use frameworks built on SE)

### Jakarta EE (Enterprise Edition)

Enterprise add-ons **on top of Java SE** — Servlets, JSP, JPA, EJB, JMS, etc. Used for server-side / web / distributed systems.

- **Formerly Java EE** — renamed to Jakarta EE in **2018** when Oracle transferred it to the **Eclipse Foundation**
- `javax.*` packages renamed to `jakarta.*` starting Jakarta EE 9
- Spring Framework and Spring Boot compete with / complement Jakarta EE for most real-world server apps

### Java ME (Micro Edition)

Legacy edition for **mobile / embedded / feature-phones**. Effectively **dead** for modern development.

- Modern mobile development moved to **Android** (uses Java + Kotlin but on a completely separate platform — different runtime, different APIs)
- Android is **NOT** a Java Edition — it just happens to use the Java language

### Summary

|Edition|Purpose|Status|
|---|---|---|
|**Java SE**|Core platform — language + JVM + standard libs|Active, most common|
|**Jakarta EE**|Enterprise / server-side APIs|Active (Eclipse Foundation)|
|**Java ME**|Mobile / embedded (legacy)|Dead for modern use|
|Android|Mobile (separate platform, uses Java language)|Not an edition, but fills Java ME's old role|

## Java Features

Java's original whitepaper listed ~11 buzzwords. The five that still matter in practice:

### Object-Oriented

Everything (except primitives) is an object. Code is organized around classes, inheritance, polymorphism, encapsulation.

```java
class Employee {
    private String name;
    public Employee(String name) { this.name = name; }
    public String getName() { return name; }
}
```

Full OOP coverage: [12. Classes, Objects & OOP Foundations](<12. Classes, Objects & OOP Foundations.md>).

### Platform Independent

Compile once → run on any OS with a JVM. The compiler produces **bytecode** (`.class`), not machine code. Each platform's JVM interprets / JIT-compiles that bytecode to native machine code.

- Same `.class` file runs on Windows, macOS, Linux
- Developers say "compile once, run anywhere" — accurate at the bytecode level

### Secured

Multiple layers:

- **No pointer arithmetic** — can't manipulate memory directly, preventing buffer overflow classes of bugs
- **Automatic memory management** — garbage collector; no manual `free()` / dangling pointers
- **Bytecode verification** — JVM checks class files before execution (prevents malformed bytecode exploits)

### Multithreaded

First-class concurrency — threads, `synchronized`, `java.util.concurrent`, and virtual threads (Java 21+). Full coverage: [26. Multithreading & Concurrency](<26. Multithreading & Concurrency.md>).

### Compiled AND Interpreted

Not pure-compiled (like C++) and not pure-interpreted (like classic Python). **Both**:

1. **`javac`** (compiler) → `.java` source becomes `.class` bytecode
2. **JVM** (runtime) → interprets bytecode line by line AT FIRST, then **JIT-compiles hot code paths** to native machine code for speed

This hybrid model is why Java has "warm-up" time — the first few requests are interpreted, later ones are near-native speed.

## Commonly Asked Gotchas

1. **Java ≠ JavaScript** — completely different languages. JavaScript was named to ride Java's hype in 1995; there's no technical relationship.

2. **Jakarta EE vs Spring** — Jakarta EE is a specification; Spring is a framework. Most modern Java server apps use Spring Boot even when not using Jakarta EE specs.

3. **Android is not Java** — Android uses the Java *language* (and Kotlin), but runs on ART (not JVM), uses a different standard library (no AWT/Swing/JavaFX), and has completely different APIs.

4. **Version numbering confusion** — Java 1.0 → 1.4, then jumped to Java 5 (internally still "1.5"), now Java 25. The `java -version` command may still show `1.8.xxx` for Java 8 due to legacy naming.
