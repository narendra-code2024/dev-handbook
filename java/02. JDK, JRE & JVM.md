## JDK vs JRE vs JVM

```
JDK (Java Development Kit)
├── JRE (Java Runtime Environment)
│   ├── JVM (Java Virtual Machine)
│   └── Core Libraries (java.lang, java.util, java.io, etc.)
├── Compiler (javac)
├── Debugger (jdb)
├── Other tools (jar, javadoc, jstack, jmap, etc.)
```

||JDK|JRE|JVM|
|---|---|---|---|
|What|Development kit|Runtime environment|Virtual machine|
|Purpose|Develop + Run|Run only|Execute bytecode|
|Contains|JRE + compiler + tools|JVM + libraries|Execution engine|
|Who needs it|Developers|End users|Both|

```java
// Flow: write → compile → run
HelloWorld.java  →  javac  →  HelloWorld.class  →  java  →  Output
   (source)       (JDK)       (bytecode)          (JVM)
```

**"Write Once, Run Anywhere"** — `javac` compiles to platform-independent bytecode (`.class`). JVMs on Windows, Mac, Linux each interpret/compile that same bytecode to their own native code.

> **Install note (Java 11+):** Oracle no longer ships the JRE as a separate download. The JDK now bundles everything you need to run Java — developers and end users both install the JDK. The conceptual three-layer distinction (JDK ⊇ JRE ⊇ JVM) still holds for understanding *what runs where*.

## JVM Architecture (high-level)

```
ClassLoader Subsystem
        ↓
Runtime Data Areas (Memory)
  ├── Method Area (Metaspace)   ← class metadata, static vars
  ├── Heap                      ← objects, instance vars, String Constant Pool
  ├── Stack (per thread)        ← local vars, method frames
  ├── PC Register (per thread)  ← current instruction pointer
  └── Native Method Stack (per thread)
        ↓
Execution Engine
  ├── Interpreter
  ├── JIT Compiler
  └── Garbage Collector
        ↓
Java Native Interface (JNI) → Native Libraries
```

### Quick reference

- **Metaspace** replaced PermGen in Java 8 — uses native memory, grows dynamically
- **String Constant Pool** lives in the Heap since Java 7 (was in PermGen before)
- **Interpreter + JIT** work together — interpreter for fast startup, JIT compiles hot code to native for speed
- **G1 GC** is the default since Java 9
