## Program Structure

### Hello World

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

Structural rules:

- **One public class per file** ‚Äî filename must match the class name: `class HelloWorld` ‚Üí `HelloWorld.java`
- Every program starts in the **`main` method**
- Code lives inside a **class**; no free-floating functions or statements
- Statements end with **`;`**; blocks are delimited by **`{ }`**

### `main` Method Signature

```java
public static void main(String[] args)
```

Each piece is mandatory for the JVM to recognize this as the entry point:

- **`public`** ‚Äî JVM needs to access it from outside
- **`static`** ‚Äî callable without an object instance (no constructor to invoke first)
- **`void`** ‚Äî `main` doesn't return a value to the OS
- **`main`** ‚Äî JVM looks for this exact name
- **`String[] args`** ‚Äî array of command-line arguments

**Alternate valid signatures:**

```java
public static void main(String... args)                    // varargs form
public static final synchronized void main(String[] args)  // modifiers can combine
```

### Command-Line Arguments

Whatever you type after the class name becomes the `args` array:

```bash
java MyApp foo bar baz
```

```java
public class MyApp {
    public static void main(String[] args) {
        System.out.println(args.length);   // 3
        System.out.println(args[0]);        // "foo"
        System.out.println(args[1]);        // "bar"
    }
}
```

- `args` is never `null` ‚Äî it's an empty array if no arguments passed
- Elements are always `String` ‚Äî parse to numbers yourself via `Integer.parseInt(args[0])` etc.

### Parameters vs Arguments

Frequently confused terminology:

```java
// "name" is a PARAMETER ‚Äî the placeholder in the method declaration
void greet(String name) {
    System.out.println("Hi " + name);
}

greet("Alice");   // "Alice" is an ARGUMENT ‚Äî the actual value passed at call time
```

|Parameter|Argument|
|---|---|
|In the method declaration|At the call site|
|A placeholder / variable|The actual value|
|Type is declared|Type comes from the value being passed|

## Statements, Expressions & Blocks

Three syntactic units that make up Java code.

**Statements** ‚Äî instructions that do something; end with `;`.

```java
int x = 5;                        // assignment
System.out.println("hello");      // expression statement
return x;                         // return
if (x > 0) { /* ... */ }          // control-flow (no ; after })
```

**Expressions** ‚Äî produce a value.

```java
5 + 3                             // ‚Üí 8
x > 0                             // ‚Üí true/false
name.toUpperCase()                // ‚Üí String
new Employee("Alice")             // ‚Üí Employee instance
```

Only **assignments, increment/decrement, method calls, and object creation** can stand alone as statements (followed by `;`). Pure arithmetic (`5 + 3;`) and comparisons (`x > 0;`) are compile errors ‚Äî they produce a value with no side effect.

**Blocks** ‚Äî code grouped with `{ }` (method bodies, loops, conditionals). Variables declared in a block are only visible within it.

```java
if (condition) {
    int x = 10;                   // block-scoped
    System.out.println(x);
}                                  // x goes out of scope here
```

## Identifiers, Keywords & Naming Conventions

### Identifier Rules

Names you give to classes, methods, variables, etc. must follow these rules:

- Can contain: **letters, digits, `$`, `_`**
- **Must start with a letter, `$`, or `_`** ‚Äî NOT a digit
- **Case-sensitive** ‚Äî `name` and `Name` are different
- **No spaces, hyphens, or punctuation** (other than `$` and `_`)
- **Cannot be a reserved keyword** (`int`, `class`, `public`, etc.)

```java
int userName;         // ‚úÖ valid
int _private;         // ‚úÖ valid (leading _ allowed)
int $price;           // ‚úÖ valid (leading $ allowed, but rare convention ‚Äî reserved for machine-generated code)
int user_name;        // ‚úÖ valid
int 2names;           // ‚ùå can't start with digit
int user-name;        // ‚ùå hyphen not allowed
int class;            // ‚ùå reserved keyword
int user name;        // ‚ùå space not allowed
```

### Reserved Keywords

Words the compiler reserves ‚Äî can't be used as identifiers:

```
abstract    continue    for           new         switch
assert      default     if            package     synchronized
boolean     do          goto*         private     this
break       double      implements    protected   throw
byte        else        import        public      throws
case        enum        instanceof    return      transient
catch       extends     int           short       try
char        final       interface     static      void
class       finally     long          strictfp    volatile
const*      float       native        super       while

// Java 9+ module-related (contextual keywords)
module  requires  exports  opens  uses  provides  to  with

// Literals (technically reserved)
true  false  null
```

`goto` and `const` are **reserved but not used** (legacy reservations to prevent future clashes).

**Contextual keywords** (restricted in specific contexts but usable as identifiers elsewhere):

- `var` (Java 10+) ‚Äî local variable type inference
- `record` (Java 16+) ‚Äî record declaration
- `sealed`, `non-sealed`, `permits` (Java 17+) ‚Äî sealed classes
- `yield` (Java 14+) ‚Äî return a value from a switch expression block

### Naming Conventions

Not enforced by the compiler, but universal in the Java ecosystem:

|Entity|Convention|Example|
|---|---|---|
|**Packages**|lowercase, dot-separated|`com.example.order`|
|**Classes / Interfaces**|`PascalCase`|`StringBuilder`, `Runnable`|
|**Methods**|`camelCase` ‚Äî verbs|`getName()`, `calculateSalary()`|
|**Variables / Fields**|`camelCase` ‚Äî nouns|`userName`, `totalPrice`|
|**Constants** (`static final`)|`UPPER_SNAKE_CASE`|`MAX_RETRIES`, `DEFAULT_TIMEOUT`|
|**Generic type parameters**|Single uppercase letter|`T`, `E`, `K`, `V`|
|**Enum constants**|`UPPER_SNAKE_CASE`|`PENDING`, `MAX_RETRIES`|

Breaking convention compiles but makes you an outlier ‚Äî every Java codebase, library, and IDE expects these.

## Comments

### Single-line `//`

```java
// TODO: validate input
int x = 5;  // inline comment at end of statement
```

### Multi-line `/* */`

```java
/*
 * Calculates the daily interest accrued on this account.
 * Formula: (principal * rate) / 365
 */
double dailyInterest() { ... }

int x = /* inline block comment */ 5;   // valid but rare
```

### Javadoc `/** */`

Special multi-line comment that the `javadoc` tool extracts to generate API docs.

```java
/**
 * Transfers funds from this account to the target.
 *
 * @param target the destination account ‚Äî must not be null
 * @param amount amount to transfer in smallest currency unit (e.g., cents)
 * @return true if transfer succeeded, false if insufficient funds
 * @throws IllegalArgumentException if amount is negative
 * @since 1.2
 */
public boolean transfer(Account target, long amount) { ... }
```

Common Javadoc tags:

|Tag|Use|
|---|---|
|`@param`|Describe each parameter|
|`@return`|Describe the return value|
|`@throws`|Exceptions the method may throw|
|`@since`|Version this was introduced|
|`@deprecated`|Mark as deprecated, say what to use instead|
|`@see`|Link to related class / method|
|`{@link ClassName}`|Inline link to another API element|

Javadoc comments **must** appear immediately before the class, method, or field they describe.

## Packages and Imports

### `package` Statement

First non-comment line in a file. Groups related classes into a namespace (and maps to the directory structure on disk):

```java
package com.example.orders;   // ‚Üí file must live in com/example/orders/ folder

public class OrderService { }
```

- Only **one** `package` per file
- Package name is **lowercase, dot-separated**
- Universal convention: reverse your organization's domain (`com.company.project.module`)
- Classes in the same package can see each other's **package-private** members (no modifier)

### `import` Statement

Appears after `package`, before class declaration. Lets you use a class by its short name instead of fully qualified name:

```java
package com.example.orders;

import java.util.List;
import java.util.ArrayList;

public class OrderService {
    private List<Order> orders = new ArrayList<>();   // instead of java.util.List
}
```

Wildcard import (brings in ALL classes from a package):

```java
import java.util.*;      // imports List, ArrayList, Map, HashMap, etc.
```

Wildcard imports are concise but can cause name conflicts. Most codebases use explicit imports (and IDEs auto-manage them).

**Classes that don't need import** ‚Äî `java.lang` classes (`String`, `Integer`, `Object`, `System`, `Math`, etc.) are always available, and classes in the same package can see each other directly.

### Static Imports

Import static members (fields, methods) so you can use them **without the class name prefix**:

```java
// Without static import
double result = Math.sqrt(Math.pow(x, 2) + Math.pow(y, 2));

// With static import
import static java.lang.Math.sqrt;
import static java.lang.Math.pow;

double result = sqrt(pow(x, 2) + pow(y, 2));   // cleaner
```

Common uses:

```java
import static java.util.Arrays.asList;
import static org.junit.jupiter.api.Assertions.*;   // junit assertions
```

Use sparingly ‚Äî overuse reduces readability (reader can't tell where `sqrt` comes from).

### Java Library Packages

The standard library is organized into packages. Common ones:

|Package|Contents|
|---|---|
|`java.lang`|`String`, `Integer`, `Object`, `System`, `Math`, `Thread`, `Throwable` ‚Äî auto-imported|
|`java.util`|`List`, `Map`, `Set`, `ArrayList`, `HashMap`, `Collections`, `Random`, `Scanner`|
|`java.io`|Classic I/O ‚Äî `File`, `InputStream`, `OutputStream`, `Reader`, `Writer`|
|`java.nio`|New I/O ‚Äî `Path`, `Files`, buffers, channels|
|`java.time`|Modern date/time (Java 8+) ‚Äî `LocalDate`, `LocalDateTime`, `Duration`|
|`java.util.concurrent`|Concurrency utilities ‚Äî `ExecutorService`, `Future`, `ConcurrentHashMap`|
|`java.util.stream`|Streams API|
|`java.net`|Networking ‚Äî `URL`, `Socket`, `HttpURLConnection`|
|`java.sql`|JDBC ‚Äî `Connection`, `Statement`, `ResultSet`|

## Commonly Asked Gotchas

1. **One public class per file** ‚Äî the filename must match the public class exactly (case-sensitive). `Foo.java` can have one `public class Foo` and any number of non-public classes.

2. **`String[] args` is never null** ‚Äî it's an empty array `new String[0]` if nothing was passed. `args.length` is safe to check.

3. **Case matters everywhere** ‚Äî `String` ‚âÝ `string`, `MyClass` ‚âÝ `myclass`. Even filenames must match public class case on case-sensitive filesystems (Linux).

4. **Javadoc marker** ‚Äî `/**` (two asterisks), not `/*`. A single asterisk is just a regular block comment.

5. **Wildcard imports don't cascade** ‚Äî `import java.util.*` does NOT import `java.util.concurrent.*`. Each sub-package needs its own import.

6. **`$` in identifiers is legal but reserved convention** ‚Äî the Java community reserves `$` for machine-generated code (compiler-generated inner class names like `Outer$Inner`). Don't use `$` in hand-written code.

7. **`package` must be the first line** ‚Äî only blank lines and comments can precede it. `package` after any other code is a compile error.

8. **Static import vs regular import are different namespaces** ‚Äî `import java.util.Arrays` brings in the class `Arrays`; `import static java.util.Arrays.asList` brings in the static method `asList`. They don't interfere but they're different.
