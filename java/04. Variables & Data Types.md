## Variables

### Declaration and Initialization

```java
// Declaration only — gets default value if instance/static; unassigned for locals
int count;

// Declaration + initialization
int count = 0;
String name = "Alice";

// Multiple in one statement
int x = 1, y = 2, z = 3;

// Chained assignment
int a, b;
a = b = 10;             // both set to 10
```

### Scope and Lifetime

Three scopes control where a variable is visible and how long it lives.

|Scope|Where declared|Default value?|Lifetime|
|---|---|---|---|
|**Local**|Inside a method / block|❌ NO — must initialize before use|Until the block exits|
|**Instance**|Inside a class (not `static`)|✅ Yes (0, null, false)|Same as the object|
|**Static (class)**|Inside a class with `static`|✅ Yes (0, null, false)|Same as the class (program lifetime)|

```java
class Employee {
    static int count;           // static — lives for program duration
    String name;                // instance — lives with the object

    void greet() {
        int tmp;                // local — lives only in greet()
        // System.out.println(tmp); // ❌ COMPILE ERROR — local vars must be initialized before read
    }
}
```

> **Key rule**: instance and static variables **auto-initialize** to defaults. Local variables do NOT — the compiler will error if you try to read an uninitialized local.

Full coverage: [12. Classes, Objects & OOP Foundations](<12. Classes, Objects & OOP Foundations.md>).

### `final` Keyword (basics)

Makes a variable's reference unchangeable after initial assignment.

```java
final int MAX = 100;
MAX = 200;                 // ❌ COMPILE ERROR — can't reassign

final List<String> items = new ArrayList<>();
items.add("apple");        // ✅ allowed — modifying the object, not the reference
items = new ArrayList<>(); // ❌ COMPILE ERROR — can't reassign the reference
```

- `final` variables must be assigned **exactly once** (at declaration or in constructor for instance fields)
- `final` prevents **reference reassignment**, NOT object mutation
- For true immutability, the object itself must be immutable (e.g., `String`, `Integer`)

Full coverage: [18. Important Keywords (Deep Dive)](<18. Important Keywords (Deep Dive).md>).

### `var` Keyword — Local Variable Type Inference (Java 10+)

Compiler infers the type from the right-hand side. Only for **local variables**.

```java
var name = "Alice";                    // String
var count = 42;                        // int
var list = new ArrayList<String>();    // ArrayList<String>

// Can't use for these:
var x;                                 // ❌ no initializer — compiler can't infer
var fn = null;                         // ❌ null has no type
public var name = "...";               // ❌ not for instance/static fields
public void go(var x) { }              // ❌ not for method parameters
```

**Rules**:

- Only for locals (method / block scope)
- Must have initializer in the declaration
- Initializer must not be `null`
- Works with loops: `for (var i = 0; i < 10; i++) { }`
- Allowed in **lambda parameters** since Java 11: `(var x, var y) -> x + y`

**Style**: use `var` when the type is obvious from the right-hand side (`var list = new ArrayList<Order>();`). Avoid when it obscures the type (`var result = service.process(input);` — what IS `result`?).

## Primitive Data Types

Java has exactly 8 primitive types — built into the language, not objects.

|Type|Size|Range|Default|Wrapper|
|---|---|---|---|---|
|`byte`|1 byte (8 bits)|-128 to 127|`0`|`Byte`|
|`short`|2 bytes (16 bits)|-32,768 to 32,767|`0`|`Short`|
|`int`|4 bytes (32 bits)|-2,147,483,648 to 2,147,483,647|`0`|`Integer`|
|`long`|8 bytes (64 bits)|-9.2 × 10¹⁸ to 9.2 × 10¹⁸|`0L`|`Long`|
|`float`|4 bytes (32 bits)|IEEE 754 single precision (~7 digits)|`0.0f`|`Float`|
|`double`|8 bytes (64 bits)|IEEE 754 double precision (~15 digits)|`0.0`|`Double`|
|`char`|2 bytes (16 bits)|0 to 65,535 (Unicode — **unsigned**)|`'\u0000'`|`Character`|
|`boolean`|implementation-defined (typically 1 byte)|`true` / `false`|`false`|`Boolean`|

**Key points**:

- `char` is **unsigned** — all other numeric types are signed
- `int` is Java's default integer type (literal `42` is an `int`)
- `double` is Java's default floating-point type (literal `3.14` is a `double`)
- `float` literals need the `f` suffix: `3.14f`
- `long` literals larger than `int` range need the `L` suffix: `10000000000L`

```java
int age = 30;
long population = 7_800_000_000L;
float price = 19.99f;
double pi = 3.14159265358979;
char grade = 'A';
boolean active = true;
```

## Literals

Literal values you can write directly in code.

### Types

Four categories:

- **Integer** — `42`, `0x2A`, `0b101010`, `052`
- **Character** — `'A'`, `'\n'`, `'\u0041'` (unicode escape — same as `'A'`)
- **String** — `"hello"`, text blocks `"""..."""`
- **Boolean** — `true`, `false`

### Integer formats

```java
int decimal = 42;          // base 10
int hex     = 0x2A;        // base 16 — prefix 0x or 0X
int binary  = 0b101010;    // base 2 (Java 7+) — prefix 0b or 0B
int octal   = 052;         // base 8 — prefix 0 (rarely used, easy to mistake)
```

All four assign the value 42. Choose the format that reads best in context (hex for bit flags, binary for bit patterns, decimal for counts).

### Type suffixes

Literals have default types; use suffixes to override:

```java
int i = 100;                    // int by default
long l = 10_000_000_000L;       // L (or l) — long; without L, literal is int and overflows
float f = 3.14f;                // f (or F) — float; without f, 3.14 is double and won't fit
double d = 3.14;                // double by default
double d2 = 3.14d;              // d (or D) — optional; double is the default for floating-point
```

**Lowercase `l` vs uppercase `L`** — always use `L`. Lowercase looks like the digit `1`.

### Underscore separators (Java 7+)

Improves readability of long numbers:

```java
int million = 1_000_000;
long bigNumber = 1_000_000_000_000L;
int hexMask = 0xFF_00_FF_00;
int binary = 0b1010_0101_1100_0011;
```

Rules:

- Only BETWEEN digits — `1_000` ✅, `_1000` ❌, `1000_` ❌
- Compiler strips underscores before parsing
- Doesn't work in input strings: `Integer.parseInt("1_000")` ❌ fails

### Character literals

Single-quoted, single character OR escape sequence:

```java
char c1 = 'A';           // regular char
char c2 = '\n';          // newline escape
char c3 = '\u0041';  // Unicode escape — same as 'A' (U+0041)
char c4 = '\t';          // tab
char c5 = '\\';          // literal backslash
```

`char` is a numeric type under the hood — you can do arithmetic on it:

```java
char c = 'A';
int i = c;        // 65 (implicit widening char → int)
char next = (char)(c + 1);  // 'B'
```

### Escape sequences

|Escape|Meaning|
|---|---|
|`\n`|Newline|
|`\t`|Tab|
|`\r`|Carriage return|
|`\\`|Literal backslash|
|`\'`|Literal single quote|
|`\"`|Literal double quote|
|`\b`|Backspace|
|`\f`|Form feed|
|`\0`|Null character|
|`\uXXXX`|Unicode character (4 hex digits)|

## Wrapper Classes

For each primitive, Java has an **object wrapper class** — needed when you must use objects (Collections, generics, nullable values).

|Primitive|Wrapper|
|---|---|
|`byte`|`Byte`|
|`short`|`Short`|
|`int`|`Integer`|
|`long`|`Long`|
|`float`|`Float`|
|`double`|`Double`|
|`char`|`Character`|
|`boolean`|`Boolean`|

**When to use wrappers**:

- **Collections** — can't store primitives: `List<Integer>`, not `List<int>`
- **Nullability** — primitives can't be `null`; wrappers can (`Integer x = null`)
- **Number API** — methods like `Integer.parseInt()`, `Long.valueOf()`, `Integer.MAX_VALUE`
- **Generics** — type parameters are always reference types (`<T>` needs a class, not a primitive)

```java
// Boxing (primitive → wrapper)
int primitive = 42;
Integer wrapped = primitive;      // auto-boxing (Java 5+)

// Unboxing (wrapper → primitive)
Integer wrapped2 = 42;
int primitive2 = wrapped2;        // auto-unboxing

// Collections — must use wrappers
List<Integer> nums = new ArrayList<>();
nums.add(42);                     // auto-boxing: int → Integer
int first = nums.get(0);          // auto-unboxing: Integer → int
```

Full coverage: [Operators — Autoboxing](<05. Operators & Type System.md#Boxing, Unboxing, and Auto-Boxing>) and [Numbers — java.lang.Number](<10. Numbers, Math & Random.md#`java.lang.Number` — the numeric root>).

## Non-Primitive Data Types

Everything that isn't a primitive is a **reference type** — the variable holds a reference (pointer) to an object on the heap.

|Type|Description|Full coverage|
|---|---|---|
|**String**|Immutable character sequence|[9. Strings](<09. Strings.md>)|
|**Arrays**|Fixed-size ordered collection of same-type values|[8. Arrays](<08. Arrays.md>)|
|**Classes**|User-defined types — blueprint for objects|[12. Classes, Objects & OOP Foundations](<12. Classes, Objects & OOP Foundations.md>)|
|**Interfaces**|Contract of behaviors — no state, implementations provided by classes|[16. Abstraction & Interfaces](<16. Abstraction & Interfaces.md>)|

```java

int[] numbers = {1, 2, 3};                   // array
Employee emp = new Employee("Alice");         // class instance
```

## Commonly Asked Gotchas

1. **Local variables don't auto-init** — `int x; System.out.println(x);` is a compile error. Instance and static variables DO auto-init to zeros/nulls.

2. **Integer literals default to `int`** — `long l = 10000000000;` fails (literal overflows int). Must be `10000000000L`.

3. **Floating-point literals default to `double`** — `float f = 3.14;` is a compile error. Must be `3.14f`.

4. **`var` is not dynamic typing** — still statically typed, just the compiler infers the type. Once inferred, the variable's type is fixed.

5. **`char` is unsigned** — unique among Java numeric types. `char + char` returns `int` (with both promoted to int).

6. **Can't use primitives in Collections** — `List<int>` is invalid. Use `List<Integer>` (boxing happens automatically).

7. **`final` only prevents reassignment, not mutation** — `final List<String> items = new ArrayList<>(); items.add("x");` is perfectly legal.

8. **`var` + diamond operator** — with `var`, the diamond `<>` has no target type, so it falls back to `Object`. `var list = new ArrayList<>();` → `ArrayList<Object>`. Always spell out the type arg: `new ArrayList<String>()`.

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Local var default|None — must initialize before use|
|Instance/static var default|0, 0.0, false, null (auto-initialized)|
|`final`|Reassignment forbidden; object mutation still allowed|
|`var` (Java 10+)|Local-only type inference; no initializer = error|
|Primitives|8 types — `byte`, `short`, `int`, `long`, `float`, `double`, `char`, `boolean`|
|`int` default|Integer literals are `int` unless suffixed `L`|
|`double` default|Floating-point literals are `double` (explicit `d`/`D`); use `f`/`F` for `float`|
|`char`|16-bit unsigned — unique among numeric types|
|Integer formats|Decimal, `0x` hex, `0b` binary (Java 7+), `0` octal|
|Underscore separators|`1_000_000` — between digits only (Java 7+)|
|Wrappers|`Integer`, `Long`, `Double`, etc. — needed for Collections, nullability|
|Reference types|String, Arrays, Classes, Interfaces, Enums — variables hold pointers|
