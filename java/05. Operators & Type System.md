## 1. Operators

### Arithmetic Operators

```java
int a = 10, b = 3;
a + b    // 13   (addition)
a - b    // 7    (subtraction)
a * b    // 30   (multiplication)
a / b    // 3    (integer division — truncates decimal)
a % b    // 1    (modulus — remainder)

// Division gotcha
10 / 3     // 3 (int / int = int — decimal lost)
10.0 / 3   // 3.333... (double / int = double)
10 / 0     // ArithmeticException
10.0 / 0   // Infinity (not exception — double division by zero)
```

### Unary Operators

```java
int a = 5;
+a     // 5   (unary plus — rarely used)
-a     // -5  (unary minus — negation)
!true  // false (logical NOT)
~a     // -6  (bitwise complement — flips all bits)

// Increment / Decrement
a++    // post-increment: use value first, then increment
++a    // pre-increment: increment first, then use value
a--    // post-decrement
--a    // pre-decrement

// Classic gotcha
int x = 5;
int y = x++;    // y = 5, x = 6 (used then incremented)
int z = ++x;    // z = 7, x = 7 (incremented then used)
```

### Assignment Operators

```java
int a = 10;
a += 5;    // a = 15  (a = a + 5)
a -= 3;    // a = 12
a *= 2;    // a = 24
a /= 4;    // a = 6
a %= 4;    // a = 2

// Compound assignment does implicit casting
byte b = 10;
b = b + 1;     // ❌ COMPILE ERROR — b + 1 is int (type promotion)
b += 1;        // ✅ works — compound assignment auto-casts
b = (byte)(b + 1);  // ✅ explicit cast also works
```

### Relational (Comparison) Operators

```java
a == b    // equal to
a != b    // not equal to
a > b     // greater than
a < b     // less than
a >= b    // greater than or equal
a <= b    // less than or equal

// == for primitives → compares values
// == for objects → compares references (use .equals() for content)
```

### Logical Operators

```java
true && false   // false (AND — both must be true)
true || false   // true  (OR — at least one true)
!true           // false (NOT)

// Short-circuit evaluation
// && → if left is false, right is NOT evaluated
// || → if left is true, right is NOT evaluated

int x = 0;
if (x != 0 && 10 / x > 2) { }   // safe — 10/x never executes when x=0

// Non-short-circuit logical operators (also work as bitwise for integers)
true & false    // false (both sides always evaluated)
true | false    // true  (both sides always evaluated)
```

### Bitwise Operators

```java
int a = 5, b = 3;   // a = 0101, b = 0011 in binary
a & b    // 1  (0001) — AND: both bits 1
a | b    // 7  (0111) — OR: either bit 1
a ^ b    // 6  (0110) — XOR: bits differ
~a       // -6        — NOT: flip all bits

// Common use cases
n & 1          // check odd/even (0 = even, 1 = odd)
n ^ n          // 0 (XOR with itself = 0 — used to find unique element)
flags & MASK   // check if specific bit is set
```

### Shift Operators

```java
int a = 8;     // 1000 in binary
a << 1    // 16 (10000) — left shift: multiply by 2
a >> 1    // 4  (0100)  — right shift: divide by 2 (preserves sign)
a >>> 1   // 4  (0100)  — unsigned right shift: fills with 0 (ignores sign)

// Negative number difference
int neg = -8;
neg >> 1    // -4  (sign preserved)
neg >>> 1   // large positive number (sign bit filled with 0)

// Shift distance is masked: mod 32 for int, mod 64 for long
int x = 1;
x << 32    // 1, NOT 0! (32 % 32 = 0, so effectively x << 0)

// Quick multiply/divide by powers of 2 (positive numbers only — negative numbers differ due to sign extension)
n << 2    // n * 4
n >> 3    // ≈ n / 8
```

### Ternary Operator

```java
int max = (a > b) ? a : b;    // if true → a, else → b

// Nested (avoid — hard to read)
String result = (score >= 90) ? "A" : (score >= 80) ? "B" : "C";
```

### `instanceof` Operator

```java
// Checks if object is an instance of a class/interface
String s = "hello";
s instanceof String     // true
s instanceof Object     // true (all objects extend Object)
s instanceof Integer    // false

null instanceof String  // always false — null is not an instance of anything

// Pattern matching (Java 16+)
if (obj instanceof String s) {
    System.out.println(s.length());   // s already cast — no explicit cast needed
}
```

## 2. Operator Precedence (Simplified)

From highest to lowest:

```
1. Postfix:       a++, a--
2. Unary:         ++a, --a, +a, -a, ~, !
3. Multiplicative: *, /, %
4. Additive:      +, -
5. Shift:         <<, >>, >>>
6. Relational:    <, >, <=, >=, instanceof
7. Equality:      ==, !=
8. Bitwise:       &, ^, |
9. Logical:       &&, ||
10. Ternary:      ? :
11. Assignment:   =, +=, -=, etc.
```

**When in doubt, use parentheses** — readability > memorizing precedence.

## 3. Typecasting

### Widening (Implicit) — small → large, no data loss

```java
byte → short → int → long → float → double
char → int

int i = 100;
long l = i;       // ✅ automatic — int fits in long
double d = i;     // ✅ automatic — int fits in double (100.0)
```

### Narrowing (Explicit) — large → small, possible data loss

```java
double d = 9.99;
int i = (int) d;      // 9 — decimal truncated (not rounded)

long l = 130;
byte b = (byte) l;    // -126 — overflow! (byte range: -128 to 127)

int big = 300;
byte small = (byte) big;  // 44 — data loss
```

### char ↔ int

```java
char c = 'A';
int i = c;           // 65 (ASCII value — widening)
char c2 = (char) 66; // 'B' (narrowing)
```

## 4. Type Promotion

In expressions, smaller types are **automatically promoted** to larger types before operations.

```java
byte a = 10;
byte b = 20;
// byte c = a + b;    // ❌ COMPILE ERROR — a + b promotes to int
int c = a + b;        // ✅ result is int

// Rules:
// 1. byte, short, char → always promoted to int in expressions
// Expression result is always AT LEAST int, even if all operands are byte/short/char
// 2. If any operand is long → whole expression becomes long
// 3. If any operand is float → whole expression becomes float
// 4. If any operand is double → whole expression becomes double

byte b = 10;
short s = 20;
int result = b + s;      // both promoted to int

long l = 100L;
long result = b + l;     // b promoted to long

float f = 1.5f;
float result = b + f;    // b promoted to float

double d = 2.5;
double result = f + d;   // f promoted to double
```

## 5. Boxing, Auto-Boxing, and Unboxing

### What it is

```java
// Boxing — primitive → wrapper object (manual, pre-Java 5)
Integer obj = Integer.valueOf(5);    // explicit boxing

// Auto-boxing — primitive → wrapper object (automatic, Java 5+)
Integer obj = 5;                     // compiler does Integer.valueOf(5)

// Unboxing — wrapper object → primitive (automatic)
int val = obj;                       // compiler does obj.intValue()
```

### Why auto-boxing was added (the pre-Java 5 pain)

- **Primitives and objects lived in separate worlds** — `int` was not an `Object`. You couldn't put an `int` in any Collection (`List`, `Map`, `Set`) because Collections stored `Object`.
- **Every primitive/wrapper conversion was manual boilerplate** — every add to a list: `list.add(Integer.valueOf(5))`. Every read back to primitive: `int x = ((Integer)list.get(0)).intValue()`. Verbose and easy to get wrong.
- **Generics made the problem louder** — when generics arrived in Java 5 (`List<Integer>`), writing `list.add(Integer.valueOf(5))` every time was friction against the very thing generics made convenient.
- **Workarounds were ugly** — code defensively wrapped every primitive in conversions, or created parallel APIs (`IntArrayList` libraries). Java stdlib didn't have primitive collections until Java 8's `IntStream`.

### How auto-boxing solves it

- **Compiler does the conversion implicitly** — `list.add(5)` becomes `list.add(Integer.valueOf(5))`. `int x = list.get(0)` becomes `int x = list.get(0).intValue()`. Source code is clean.
- **Works everywhere primitives meet objects** — Collections, method arguments, assignments, comparisons. Transparent.
- **Paired with wrapper caching** — `Integer.valueOf()` caches values from -128 to 127, so small autoboxed ints don't create new objects each time. Free memory optimization.

### Gotchas it introduced (important for 3+ YOE)

- **`NullPointerException` on unboxing** — `Integer x = null; int y = x;` throws NPE because null → `null.intValue()`. A hidden NPE in innocuous-looking code.
- **`==` compares references, not values** — `Integer a = 128; Integer b = 128; a == b` is `false` (outside cache range). `Integer a = 127; Integer b = 127; a == b` is `true` (cached same reference). Use `.equals()` always.
- **Hidden performance cost in hot paths** — every autobox allocates an `Integer` object (unless cached). In tight loops over numbers, this can dominate runtime. Fix: use primitive streams (`IntStream`) or primitive arrays.

### All primitive ↔ wrapper pairs

|Primitive|Wrapper|
|---|---|
|`byte`|`Byte`|
|`short`|`Short`|
|`int`|`Integer`|
|`long`|`Long`|
|`float`|`Float`|
|`double`|`Double`|
|`char`|`Character`|
|`boolean`|`Boolean`|

### Where auto-boxing/unboxing happens

```java
// Collections (can't store primitives — auto-boxing happens)
List<Integer> list = new ArrayList<>();
list.add(5);              // auto-boxing: 5 → Integer.valueOf(5)
int val = list.get(0);    // unboxing: Integer → int

// Method calls
void process(Integer n) { }
process(5);               // auto-boxing

// Expressions
Integer a = 10;
Integer b = 20;
int sum = a + b;          // unboxing both, then adding

// Comparisons — CAREFUL!
Integer x = 127, y = 127;
x == y     // true — Integer cache (-128 to 127)

Integer x = 128, y = 128;
x == y     // false — outside cache, different objects!
x.equals(y) // true — always use equals() for wrapper comparison
```

### Integer Cache (-128 to 127)

```java
// JVM caches Integer objects for values -128 to 127
// valueOf() returns cached object for these values
Integer a = 127;     // cached
Integer b = 127;     // same cached object
a == b               // true

Integer c = 128;     // new object (outside cache)
Integer d = 128;     // another new object
c == d               // false — different objects!

// ⚠️ new Integer() NEVER uses cache — also deprecated since Java 9, use Integer.valueOf()
Integer e = new Integer(127);
Integer f = new Integer(127);
e == f               // false — always new objects, bypasses cache

// Wrapper == primitive → unboxing happens (value comparison)
Integer g = 100;
int h = 100;
g == h               // true — g is unboxed to int, then compared by value

// Same caching applies to: Byte, Short, Long, Character (0-127)
// Boolean caches TRUE and FALSE
```

## 6. Commonly Asked Gotchas

1. **Integer division truncates** — `10 / 3 = 3` not `3.33`. Cast to double first: `(double) 10 / 3`.
    
2. **Compound assignment auto-casts** — `b += 1` works for byte but `b = b + 1` doesn't (type promotion to int).
    
3. **`==` for wrapper objects** — compares references, not values. Use `equals()`. Exception: Integer cache (-128 to 127).
    
4. **Short-circuit vs non-short-circuit** — `&&`/`||` short-circuit (skip right if result known). `&`/`|` always evaluate both sides.
    

```java
// && is safe — right side skipped when left is false
if (x != 0 && 10 / x > 2) { }   // ✅ safe

// & is dangerous — both sides always evaluated
if (x != 0 & 10 / x > 2) { }    // ❌ crashes when x = 0
```

5. **Unboxing null throws NPE:**

```java
Integer x = null;
int y = x;           // ❌ NullPointerException — unboxing null
if (x > 0) { }       // ❌ NPE — unboxing for comparison
```

6. **`double / 0` = Infinity, `int / 0` = ArithmeticException** — floating point has special values (Infinity, NaN), integers don't.
    
7. **Type promotion in ternary:**
    

```java
Object obj = true ? new Integer(1) : new Double(2.0);
// Expression type → Double (common type between Integer and Double)
// true branch: 1 becomes 1.0 (promoted to Double)
// false branch: would be 2.0
// Result: Double(1.0)
```

8. **Post-increment in expressions:**

```java
int a = 5;
a = a++;     // a = 5! Post-increment returns old value (5), then a becomes 6,
             // but assignment overwrites with 5. Increment happens but gets overwritten.
```

9. **`char` is numeric** — can do arithmetic: `'A' + 1 = 66` (not `'B'`). Cast to char for letter: `(char)('A' + 1) = 'B'`.
    
10. **`float` precision loss:**
    

```java
float f = 0.1f + 0.2f;
// f = 0.30000001 (not 0.3) — binary representation can't express all decimals exactly
// Use BigDecimal for precision-critical calculations (money, etc.)
```

11. **Auto-boxing in loops is expensive:**

```java
// ❌ Bad — auto-boxes every iteration
Long sum = 0L;
for (long i = 0; i < 1000000; i++) {
    sum += i;    // each iteration: unbox Long→long, add, box long→Long = new object + GC pressure
}

// ✅ Good — use primitive
long sum = 0L;
```

12. **`double / 0` = Infinity, `int / 0` = ArithmeticException.** `0.0 / 0.0` = `NaN`. Floating point has special values because they use binary representation — many decimal values can't be represented exactly.

```java
Double.NaN == Double.NaN     // false! NaN is not equal to anything, including itself
Double.NaN != Double.NaN     // true! NaN is unordered
Double.isNaN(Double.NaN)     // true — use this to check
```

13. **Null-safe equals for wrappers** — use `Objects.equals(x, y)` instead of `x.equals(y)` to avoid NPE when x is null.
