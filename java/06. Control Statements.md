## Conditional Statements

### `if` / `if-else` / `else-if`

```java
if (condition) {
    // runs if condition is true
}

if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
} else if (score >= 70) {
    grade = "C";
} else {
    grade = "F";
}
```

- Condition must be a `boolean` (or unbox to boolean)
- Braces are optional for single statements but **recommended always** — prevents dangling-else bugs
- `else if` isn't a separate keyword — it's `else` followed by another `if`

```java
// ❌ Dangerous — no braces; indentation lies
if (flag)
    doThing();
    doOther();    // ← looks like it's in the `if`, but runs UNCONDITIONALLY

// ✅ Safe — braces match the real scope
if (flag) {
    doThing();
    doOther();
}
```

### `switch` Statement

Original form — requires `break`, allows fall-through.

```java
switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:                             // fall-through — no break
    case 4:
    case 5:
        System.out.println("Midweek");  // runs for 3, 4, AND 5
        break;
    default:
        System.out.println("Unknown");
}
```

**Supports**: `byte`, `short`, `int`, `char` (+ wrappers), `String` (Java 7+), `enum`.

**Fall-through** — without `break`, execution continues to the next case. Useful for multi-value grouping (as shown above) but error-prone when forgotten. Modern `switch` expressions eliminate fall-through entirely.

### `switch` Expressions (Java 14+)

Modern form — arrow syntax, no fall-through, returns a value.

```java
String name = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3, 4, 5 -> "Midweek";               // multi-value case
    default -> {
        log.warn("Unknown day: " + day);
        yield "Unknown";                      // `yield` returns from a block arm
    }
};
```

**Key features**:

- Arrow (`->`) replaces `case N:` + `break`
- No fall-through — each arm runs only its right-hand side
- Multi-value cases — `case 1, 2, 3 ->`
- Returns a value — assignable to a variable
- Exhaustiveness check for `enum` / `sealed` — no `default` needed

**Single-expression arm vs block arm.** An arm can be either a single expression (`case 1 -> "Monday";`) where the expression itself IS the result, OR a block `{ ... }` for when you need multiple statements. In a block arm, you must use `yield` to produce the value — analogous to `return` in a method body. Without `yield`, the compiler doesn't know what the block evaluates to.

### Pattern Matching in `switch` (Java 21+)

Match on type + destructure in one step. Replaces `if-else` chains of `instanceof` + cast.

```java
String desc = switch (obj) {
    case Integer i                                 -> "int: " + i;
    case String s                                  -> "string of length " + s.length();
    case Point(int x, int y) when x == 0 && y == 0 -> "origin";    // record pattern + guard
    case Point(int x, int y)                       -> x + ", " + y;
    case null                                      -> "null!";
    default                                        -> "unknown type";
};
```

- Type patterns — `case Type var ->` captures the matched value
- Guards — `when <condition>` adds extra checks
- Record patterns — destructure records into their components
- `case null` explicitly handles null; otherwise a null value throws NPE

### `switch` Statement vs `switch` Expression

||`switch` Statement|`switch` Expression (Java 14+)|
|---|---|---|
|Syntax|`case X: ... break;`|`case X -> ...`|
|Fall-through|Yes (requires `break`)|No|
|Returns value|No (`switch` is a statement)|Yes (expression)|
|Exhaustiveness check|No|Yes (for enum / sealed)|
|Multi-value case|Multiple `case` labels|`case 1, 2, 3 -> ...`|
|Use when|Legacy code, side-effect-only branching|Producing a value; modern code|

## Iterative Statements

### `for`

Classic C-style loop — `init; condition; update`. All three parts are optional (`for (;;) { }` is a legal infinite loop).

```java
for (int i = 0; i < 10; i++) {
    System.out.println(i);
}
```

### `while`

Condition-first — checks BEFORE body, may run zero times. Use when iteration count isn't known upfront.

```java
while (!queue.isEmpty()) {
    process(queue.poll());
}
```

### `do-while`

Condition-last — runs body AT LEAST ONCE, then checks.

```java
int attempts = 0;
do {
    System.out.println("Try " + attempts);
    attempts++;
} while (attempts < 3);
```

Use when the body must run before the check makes sense (user input validation, retry logic).

### `for` vs `while` vs `do-while`

||`for`|`while`|`do-while`|
|---|---|---|---|
|Check order|Before body|Before body|After body|
|Min iterations|0|0|1|
|Best for|Known iteration count|Unknown iteration count, may skip|Must run at least once|

### Enhanced `for-each` (Java 5+)

Iterate over arrays or anything implementing `Iterable`. No index management, no off-by-one errors.

```java
List<String> names = List.of("Alice", "Bob", "Charlie");
for (String name : names) {
    System.out.println(name);
}

// Works on arrays and any Iterable; use entrySet() for maps
Map<String, Integer> scores = Map.of("Alice", 90, "Bob", 85);
for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}
```

**Limits** (use classic `for` instead):

- Can't modify the collection during iteration (throws `ConcurrentModificationException`)
- Can't access the index
- Can't iterate backwards
- Can't iterate two collections in parallel

## Branching Statements

### `break`

Exits the nearest enclosing loop or `switch`.

```java
for (int i = 0; i < 100; i++) {
    if (i == 50) break;   // exits the loop
    System.out.println(i);
}
```

### `continue`

Skips the rest of the current iteration and moves to the next.

```java
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) continue;   // skip even numbers
    System.out.println(i);       // prints 1, 3, 5, 7, 9
}
```

### `return`

Exits the current method, optionally returning a value.

```java
boolean isValid(String s) {
    if (s == null) return false;
    if (s.isEmpty()) return false;
    return s.length() < 100;
}
```

### Labeled `break` / `continue`

For nested loops, `break` / `continue` targets only the innermost loop. Labels let you jump out of outer loops.

```java
search:
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        if (matrix[i][j] == target) {
            System.out.println("Found at " + i + "," + j);
            break search;   // exits BOTH loops
        }
    }
}
```

Labels are legal but rare — usually a sign the logic should be extracted to its own method with an early `return`.

## Commonly Asked Gotchas

1. **Missing `break` in classic switch** — silent fall-through. Easy to miss:

   ```java
   switch (x) {
       case 1:
           doA();
           // forgot break
       case 2:
           doB();     // runs when x is 1 too — often unintended
           break;
   }
   ```

   Switch expressions (Java 14+) eliminate this.

2. **`if (a = true)` vs `if (a == true)`** — first is assignment (compiles, always true), second is comparison. Compiler catches this only when `a` is a non-boolean.

3. **`break` in nested loops only breaks the innermost** — need a label to break outer loops.

4. **`ConcurrentModificationException` in for-each** — modifying a collection while iterating via for-each throws at runtime. Use `Iterator.remove()` or collect changes and apply after.

   ```java
   for (String s : list) {
       if (s.isEmpty()) list.remove(s);   // ❌ throws ConcurrentModificationException
   }
   ```

5. **Switch expression without `default`** — required UNLESS switching on an enum and covering all cases, or on a sealed type and covering all permits. Otherwise compile error.

6. **Integer overflow in loop counters** — `for (int i = 0; i <= Integer.MAX_VALUE; i++)` — infinite loop. After `Integer.MAX_VALUE`, `i++` wraps to `Integer.MIN_VALUE`.

7. **Pattern matching requires exhaustiveness** — switch on a sealed hierarchy must cover every permitted subtype, or compile error.

8. **Enum case labels must NOT be qualified** — the compiler requires the bare constant name:

    ```java
    enum Color { RED, GREEN, BLUE }
    switch (c) {
        case Color.RED: ...   // ❌ COMPILE ERROR — can't qualify with enum name
        case RED: ...         // ✅ correct
    }
    ```

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|`if / else if / else`|Braces recommended even for single-line bodies|
|`switch` Statement|Needs `break`; supports fall-through; no return value|
|`switch` expression (Java 14+)|Arrow syntax, no fall-through, returns value, exhaustive check|
|Pattern matching `switch` (Java 21+)|`case Type var -> ...`, with `when` guards and record destructuring|
|`for`|Check-first; known count|
|`while`|Check-first; may skip|
|`do-while`|Check-last; min 1 iteration|
|Enhanced `for-each`|Clean syntax but can't modify collection mid-iteration|
|`break`|Exits nearest loop / switch|
|`continue`|Skips rest of current iteration|
|`return`|Exits current method|
|Labeled `break` / `continue`|Jumps out of a specific outer loop|
|`yield` (switch expression)|Return a value from a block arm|
