## Reading User Input

### `Scanner` Class

The most common way to read input from the console — simple, flexible, handles parsing.

```java
import java.util.Scanner;

public class ScannerDemo {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter name: ");
        String name = sc.nextLine();
        System.out.print("Enter age: ");
        int age = sc.nextInt();
        sc.close();
    }
}
```

#### What is `System.in`?

`System.in` is a `public static final InputStream` — the standard input stream provided by the JVM (keyboard by default).

- Type: `java.io.InputStream` (a byte stream)
- Scanner wraps it and provides typed parsing methods (`nextInt`, `nextLine`, etc.)

#### Common Scanner methods

```java
sc.next()          // next whitespace-delimited token (word)
sc.nextLine()      // rest of the current line (as String)
sc.nextInt()       // next token parsed as int
sc.nextLong()      // next token parsed as long
sc.nextDouble()    // next token parsed as double
sc.nextBoolean()   // next token parsed as boolean
sc.nextByte()      // byte
sc.nextShort()     // short
sc.nextFloat()     // float

sc.hasNext()       // is there more input? (used in loops)
sc.hasNextInt()    // is the next token an int?
sc.hasNextLine()   // is there another line?
```

#### The `nextLine()` gotcha

The classic Scanner trap — typed-value methods (`nextInt`, `nextDouble`, etc.) **leave the trailing newline** in the buffer. A subsequent `nextLine()` reads that empty newline, not the value you wanted.

```java
Scanner sc = new Scanner(System.in);
System.out.print("Age: ");
int age = sc.nextInt();           // reads "30" — leaves \n in buffer
System.out.print("Name: ");
String name = sc.nextLine();      // reads the empty \n — doesn't wait for input!
```

**Fix**: consume the leftover newline explicitly:

```java
int age = sc.nextInt();
sc.nextLine();                    // consume the stray newline
String name = sc.nextLine();      // now reads actual name input
```

**Alternative fix**: read everything as strings and parse manually:

```java
int age = Integer.parseInt(sc.nextLine().trim());
String name = sc.nextLine();
```

#### Parsing Input with Wrapper Classes

Reading "42" from input gives you a String. Converting to int uses wrapper class static methods:

```java
String input = sc.nextLine();
int n = Integer.parseInt(input);              // throws NumberFormatException if invalid
long l = Long.parseLong(input);
double d = Double.parseDouble(input);
boolean b = Boolean.parseBoolean(input);      // "true" → true; anything else → false
```

These `parseX` methods are **static methods on wrapper classes** because primitives themselves can't have methods. The wrapper class is the natural home for these utility methods.

### `Console` Class (for password input)

`System.console()` returns a `java.io.Console` that offers a `readPassword()` method — reads input **without echoing to the screen**.

```java
Console console = System.console();
if (console != null) {
    char[] password = console.readPassword("Password: ");
    // use password...
    Arrays.fill(password, ' ');                // clear after use for security
} else {
    // Console is null when running inside IDEs or pipes — fallback to Scanner
    System.out.println("No console available");
}
```

- Returns `char[]`, NOT `String` — so you can zero it out after use (Strings are immutable and linger in memory until GC)
- `System.console()` returns `null` when stdin is redirected or when run from most IDEs — always check

## Printing Output

### `PrintStream` Class

`System.out` and `System.err` are `java.io.PrintStream` instances:

- **`System.out`** — standard output (stdout), line-buffered to the terminal
- **`System.err`** — standard error (stderr), usually unbuffered, red in most IDEs

### `println()`, `print()`, `printf()`

```java
System.out.println("Hello");       // prints "Hello\n"
System.out.print("Hello");         // prints "Hello" (no newline)
System.out.println();              // just a newline
System.out.println(42);            // converts via String.valueOf
System.out.println(new int[]{1,2,3});  // prints "[I@hash" — use Arrays.toString()

System.out.printf("Name: %s, Age: %d%n", "Alice", 30);
// prints "Name: Alice, Age: 30\n"
```

**`%n` vs `\n`**: `%n` is platform-specific line separator (`\r\n` on Windows, `\n` on Unix). `\n` is always `\n`. Prefer `%n` in `printf` for portability.

### Format specifiers

```java
System.out.printf("%d", 42);            // 42                (integer)
System.out.printf("%,d", 1_000_000);    // 1,000,000         (grouping)
System.out.printf("%5d", 42);           //    42             (width 5, right-aligned)
System.out.printf("%-5d|", 42);         // 42   |            (left-aligned)
System.out.printf("%05d", 42);          // 00042             (zero-padded)

System.out.printf("%f", 3.14159);       // 3.141590
System.out.printf("%.2f", 3.14159);     // 3.14              (2 decimal places)
System.out.printf("%10.2f", 3.14);      //       3.14        (width 10, 2 decimals)

System.out.printf("%s", "hello");       // hello
System.out.printf("%10s", "hi");        //         hi        (right-aligned in width 10)
System.out.printf("%-10s|", "hi");      // hi        |       (left-aligned)

System.out.printf("%c", 'A');           // A
System.out.printf("%b", true);          // true
System.out.printf("%%");                // %                 (literal percent sign)
```

## Commonly Asked Gotchas

1. **Scanner's `nextLine()` gotcha** — typed-value methods leave `\n` in the buffer; the next `nextLine()` reads the empty newline. Always consume the newline with an extra `sc.nextLine()` or read everything as strings and parse manually.

2. **`System.console()` can be null** — when stdin is redirected (piped from a file), or when running from most IDEs. Always null-check before calling `readPassword`.

3. **Password as `char[]` not `String`** — readPassword returns `char[]` so you can zero it out after use. Strings are immutable and remain in memory until GC, exposing passwords to memory dumps.

4. **`println(int[])` prints `[I@hash`**, not the elements. Use `Arrays.toString(arr)` for 1D or `Arrays.deepToString(arr)` for multi-dimensional.

5. **`%n` vs `\n`** — prefer `%n` in `printf` for cross-platform correctness. Matters on Windows for text files consumed by other tools.

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|`Scanner`|Wrap `System.in`; typed-parse methods (`nextInt`, `nextLine`, etc.)|
|`System.in`|`InputStream` — the underlying byte stream|
|`nextLine()` gotcha|Typed reads leave `\n`; extra `nextLine()` or parse strings manually|
|Parse methods|`Integer.parseInt(s)`, `Double.parseDouble(s)`, etc. — on wrapper classes|
|`Console`|`readPassword()` for hidden input; returns `char[]` (not `String`)|
|`System.console()`|Null in IDEs and when stdin is piped — check before use|
|`System.out`|`PrintStream` — `println`, `print`, `printf`|

|Print an array|`Arrays.toString(arr)`, not `println(arr)`|
|`System.err`|Unbuffered; for errors — prints immediately|
