## What & Why

Arrays are **fixed-size, ordered** containers that store elements of the **same type** in contiguous memory (for primitives) or contiguous references (for objects). Index-based access (starts at 0).

```java
int[] arr = new int[5];     // [0, 0, 0, 0, 0] — default values
arr[0] = 10;
arr[4] = 50;
System.out.println(arr.length);  // 5 — length is a field, not a method
```

## Declaration & Initialization

```java
// Declaration
int[] arr;        // preferred style
int arr[];        // valid but less common (C-style)

// Initialization
int[] arr = new int[5];                   // default values (0 for int)
int[] arr = {1, 2, 3, 4, 5};             // literal — size inferred
int[] arr = new int[]{1, 2, 3, 4, 5};    // explicit type + values

// Default values
int[]     → 0
double[]  → 0.0
boolean[] → false
char[]    → '\u0000'
String[]  → null  (reference types default to null)
```

## Multi-Dimensional Arrays

```java
// 2D array (matrix)
int[][] matrix = new int[3][4];     // 3 rows, 4 columns
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

matrix[0][0] = 1;     // first row, first column
matrix.length;         // 3 (number of rows)
matrix[0].length;      // 3 (number of columns in first row)

// Jagged array — rows can have different lengths
int[][] jagged = new int[3][];
jagged[0] = new int[2];     // row 0 has 2 elements
jagged[1] = new int[4];     // row 1 has 4 elements
jagged[2] = new int[1];     // row 2 has 1 element

// Iterating 2D array
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        System.out.print(matrix[i][j] + " ");
    }
}

// for-each
for (int[] row : matrix) {
    for (int val : row) {
        System.out.print(val + " ");
    }
}
```

## `Arrays` Utility Class (java.util.Arrays)

```java
int[] arr = {5, 3, 1, 4, 2};

// Sort
Arrays.sort(arr);                    // [1, 2, 3, 4, 5] — in-place, ascending
Arrays.sort(arr, 1, 4);             // sort index 1 to 3 only

// Sort objects with Comparator (requires wrapper type)
Integer[] boxedArr = {5, 3, 1, 4, 2};
Arrays.sort(boxedArr, Comparator.reverseOrder());   // [5, 4, 3, 2, 1]

// Search (array must be sorted first)
Arrays.sort(arr);
int index = Arrays.binarySearch(arr, 3);   // returns index, or negative if not found

// Fill
Arrays.fill(arr, 0);                // all elements become 0
Arrays.fill(arr, 1, 3, 99);         // index 1 to 2 become 99

// Compare
Arrays.equals(arr1, arr2);          // true if same length + same elements
Arrays.deepEquals(arr2D_1, arr2D_2); // for multi-dimensional arrays

// Copy
int[] copy     = Arrays.copyOf(arr, arr.length);    // full copy
int[] extended = Arrays.copyOf(arr, 10);            // copy + extend (padded with 0)
int[] slice    = Arrays.copyOfRange(arr, 1, 4);     // copy index 1 to 3

// Print
System.out.println(arr);                   // [I@hashcode — NOT useful
System.out.println(Arrays.toString(arr));  // [1, 2, 3, 4, 5]
System.out.println(Arrays.deepToString(matrix));  // [[1, 2], [3, 4]] — for nested arrays (2D/3D)
// toString() is shallow, deepToString() handles nested arrays

// Convert to List
List<Integer> fixed   = Arrays.asList(1, 2, 3);             // fixed-size list
List<String>  backed  = Arrays.asList(strArr);              // backed by array — shared memory
List<Integer> mutable = new ArrayList<>(Arrays.asList(arr));// mutable copy

// Stream
Arrays.stream(arr).sum();
int[] filtered = Arrays.stream(arr).filter(n -> n > 3).toArray();  // returns int[] (primitive stream)

// Parallel sort (faster for large arrays, uses ForkJoinPool)
Arrays.parallelSort(arr);

// Set all elements using index-based function
int[] arr2 = new int[5];
Arrays.setAll(arr2, i -> i * 2);   // [0, 2, 4, 6, 8]

// Mismatch (Java 9+) — first index where arrays differ
int idx = Arrays.mismatch(arr1, arr2);   // -1 if identical
```

## `length` vs `length()` vs `size()`

||`length`|`length()`|`size()`|
|---|---|---|---|
|Type|Field|Method|Method|
|Used on|Arrays|String|Collections|
|Example|`arr.length`|`str.length()`|`list.size()`|

```java
int[] arr = {1, 2, 3};
String s = "hello";
List<Integer> list = List.of(1, 2, 3);

arr.length;      // 3 — field (no parentheses)
s.length();      // 5 — method (with parentheses)
list.size();     // 3 — method
```

## Array Copying

```java
int[] src = {1, 2, 3, 4, 5};

// 1. Arrays.copyOf — returns new array
int[] copy = Arrays.copyOf(src, src.length);

// 2. System.arraycopy — low-level, fast
int[] dest = new int[5];
System.arraycopy(src, 0, dest, 0, src.length);
// params: (source, srcPos, dest, destPos, length)

// 3. clone() — returns new array
int[] copy = src.clone();

// All three create SHALLOW copies for object arrays
// Primitive arrays — values are copied (no sharing)
// Object arrays — references are copied (objects shared)
String[] original = {"hello", "world"};
String[] copy = original.clone();
copy[0] = "changed";         // original[0] still "hello" — String is immutable
// But for mutable objects, changes through copy affect original
```

## Arrays and Generics

```java
// Can't create generic arrays
T[] arr = new T[10];           // ❌ COMPILE ERROR — type erasure

// Workaround
@SuppressWarnings("unchecked")
T[] arr = (T[]) new Object[10];

// Arrays are covariant (unlike generics which are invariant)
Object[] arr = new String[5];   // ✅ compiles
arr[0] = 123;                   // ❌ ArrayStoreException at runtime!

// This is why generics use List<T> instead of T[]
```

## Varargs (Variable Arguments)

```java
// Method accepts variable number of arguments — treated as array internally
public static int sum(int... numbers) {
    int total = 0;
    for (int n : numbers) total += n;
    return total;
}

sum(1, 2, 3);           // 6
sum(1, 2, 3, 4, 5);     // 15
sum(new int[]{1, 2});   // 3 — can pass array too
sum();                   // 0 — empty is valid

// Rules
// - Only one varargs per method
// - Must be the last parameter
// - Internally it's an array
void method(String name, int... scores) { }  // ✅
void method(int... a, int... b) { }          // ❌ only one varargs
void method(int... scores, String name) { }  // ❌ must be last
```

## Common Array Patterns

```java
int[] arr = {5, 3, 9, 1, 7};

// Find max/min
int max = Arrays.stream(arr).max().getAsInt();        // 9
int min = Arrays.stream(arr).min().getAsInt();        // 1

// Sum and average
int sum = Arrays.stream(arr).sum();                    // 25
double avg = Arrays.stream(arr).average().getAsDouble(); // 5.0

// Reverse array (no built-in method)
int left = 0, right = arr.length - 1;
while (left < right) {
    int temp = arr[left];
    arr[left] = arr[right];
    arr[right] = temp;
    left++; right--;
}

// Second largest (returns Optional — empty if < 2 distinct elements)
Optional<Integer> secondMax = Arrays.stream(arr)
    .distinct().boxed()
    .sorted(Comparator.reverseOrder())
    .skip(1).findFirst();

// Find duplicates
Set<Integer> seen = new HashSet<>();
List<Integer> dups = Arrays.stream(arr).boxed()
    .filter(n -> !seen.add(n))
    .collect(Collectors.toList());

// Remove duplicates (preserving order)
int[] unique = Arrays.stream(arr).distinct().toArray();

// Check if sorted
boolean sorted = IntStream.range(0, arr.length - 1)
    .allMatch(i -> arr[i] <= arr[i + 1]);

// Two sum (find pair that sums to target)
Set<Integer> set = new HashSet<>();
for (int n : arr) {
    if (set.contains(target - n)) {
        System.out.println(n + " + " + (target - n));
        break;
    }
    set.add(n);
}
```

## Commonly Asked Gotchas

1. **Arrays are objects in Java** — stored in heap. Java is always pass-by-value, but for arrays the value IS the reference. So modifying array contents inside a method affects the original.

```java
void modify(int[] arr) {
    arr[0] = 100;   // modifies original array
}

void reassign(int[] arr) {
    arr = new int[]{9, 9, 9};   // does NOT affect original — local reference changed
}
```

2. **`toString()` on arrays doesn't print contents:**

```java
System.out.println(arr);                  // [I@1a2b3c — useless
System.out.println(Arrays.toString(arr)); // [1, 2, 3] — useful
```

3. **Array covariance can cause runtime errors:**

```java
Object[] arr = new String[3];
arr[0] = 123;   // compiles but throws ArrayStoreException at runtime
```

4. **`Arrays.asList()` on primitive array doesn't work as expected:**

```java
int[] arr = {1, 2, 3};
List<int[]> list = Arrays.asList(arr);   // List of ONE element (the array itself)
// Does NOT auto-box primitives — treats whole array as single element

// Fix: use Integer[] or stream
List<Integer> list = Arrays.stream(arr).boxed().toList();
```

5. **`Arrays.sort()` for primitives uses Dual-Pivot Quicksort (NOT stable). For objects uses TimSort (stable — preserves order of equal elements).**

6. **Shallow copy for object arrays** — `clone()`, `Arrays.copyOf()`, `System.arraycopy()` all copy references, not objects.

7. **Enhanced for-each can't modify array** — the loop variable is a copy.

```java
for (int n : arr) {
    n = 0;    // doesn't change arr — n is a copy
}
```

8. **Array declaration gotcha:**

```java
int[] a, b;    // both are int arrays
int a[], b;    // a is int array, b is just int!
```

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Fixed size|Can't grow/shrink after creation. Use ArrayList for dynamic|
|Default values|`0`, `0.0`, `false`, `null` depending on type|
|`length`|Field, not method. No parentheses: `arr.length`|
|`Arrays.toString()`|Use this to print — direct print shows hashcode|
|`Arrays.sort()`|Primitives = Dual-Pivot Quicksort, Objects = TimSort (stable)|
|`Arrays.copyOf()`|Shallow copy. New array, but object references shared|
|`System.arraycopy()`|Fastest copy method (native)|
|Covariant|`Object[] = new String[]` compiles but can cause `ArrayStoreException`|
|Varargs|`int... nums` — must be last param, only one per method|
|`Arrays.asList(int[])`|Creates `List<int[]>` not `List<Integer>` — use `Integer[]` or stream|
|2D arrays|`matrix.length` = rows, `matrix[0].length` = columns|
|Jagged arrays|Rows can have different lengths|
