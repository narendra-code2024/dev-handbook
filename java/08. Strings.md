## 1. String Basics

Strings are **objects** in Java, not primitives. They are instances of `java.lang.String` class. `String` is a **final class** — cannot be extended.

```java
// Two ways to create strings
String s1 = "hello";              // string literal — goes to String Pool
String s2 = new String("hello");  // new object — goes to Heap (outside pool)
```

---

## 2. String Immutability

Strings are **immutable** — once created, the value cannot be changed. Any modification creates a new String object.

```java
String s = "hello";
s.concat(" world");        // creates new String, original unchanged
System.out.println(s);     // "hello" — not "hello world"

s = s.concat(" world");   // s now points to new object "hello world"
                            // old "hello" is still in pool, just unreferenced
```

**Why immutable?**

- **String Pool safety** — if strings were mutable, changing one would affect all references
- **Thread safety** — immutable objects are inherently thread-safe
- **Security** — class names, URLs, file paths use strings — mutation would be dangerous
- **hashCode caching** — String caches its hashCode after first computation (computed once, reused) — improves performance in hash-based collections like HashMap

---

## 3. String Constant Pool

A special area in the Heap (called the **String Pool / String Table**) where Java stores **string literals** to save memory. If the same literal exists, JVM reuses it.

```java
String a = "hello";           // creates in pool
String b = "hello";           // reuses same pool reference
String c = new String("hello"); // creates new object in heap (NOT in pool)

a == b       // true — same pool reference
a == c       // false — different objects (pool vs heap)
a.equals(c)  // true — same content

// intern() — moves heap string to pool (or returns existing pool reference)
String d = c.intern();
a == d       // true — d now points to pool reference
```

**How many objects?**

```java
String s = new String("hello");
// Answer: 2 objects
// 1. "hello" literal in String Pool (if not already there)
// 2. new String object in Heap
```

## 4. String vs StringBuilder vs StringBuffer

||String|StringBuilder|StringBuffer|
|---|---|---|---|
|Mutable?|No (creates new object)|Yes|Yes|
|Thread-safe?|Yes (immutable)|No|Yes (synchronized)|
|Speed|Slow for concatenation|Fastest|Slower than StringBuilder|
|Storage|String Pool + Heap|Heap|Heap|

```java
// String — every + creates new object (bad in loops)
String s = "";
for (int i = 0; i < 1000; i++) {
    s += i;   // creates 1000 new String objects — terrible performance
}

// StringBuilder — modifies same object (use this)
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append(i);   // same object modified — fast
}
String result = sb.toString();
```

**When to use:**

- `String` — when value doesn't change (constants, keys, labels)
- `StringBuilder` — 99% of the time for string building/manipulation
- `StringBuffer` — only if multiple threads build the same string (almost never)

### StringBuilder Key Methods

```java
StringBuilder sb = new StringBuilder("hello");

sb.append(" world");       // "hello world"
sb.insert(5, ",");         // "hello, world"
sb.replace(0, 5, "Hi");   // "Hi, world"
sb.delete(2, 4);           // "Hi world" (removes index 2 to 3)
sb.reverse();              // "dlrow iH"
sb.charAt(0);              // 'd'
sb.length();               // 8
sb.indexOf("world");       // index or -1
sb.substring(0, 5);        // returns String, doesn't modify sb
sb.toString();             // convert to String
```

## 5. String Comparison

```java
String a = "hello";
String b = "hello";
String c = new String("hello");

// == compares references (same object in memory?)
a == b       // true — same pool reference
a == c       // false — different objects

// equals() compares content (same characters?)
a.equals(c)          // true
a.equalsIgnoreCase("HELLO")  // true

// compareTo() — lexicographic comparison (returns int)
"apple".compareTo("banana")   // negative (apple < banana)
"banana".compareTo("apple")   // positive (banana > apple)
"apple".compareTo("apple")    // 0 (equal)
```

**Rule:** Always use `equals()` for String comparison, never `==`.

## 6. Commonly Used String Methods

```java
String s = "  Hello, World!  ";

// Length and checks
s.length();                  // 19 (includes spaces)
s.isEmpty();                 // false
s.isBlank();                 // false (Java 11+ — true if only whitespace)

// Case
s.toUpperCase();             // "  HELLO, WORLD!  "
s.toLowerCase();             // "  hello, world!  "

// Trim and strip
s.trim();                    // "Hello, World!" (removes leading/trailing spaces)
s.strip();                   // "Hello, World!" (Java 11+ — Unicode-aware trim)
s.stripLeading();            // "Hello, World!  "
s.stripTrailing();           // "  Hello, World!"

// Search
s.contains("World");         // true
s.indexOf("World");          // 9 (first occurrence index)
s.lastIndexOf("l");          // last occurrence
s.startsWith("  He");        // true
s.endsWith("!  ");           // true

// Extract
s.charAt(2);                 // 'H'
s.substring(2, 7);           // "Hello" (from index 2 to 6)
s.toCharArray();             // char[] {'H', 'e', 'l', 'l', 'o', ...}

// Replace
s.replace('l', 'L');         // replaces all 'l' with 'L'
s.replace("World", "Java");  // replaces substring
s.replaceAll("[^a-zA-Z]", ""); // regex — remove non-alphabetic
s.replaceFirst("l", "L");   // replaces first occurrence only

// Split and join
"a,b,c".split(",");          // String[] {"a", "b", "c"}
"a,,b".split(",");           // String[] {"a", "", "b"} — empty string for consecutive delimiters
"a,,b".split(",", -1);       // preserves trailing empty strings
String.join("-", "a", "b", "c");  // "a-b-c"
String.join(", ", list);     // join collection elements

// Check content
s.matches("[A-Za-z]+");      // regex match — true/false

// Repeat (Java 11+)
"ha".repeat(3);              // "hahaha"

// Format
String.format("Name: %s, Age: %d", "Alice", 25);  // "Name: Alice, Age: 25"
// Formatted (Java 15+)
"Name: %s".formatted("Alice");  // "Name: Alice"
```

## 7. String Conversion

```java
// Primitive → String
String.valueOf(123);         // "123"
String.valueOf(true);        // "true"
String.valueOf(3.14);        // "3.14"
Integer.toString(123);       // "123"
"" + 123;                    // "123" (concat with empty string — simple but creates objects)

// String → Primitive
Integer.parseInt("123");     // 123
Double.parseDouble("3.14");  // 3.14
Boolean.parseBoolean("true"); // true
// NumberFormatException if string is not a valid number

// String → char[]
char[] chars = "hello".toCharArray();

// char[] → String
String s = new String(chars);
String s = String.valueOf(chars);

// String → byte[]
byte[] bytes = "hello".getBytes(StandardCharsets.UTF_8);

// byte[] → String
String s = new String(bytes, StandardCharsets.UTF_8);
```

## 8. String Concatenation Internals

```java
// + operator — compiler converts to StringBuilder (since Java 5)
String s = "a" + "b" + "c";
// Compiler optimizes literals: "abc" (compile-time constant)

String x = "a";
String s = x + "b" + "c";
// Compiler creates: new StringBuilder().append(x).append("b").append("c").toString()

// Java 9+ uses invokedynamic (StringConcatFactory) for better performance
// But in loops, still use StringBuilder explicitly
```

## 9. Text Blocks (Java 15+)

Multi-line string literals — no more `\n` and `+` concatenation.

```java
// Before
String json = "{\n" +
              "  \"name\": \"Alice\",\n" +
              "  \"age\": 25\n" +
              "}";

// Text block
String json = """
              {
                "name": "Alice",
                "age": 25
              }
              """;
```

## 10. Common String Patterns (Interview Favorites)

```java
// Reverse a string
String rev = new StringBuilder("hello").reverse().toString();  // "olleh"

// Check palindrome
boolean isPalin = s.equals(new StringBuilder(s).reverse().toString());

// Count character occurrences
long count = "hello".chars().filter(c -> c == 'l').count();  // 2

// Remove duplicates preserving order
String unique = "aabbcc".chars()
    .distinct()
    .collect(StringBuilder::new, StringBuilder::appendCodePoint, StringBuilder::append)
    .toString();  // "abc"

// Check if two strings are anagrams
char[] c1 = s1.toLowerCase().toCharArray();
char[] c2 = s2.toLowerCase().toCharArray();
Arrays.sort(c1);
Arrays.sort(c2);
boolean isAnagram = Arrays.equals(c1, c2);

// First non-repeating character
Character first = str.chars()
    .mapToObj(c -> (char) c)
    .collect(Collectors.groupingBy(c -> c, LinkedHashMap::new, Collectors.counting()))
    .entrySet().stream()
    .filter(e -> e.getValue() == 1)
    .map(Map.Entry::getKey)
    .findFirst().orElse(null);

// Count words
long wordCount = "hello world foo".trim().split("\\s+").length;  // 3
```

## 11. Commonly Asked Gotchas

1. **Strings are immutable** — methods like `concat()`, `replace()`, `toUpperCase()` return a **new** String. Original is unchanged. Forgetting to assign the result is a common bug.
    
2. **`==` vs `equals()` for Strings** — `==` compares references, `equals()` compares content. Always use `equals()`.
    
3. **String Pool only for literals** — `new String("hello")` creates a heap object outside the pool. Use `intern()` to add to pool.
    
4. **`+` in loops is slow** — each `+` creates a new StringBuilder → String. Use `StringBuilder` explicitly in loops.
    
5. **`split()` with special regex characters** — `.`, `|`, `*`, `+`, etc. need escaping.
    

```java
"a.b.c".split(".");     // ❌ empty array — "." is regex wildcard
"a.b.c".split("\\.");   // ✅ ["a", "b", "c"]
```

6. **`substring()` creates a new String** — since Java 7u6, substring creates a new char array (not a view). No memory leak concern anymore.
    
7. **String `+` with null** — concatenating null doesn't throw NPE, it becomes "null".
    

```java
String s = null;
System.out.println("Value: " + s);  // "Value: null"
// But: s.concat("x") → NullPointerException!
```

8. **`trim()` vs `strip()`** — `trim()` removes ASCII whitespace only (≤ U+0020). `strip()` (Java 11+) removes Unicode whitespace too.
    
9. **String `switch` (Java 7+)** — Strings can be used in switch statements. Uses `equals()` internally, not `==`.
    
10. **`charAt()` returns `char`, not `String`** — for single character operations.
    
11. **Integer caching example — concatenation creates new objects:**
    

```java
Integer a = 127, b = 127;
(a + "" == b + "")   // false — each concatenation creates a new String object
```

12. **`getBytes()` without charset uses platform default** — can cause bugs across OS.

```java
"hello".getBytes();                        // ❌ platform-dependent
"hello".getBytes(StandardCharsets.UTF_8);  // ✅ explicit
```

13. **Call `equals()` on constant to avoid NPE:**

```java
"hello".equals(s);   // ✅ safe even if s is null
s.equals("hello");   // ❌ NPE if s is null
```

14. **Avoid excessive `intern()`** — adds pressure on String Pool and can degrade performance with too many interned strings.
    
15. **Compact Strings (Java 9+)** — internally String uses `byte[]` instead of `char[]`. Latin-1 strings use 1 byte per character (half the memory). Transparent to developers — no code changes needed.

## 12. Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Immutable|String value can't change. Methods return new String|
|String Pool|Literals reused. `new String()` creates separate heap object|
|`intern()`|Returns pool reference (or adds to pool if not exists)|
|`==` vs `equals()`|`==` = reference, `equals()` = content. Always use `equals()`|
|StringBuilder|Mutable, fast, not thread-safe. Use in loops|
|StringBuffer|Mutable, thread-safe (synchronized). Rarely needed|
|`trim()` vs `strip()`|trim = ASCII only, strip = Unicode (Java 11+)|
|`split("\\.")`|Escape regex special characters|
|`+` in loops|Avoid — use StringBuilder explicitly|
|Text blocks|`"""multi-line"""` (Java 15+)|
|`isBlank()`|True if empty or whitespace only (Java 11+)|
|Concatenation|Compiler uses StringBuilder (Java 5-8) or invokedynamic (Java 9+)|
