## 1. Core of OOP

Object-Oriented Programming organizes code around **objects** (data + behavior) rather than functions and logic.

|Concept|What it is|
|---|---|
|**Class**|Blueprint/template for creating objects. Defines fields and methods|
|**Object**|Instance of a class. Has state (fields) and behavior (methods)|
|**Attributes**|Fields/variables that hold object's state (name, age, salary)|
|**Methods**|Functions that define object's behavior (getName, calculateSalary)|

```java
// Class = blueprint
class Employee {
    String name;          // attribute
    int salary;           // attribute

    void work() {         // method (behavior)
        System.out.println(name + " is working");
    }
}

// Object = instance
Employee emp = new Employee();   // creating object from blueprint
emp.name = "Alice";              // setting state
emp.work();                      // calling behavior
```

## 2. Four Pillars of OOP

|Pillar|One-liner|Key mechanism|
|---|---|---|
|**Encapsulation**|Hide data, control access|Private fields + public getters/setters|
|**Inheritance**|Reuse parent's code|`extends` keyword|
|**Polymorphism**|Same method, different behavior|Overloading (compile) + Overriding (runtime)|
|**Abstraction**|Hide complexity, show essentials|Abstract classes + Interfaces|

Each pillar has its own detailed notes section (Sections 12–15 in ToC).

### Advantages of OOP

- **Modularity** — each class is a self-contained unit
- **Reusability** — inheritance allows code reuse
- **Flexibility** — polymorphism enables interchangeable components
- **Maintainability** — encapsulation isolates changes
- **Real-world modeling** — objects map to real entities (Employee, Order, Invoice)

## 3. Class and Object

### Class

```java
class Car {
    // Fields (state)
    String brand;
    int speed;

    // Methods (behavior)
    void accelerate() {
        speed += 10;
    }
}
```

- A class is a **template** — no memory allocated until object is created
- Can have: fields, methods, constructors, blocks, inner classes

### Object

```java
Car myCar = new Car();    // object created on Heap
myCar.brand = "Toyota";   // accessing field
myCar.accelerate();       // calling method
```

- Object = instance of a class
- Each object has its **own copy** of instance variables
- Method bytecode is stored once in Metaspace; execution is per-thread via stack frames. Each method call creates a new stack frame → ensures thread safety for local variables.

### `new` Keyword

What happens when you write `new Car()`:

1. **Memory allocated** on Heap for the object
2. **Fields initialized** to default values (0, null, false)
3. **`super()` called** — parent constructor runs first
4. **Instance blocks execute** — in order of appearance
5. **Constructor body runs** — sets actual values
6. **Reference returned** — stored in the variable on Stack

Other ways to create objects (awareness):

- `new` keyword (most common)
- Reflection (`Class.newInstance()`, `Constructor.newInstance()`)
- Cloning (`clone()`)
- Deserialization (`ObjectInputStream.readObject()`)
- Factory methods (`Integer.valueOf()`, `List.of()`)

```java
Car c1 = new Car();   // c1 on Stack → points to Car object on Heap
Car c2 = new Car();   // c2 on Stack → points to different Car object on Heap
Car c3 = c1;          // c3 points to SAME object as c1 (no new object created)
```

## 4. Methods

### Instance Methods vs Static Methods

```java
class Calculator {
    int memory = 0;                    // instance variable

    // Instance method — needs object, can access instance + static
    void addToMemory(int val) {
        memory += val;
    }

    // Static method — no object needed, can only access static directly
    // Resolved at compile-time — can be hidden in child class, but no runtime polymorphism
    static int add(int a, int b) {
        return a + b;
    }
}

// Calling
Calculator calc = new Calculator();
calc.addToMemory(5);           // instance method — needs object
Calculator.add(3, 4);          // static method — called on class
```

||Instance Method|Static Method|
|---|---|---|
|Belongs to|Object|Class|
|Access|Instance + static members|Static members only (no `this`)|
|Call|`obj.method()`|`ClassName.method()`|
|`this` keyword|Available|Not available|
|Use for|Object-specific behavior|Utility/helper functions|

### Method Signature

```java
// Method signature = method name + parameter types
// Return type is NOT part of signature
int add(int a, int b)      // signature: add(int, int)
double add(int a, int b)   // ❌ same signature — can't overload by return type only
```

## 5. Blocks

### Instance Block (Initializer Block)

Runs every time an object is created, **before constructor**. Useful for common initialization shared across multiple constructors.

```java
class Employee {
    String name;

    // Instance block — shared initialization logic
    {
        System.out.println("Instance block");
        name = "Default";
    }

    Employee() { System.out.println("Constructor"); }
    Employee(String name) {
        this.name = name;
        System.out.println("Constructor");
    }
}

new Employee();
// Output:
// Instance block
// Constructor
```

### Static Block

```java
class Config {
    static String dbUrl;

    // Static block — runs once when class is loaded, before any object creation
    // Real-world: DB driver loading, config init, cache setup
    static {
        System.out.println("Static block");
        dbUrl = "jdbc:mysql://localhost:3306/db";
    }
}
```

### Execution Order

```java
class Demo {
    static { System.out.println("1. Static block"); }
    { System.out.println("2. Instance block"); }
    Demo() { System.out.println("3. Constructor"); }
}

Demo d1 = new Demo();
// Output: 1. Static block → 2. Instance block → 3. Constructor

Demo d2 = new Demo();
// Output: 2. Instance block → 3. Constructor
// (Static block runs only ONCE — already loaded)
```

### Methods vs Blocks

||Method|Block|
|---|---|---|
|Name|Has a name|No name|
|Called|Explicitly|Automatically|
|Parameters|Can have|Cannot|
|Return|Can return value|Cannot|
|When|When called|Instance: each object creation. Static: class loading|

## 6. `static` Basics

### Static Variables (Class Variables)

```java
class Employee {
    String name;              // instance variable — one per object
    static int count = 0;     // static variable — shared across ALL objects

    Employee(String name) {
        this.name = name;
        count++;              // shared counter
    }
}

Employee e1 = new Employee("Alice");
Employee e2 = new Employee("Bob");
System.out.println(Employee.count);   // 2 — shared, not per-object
```

### Static Methods

Behavior covered in Section 4. Key gotcha unique to static: **can't be overridden — only hidden** (resolved at compile-time, no runtime polymorphism).

### Static Block

Covered in Section 5. Runs once on class load, before any object creation — used for static variable initialization and one-time setup (driver loading, config init).

### Summary

||Static|Instance|
|---|---|---|
|Memory|Metaspace (metadata), referenced objects in Heap|One per object (Heap)|
|Access|`ClassName.member`|`object.member`|
|Available|When class is loaded|When object is created|
|`this`|Not available|Available|

## 7. Variable Types by Scope and Lifetime

```java
class Employee {
    static int count = 0;       // STATIC — class-level, shared, lives until class unloaded
    String name;                // INSTANCE — object-level, lives until object is GC'd
    
    void process() {
        int temp = 10;          // LOCAL — method-level, lives until method returns
    }
}
```

||Local|Instance|Static|
|---|---|---|---|
|Declared in|Method/block|Class (outside methods)|Class with `static`|
|Default value|None (must initialize)|Yes (0, null, false)|Yes (0, null, false)|
|Scope|Within method/block|Within object|Within class (all objects)|
|Memory|Stack|Heap (inside object)|Metaspace (metadata), referenced objects in Heap|
|Lifetime|Method execution|Object lifetime|Class lifetime|
|Thread safety|Thread-confined|Safe if object not shared, needs sync if shared|Shared across threads — needs synchronization|

> Thread-confined, shared mutable state, and synchronization rules are covered in depth in **Section 24: Multithreading & Concurrency**.

## 8. Constructors

### Basics

A **constructor** is a special block of code that runs **once when an object is created** (via `new`). Its job is to initialize the object's state — set field values, validate inputs, acquire resources.

Key markers:
- **Same name as the class** (case-sensitive)
- **No return type** — not even `void`. If you write `void Employee()`, it becomes a regular method, not a constructor.
- **Called automatically** by `new` — you never call it explicitly like a method
- Can be **overloaded** (multiple constructors with different parameter lists)

```java
class Employee {
    String name;
    int salary;

    Employee(String name, int salary) {
        this.name = name;
        this.salary = salary;
    }
}

Employee emp = new Employee("Alice", 50000);
```

### Default Constructor

```java
class Employee {
    // If you don't define ANY constructor, compiler adds:
    // Employee() { super(); }  ← default constructor (no-arg, calls parent)
}

// But if you define ANY constructor, default is NOT added
class Employee {
    Employee(String name) { }     // custom constructor
}
new Employee();                    // ❌ COMPILE ERROR — no default constructor
```

### Constructor Overloading

```java
class Employee {
    String name;
    int salary;

    Employee() {
        this("Unknown", 0);        // calling another constructor — must be first line
    }

    Employee(String name) {
        this(name, 0);
    }

    Employee(String name, int salary) {
        this.name = name;
        this.salary = salary;
    }
}
```

### Constructor Chaining

```java
// Within same class — this()
Employee() { this("Default"); }
Employee(String name) { this(name, 0); }
Employee(String name, int salary) { this.name = name; this.salary = salary; }

// To parent class — super()
class Manager extends Employee {
    Manager(String name) {
        super(name, 80000);        // calls parent constructor — must be first line
    }
}

// this() and super() can't both be on line 1 — mutually exclusive
// (parent constructor must run exactly once — see Section 13 for the full chain + implicit super())
```

### Constructor vs Instance Methods vs Blocks

||Constructor|Instance Method|Instance Block|
|---|---|---|---|
|Name|Same as class|Any name|No name|
|Return type|None (not even void)|Required|None|
|Called|On `new` (once per object)|Explicitly (multiple times)|Before constructor (each object)|
|`this()` / `super()`|Yes (first line)|No|No|
|Purpose|Initialize object|Define behavior|Common initialization code|

### No Static Constructor

Java doesn't have static constructors. Use **static blocks** for static initialization:

```java
class Config {
    static Map<String, String> settings;

    // Not a constructor — it's a static block
    static {
        settings = new HashMap<>();
        settings.put("env", "prod");
    }
}
```

## 9. `this` Keyword

Refers to the **current object** — the object on which the method/constructor was called.

```java
class Employee {
    String name;

    // Most common use — distinguish field from parameter
    Employee(String name) {
        this.name = name;       // this.name = field, name = parameter
    }

    // Call another constructor (must be first line)
    Employee() {
        this("Default");
    }
}
```

**`this` is NOT available in static context** — static methods belong to the class, not any object.

Advanced uses (chaining, returning current instance, passing self) covered in Keywords Deep Dive (Section 17).

## 10. Coupling and Cohesion

### Coupling — how dependent classes are on each other

```java
// Tight coupling (bad) — directly depends on concrete class
class OrderService {
    MySQLDatabase db = new MySQLDatabase();   // can't swap DB easily
}

// Loose coupling (good) — depends on abstraction
class OrderService {
    Database db;                              // interface — can swap implementation
    OrderService(Database db) { this.db = db; }
}
```

**Low coupling is better** — classes are independent, easy to change/test. Achieved through Dependency Injection (constructor injection) and coding to interfaces.

### Cohesion — how focused a class is on a single responsibility

```java
// Low cohesion (bad) — does too many things
class Employee {
    void calculateSalary() { }
    void sendEmail() { }
    void generateReport() { }
    void connectToDatabase() { }
}

// High cohesion (good) — focused on one thing
class SalaryCalculator { void calculate() { } }
class EmailService { void send() { } }
class ReportGenerator { void generate() { } }
```

**High cohesion is better** — each class does one thing well. Related to Single Responsibility Principle (SOLID).

## 11. Commonly Asked Gotchas

1. **Constructor is NOT a method** — no return type (not even `void`). If you write `void Employee()`, it becomes a regular method, not a constructor.
    
2. **Default constructor disappears** — if you define ANY constructor, compiler doesn't add the default no-arg one. Subclasses calling `super()` will break.
    
3. **`this()` and `super()` must be first line** — and they're mutually exclusive. Can't call both in same constructor.
    
4. **Static method can't access `this`** — and can't access instance members directly. Must create/receive an object.
    
5. **Objects are passed by value of reference** — Java passes a copy of the reference, not the reference itself. You can modify object state, but cannot change the caller's reference.
    

```java
void modify(Employee e) {
    e.name = "Changed";        // ✅ modifies original — same object
}
void reassign(Employee e) {
    e = new Employee("New");   // ❌ doesn't affect original — local copy changed
}
```

6. **Multiple objects can reference same data:**

```java
Employee e1 = new Employee("Alice");
Employee e2 = e1;              // same object, two references
e2.name = "Bob";
System.out.println(e1.name);   // "Bob" — both point to same object
```

7. **`equals()` default = reference comparison** — same as `==`. Must override for content comparison. Always override `hashCode()` when overriding `equals()` — otherwise HashMap/HashSet break. (See Section 19 for full contract details.)

```java
Employee e1 = new Employee("Alice");
Employee e2 = new Employee("Alice");
e1 == e2       // false — different objects
`e1.equals(e2)  // false — unless equals() is overridden
```

8. **Static block runs before main():**

```java
class Test {
    static { System.out.println("Static block"); }
    public static void main(String[] args) {
        System.out.println("Main");
    }
}
// Output: Static block → Main
```

9. **Instance variables get default values, local variables don't:**

```java
class Test {
    int x;              // default 0
    void method() {
        int y;
        System.out.println(y);  // ❌ COMPILE ERROR — not initialized
    }
}
```

10. **Execution order with inheritance:**

```
1. Static variables initialized → 2. Static blocks (once per class, parent first)
3. Instance variables initialized → 4. Instance blocks → 5. Constructor (parent first, each object)
```

11. **`this` in constructor refers to the object being created** — even though the constructor hasn't finished yet. Fields may still have default values if accessed before assignment.
    
12. **Constructor overloading + autoboxing** — can cause ambiguity. Covered in detail in Polymorphism (Section 14).
    
13. **Method overloading resolution priority** — exact match → widening → autoboxing → varargs. Covered in detail in Polymorphism (Section 14).
    
14. **Low cohesion = God Class / Blob Class** — a class that does too many unrelated things. Anti-pattern.

## 12. Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Class|Blueprint. No memory until object created|
|Object|Instance of class. State (fields) + behavior (methods)|
|`new`|Allocate heap → defaults → super() → instance blocks → constructor → return reference|
|Instance vs Static method|Instance needs object + has `this`. Static belongs to class|

|Static block|Runs once when class loaded, before any object|
|Execution order|Static vars/blocks (once) → Instance vars/blocks → Constructor (each object)|

|`this()`|Call another constructor. Must be first line|
|`super()`|Call parent constructor. Must be first line. Mutually exclusive with `this()`|
|Local variables|No default value — must initialize|
|Coupling|Low = good (depend on abstractions)|
|Cohesion|High = good (focused responsibility). Low = God Class|
|Pass-by-value|Java passes copy of reference. Can modify state, can't change caller's reference|
