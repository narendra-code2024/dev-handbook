## What are Inner Classes

A class defined **inside another class**. Used when a class is tightly coupled with its enclosing class and not needed elsewhere.

```java
class Outer {
    class Inner {        // inner class
        void display() { System.out.println("Inner"); }
    }
}
```

**Why use:**

- Logically group classes that belong together
- Access outer class's private members
- Better encapsulation — hide inner class from outside world
- Event handlers, iterators, builder pattern implementations

---

## Member Inner Classes

### Non-Static Member Inner Class

- Associated with an **instance** of the outer class
- Can access **all** outer class members (including private)
- Cannot have `static` members (except `static final` constants) — **Java 16+ removed this restriction** (JEP 395)

```java
class Outer {
    private int x = 10;

    class Inner {
        void show() {
            System.out.println(x);           // ✅ access outer's private field
            System.out.println(Outer.this.x); // ✅ explicit outer reference
        }
    }
}

// Creating inner class object — needs outer instance
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
inner.show();

// Or in one line
Outer.Inner inner = new Outer().new Inner();
```

### Static Member Inner Class (Static Nested Class)

- Associated with the **class**, not an instance
- Can only access outer class's **static** members directly
- Can have its own static and instance members
- Behaves like a top-level class but scoped inside outer class

```java
class Outer {
    private static int staticVar = 10;
    private int instanceVar = 20;

    static class Nested {
        void show() {
            System.out.println(staticVar);      // ✅ static member
            // System.out.println(instanceVar);  // ❌ needs outer instance
        }
    }
}

// Creating — no outer instance needed
Outer.Nested nested = new Outer.Nested();
nested.show();
```

### Non-Static vs Static Inner Class

||Non-Static (Inner)|Static (Nested)|
|---|---|---|
|Needs outer instance|Yes (`outer.new Inner()`)|No (`new Outer.Nested()`)|
|Access to outer|All members (including private)|Static members only|
|Can have static members|No, except `static final` (Java 16+: yes)|Yes|
|`Outer.this`|Available|Not available|
|Common use|Event listeners, iterators|Helper/utility classes, Builder pattern|

## Local Inner Class

Defined **inside a method**. Scope limited to that method.

```java
class Outer {
    void process() {
        final int localVar = 10;    // must be effectively final

        class LocalInner {
            void show() {
                System.out.println(localVar);   // ✅ can access effectively final local vars
            }
        }

        LocalInner li = new LocalInner();
        li.show();
    }
}
```

- Can access outer class members
- Can access local variables only if **effectively final**
- Not visible outside the method
- Rarely used in modern Java — lambdas are preferred

## Anonymous Inner Class

A class **without a name** — defined and instantiated in one expression. Used to provide one-time implementation of an interface or abstract class.

```java
// Anonymous class implementing interface
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Running");
    }
};

// Anonymous class extending abstract class
abstract class Shape {
    abstract double area();
}

Shape circle = new Shape() {
    @Override
    double area() { return Math.PI * 5 * 5; }
};

// Anonymous class extending concrete class (override behavior)
Thread t = new Thread() {
    @Override
    public void run() {
        System.out.println("Custom thread");
    }
};
```

### Anonymous Class vs Lambda

```java
// Anonymous class — verbose, creates new class
Runnable r1 = new Runnable() {
    public void run() { System.out.println("Run"); }
};

// Lambda — concise, preferred for functional interfaces
Runnable r2 = () -> System.out.println("Run");
```

||Anonymous Class|Lambda|
|---|---|---|
|Can implement|Any interface or extend class|Functional interfaces only (1 method)|
|`this`|Refers to anonymous class|Refers to enclosing class|
|State (fields)|Can have instance fields|Cannot|
|Class file|Generates separate `.class`|Uses `invokedynamic`|
|Preferred when|Multiple methods, need state|Single method, functional interface|

## Inner Interfaces

An interface defined inside a class or another interface. Implicitly `static`.

```java
class Outer {
    interface Callback {          // implicitly static
        void onComplete();
    }
}

// Implementing
class MyCallback implements Outer.Callback {
    public void onComplete() { System.out.println("Done"); }
}

// Interface inside interface
interface Outer {
    interface Inner {             // implicitly public static
        void process();
    }
}
```

**Real-world example:** `Map.Entry<K,V>` — an interface nested inside `Map` interface.

```java
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    entry.getKey();
    entry.getValue();
}
```

## Inner Abstract Classes

An abstract class defined inside another class.

```java
class Outer {
    abstract class InnerAbstract {
        abstract void process();
        void log() { System.out.println("Logging"); }
    }

    class ConcreteInner extends InnerAbstract {
        void process() { System.out.println("Processing"); }
    }
}

// Static inner abstract class
class Outer {
    static abstract class Helper {
        abstract void help();
    }
}

class MyHelper extends Outer.Helper {
    void help() { System.out.println("Helping"); }
}
```

## When to Use Which Inner Class

|Use case|Type|
|---|---|
|Tightly coupled helper|Non-static inner class|
|Independent utility/builder|Static nested class (e.g., `User.Builder` — avoids holding outer reference)|
|One-time interface implementation|Anonymous class (or lambda)|
|Method-scoped logic|Local inner class (rare — prefer lambda)|

## Commonly Asked Gotchas

1. **Inner class can access outer's `private` members** — this is the main reason inner classes exist. Outer class can also access inner's private members. Internally, the compiler generates **synthetic accessor methods** to make this work.
    
2. **`Outer.this`** — refers to the enclosing class instance from within a non-static inner class. Needed when inner class has a field with same name.
    

```java
class Outer {
    int x = 10;
    class Inner {
        int x = 20;
        void show() {
            System.out.println(x);            // 20 (inner's x)
            System.out.println(Outer.this.x); // 10 (outer's x)
        }
    }
}
```

3. **Anonymous class can't have explicit constructors** — it has no name, so no constructor. Use instance initializer block `{ }` instead. Anonymous class can extend a class OR implement an interface, **but not both**.

```java
Runnable r = new Runnable() {
    { System.out.println("Initializer — acts as constructor"); }
    public void run() { }
};
```

4. **Static nested class is NOT technically an "inner class"** — Java spec distinguishes: inner class = non-static nested class. Static nested class is just a top-level class scoped inside another.
    
5. **Local and anonymous classes can only access effectively final variables:**
    

```java
void method() {
    int x = 10;
    // x = 20;    // if uncommented → anonymous class below fails
    Runnable r = () -> System.out.println(x);   // x must be effectively final
}
```

6. **Each anonymous class generates a separate `.class` file** — `Outer$1.class`, `Outer$2.class`, etc. Lambdas don't generate these (use `invokedynamic`).

7. **Non-static inner class keeps reference to outer** — this can prevent outer class from being garbage collected (memory leak potential in long-lived inner objects).

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Non-static inner|Needs outer instance. Accesses all outer members. `outer.new Inner()`|
|Static nested|No outer instance needed. Accesses static only. `new Outer.Nested()`|
|Local inner|Inside a method. Accesses effectively final locals. Rare|
|Anonymous|No name. One-time use. Can implement interface or extend class|
|Anonymous vs Lambda|Lambda only for functional interfaces. Anonymous for multiple methods/state|
|Inner interface|Implicitly static. Example: `Map.Entry`|
|`Outer.this`|Access enclosing instance from inner class|
|Effectively final|Local vars used in inner/anonymous/lambda can't be reassigned|
|Memory leak|Non-static inner holds outer reference — can prevent GC|
|Generated files|Anonymous → `Outer$1.class`. Lambda → `invokedynamic` (no class file)|
