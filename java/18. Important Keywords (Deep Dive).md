## `final` Keyword

### final Variable — can't reassign

```java
final int x = 10;
x = 20;                // ❌ COMPILE ERROR — can't reassign

final Employee emp = new Employee("Alice");
emp.setName("Bob");    // ✅ can modify object's state
emp = new Employee("Charlie");  // ❌ can't reassign reference
```

**`final` prevents reassignment of the reference, NOT modification of the object.**

### final Method — can't override

```java
class Parent {
    final void process() { System.out.println("Parent"); }
}

class Child extends Parent {
    void process() { }    // ❌ COMPILE ERROR — can't override final method
}
```

**Why:** Guarantee behavior — child can't change critical logic. Example: `Object.getClass()` is final.

### final Class — can't extend

```java
final class Utility {
    static void helper() { }
}

class MyUtil extends Utility { }   // ❌ COMPILE ERROR — can't extend final class
```

**Examples:** `String`, `Integer`, all wrapper classes are final — ensures immutability can't be broken by subclassing.

### final Parameter

```java
void process(final String name) {
    name = "other";     // ❌ can't reassign parameter
}
```

### final vs Immutability

||`final`|Immutable|
|---|---|---|
|What|Can't reassign reference|Can't change object state|
|Scope|Single variable/field|Entire object|
|Guarantees|Reference won't change|Content won't change|
|Example|`final List<String> list` — can still `add()`|`List.of("a","b")` — can't `add()`|

```java
final List<String> list = new ArrayList<>();
list.add("hello");         // ✅ final doesn't prevent modification
// list = new ArrayList<>();  // ❌ can't reassign reference

List<String> immutable = List.of("a", "b");
immutable.add("c");        // ❌ UnsupportedOperationException — truly immutable
```

**`final` is necessary but not sufficient for immutability.** Immutable class needs: final class + private final fields + no setters + defensive copies.

### Blank Final Variable

```java
class Employee {
    final String name;       // blank final — not initialized at declaration

    Employee(String name) {
        this.name = name;    // ✅ must initialize in constructor (exactly once)
    }
}
```

### static final — Constants

```java
class Config {
    static final String APP_NAME = "MyApp";     // constant — convention: UPPER_SNAKE_CASE
    static final int MAX_RETRIES = 3;
}

// Inlined by compiler ONLY for compile-time constants (primitive/String literals)
static final int A = 10;                  // inlined — changing requires recompilation of all users
static final int B = (int)(Math.random()); // NOT inlined — computed at runtime
```

## `this` Keyword (Advanced Usage)

Basics covered in [12. Classes, Objects & OOP Foundations](<12. Classes, Objects & OOP Foundations.md>). Here are advanced patterns:

### Method Chaining (Fluent API)

```java
class QueryBuilder {
    private String table;
    private String condition;
    private int limit;

    QueryBuilder from(String table) {
        this.table = table;
        return this;              // return current object for chaining
    }

    QueryBuilder where(String condition) {
        this.condition = condition;
        return this;
    }

    QueryBuilder limit(int limit) {
        this.limit = limit;
        return this;
    }

    String build() {
        return "SELECT * FROM " + table + " WHERE " + condition + " LIMIT " + limit;
    }
}

// Usage — fluent chaining
String query = new QueryBuilder()
    .from("employees")
    .where("salary > 50000")
    .limit(10)
    .build();
```

**Real-world:** `StringBuilder.append().append()`, `Stream.filter().map().collect()`, Lombok `@Builder`.

### Passing Current Object

```java
class Employee {
    void register() {
        EventBus.register(this);      // pass self as listener
        Repository.save(this);        // pass self to be saved
    }
}
```

### `this` in Inner Class

```java
class Outer {
    int x = 10;
    class Inner {
        int x = 20;
        void show() {
            int x = 30;
            System.out.println(x);            // 30 — local
            System.out.println(this.x);       // 20 — inner's field
            System.out.println(Outer.this.x); // 10 — outer's field
        }
    }
}
```

## `super` Keyword (Advanced Usage)

Basics covered in [14. Inheritance & Relationships](<14. Inheritance & Relationships.md>). Here are advanced patterns:

### Constructor Chaining Up the Hierarchy

```java
class Animal {
    String name;
    Animal(String name) { this.name = name; }
}

class Dog extends Animal {
    String breed;
    Dog(String name, String breed) {
        super(name);               // Animal(String) called
        this.breed = breed;
    }
}

class Puppy extends Dog {
    int age;
    Puppy(String name, String breed, int age) {
        super(name, breed);        // Dog(String, String) called → which calls Animal(String)
        this.age = age;
    }
}

// Chain: Puppy → Dog → Animal → Object
```

### Accessing Hidden Parent Method

```java
class Parent {
    void display() { System.out.println("Parent display"); }
}

class Child extends Parent {
    @Override
    void display() {
        super.display();           // call parent's version first
        System.out.println("Child display");   // then add child-specific logic
    }
}

// Common pattern — extend behavior rather than replace it
```

### Accessing Hidden Parent Field

```java
class Parent { int x = 10; }
class Child extends Parent {
    int x = 20;
    void show() {
        System.out.println(x);         // 20 — child's x
        System.out.println(super.x);   // 10 — parent's x
    }
}
```

### `super` Cannot Be Used

- In `static` context (no instance to reference)
- In `Object` class (no parent)
- To call grandparent (`super.super.method()` is NOT valid)

```java
class A { void show() { } }
class B extends A { void show() { } }
class C extends B {
    void show() {
        super.show();          // ✅ calls B's show()
        // super.super.show(); // ❌ can't skip to A — not allowed
    }
}
```

## `static` Keyword (Deep Dive)

Basics covered in [12. Classes, Objects & OOP Foundations](<12. Classes, Objects & OOP Foundations.md>). Here are advanced topics:

### Static Nested Class

```java
class Company {
    private static String name = "Acme";

    static class Department {
        void show() {
            System.out.println(name);    // ✅ access outer's static members
        }
    }
}

// No outer instance needed
Company.Department dept = new Company.Department();
```

**Real-world:** `Map.Entry`, `Builder` pattern classes are often static nested.

### Static Import

Import static members to use them without class name prefix.

```java
// Without static import
System.out.println(Math.PI);
Collections.sort(list);

// With static import
import static java.lang.Math.PI;
import static java.util.Collections.sort;

System.out.println(PI);     // cleaner
sort(list);                   // cleaner
```

**Use sparingly** — overuse reduces readability (hard to tell where `PI` or `sort` comes from).

### Static Context Rules

```java
class Test {
    int instanceVar = 10;
    static int staticVar = 20;

    static void staticMethod() {
        // System.out.println(instanceVar);   // ❌ can't access instance from static
        // instanceMethod();                   // ❌ can't call instance from static
        System.out.println(staticVar);         // ✅ static → static OK
        staticHelper();                        // ✅ static → static OK

        // Workaround — create an instance
        Test t = new Test();
        System.out.println(t.instanceVar);     // ✅ through object reference
    }

    void instanceMethod() {
        System.out.println(instanceVar);       // ✅ instance → instance OK
        System.out.println(staticVar);         // ✅ instance → static OK
    }

    static void staticHelper() { }
}
```

**Rule:** Static can't access instance (no `this`). Instance can access both static and instance.

### Static Initialization Order

```java
class Config {
    static int a = initA();        // 1. static variable initializers
    static int b;

    static {                       // 2. static blocks
        b = 20;
        System.out.println("Static block: a=" + a + ", b=" + b);
    }

    static int initA() {
        System.out.println("initA called");
        return 10;
    }
}

// Output:
// initA called
// Static block: a=10, b=20
```

Static variables and static blocks execute in **order of appearance** in the source code, top to bottom.

### When to Use Static

|Use case|Example|
|---|---|
|Utility methods|`Math.max()`, `Collections.sort()`|
|Constants|`static final int MAX = 100`|
|Factory methods|`Integer.valueOf()`, `List.of()`|
|Counters/shared state|`static int count`|
|Nested helper classes|`Map.Entry`, Builder classes|
|One-time initialization|Static blocks for config/driver loading|

## Commonly Asked Gotchas

1. **`final` doesn't mean immutable:**

```java
final List<String> list = new ArrayList<>();
list.add("hello");     // ✅ content can change
```

2. **`static final` String/primitives are inlined by compiler:**

```java
class A { static final String X = "hello"; }
class B { String s = A.X; }
// Compiler replaces A.X with "hello" in B → changing A.X requires recompiling B
```

3. **`super()` and `this()` are mutually exclusive** — both must be first line of constructor, so you can't have both.

4. **`super.super` is NOT valid** — can't skip a level in inheritance chain.

5. **Static methods can be called on `null` reference:**

```java
Employee e = null;
e.staticMethod();     // ✅ no NPE — static dispatch ignores the object
Employee.staticMethod(); // same thing — this is what actually happens
```

6. **Method chaining returns `this`** — but be careful with inheritance:

```java
class Parent {
    Parent setName(String n) { return this; }
}
class Child extends Parent {
    // setName() returns Parent, not Child → chaining breaks with child-specific methods
    // Fix: use generics or override with covariant return type
}
```

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|`final` variable|Can't reassign reference. Object content CAN change|
|`final` method|Can't override in child class|
|`final` class|Can't extend. String, Integer are final|
|`final` ≠ immutable|final = reference fixed. Immutable = content fixed|
|Blank final|Must initialize in constructor (exactly once)|
|`static final`|Constant. UPPER_SNAKE_CASE. Inlined by compiler|
|`this` chaining|Return `this` for fluent API (Builder, StringBuilder)|
|`this` in inner|`this` = inner. `Outer.this` = outer|
|`super()`|Must be first line. Implicit if not written. No `super.super`|
|`super` access|Call parent method/field when child hides/overrides it|
|Static import|`import static` — use sparingly for readability|
|Static context|Can't access instance members. No `this`. No `super`|
|Static on null|`null.staticMethod()` works — no NPE|
|Static init order|Variables and blocks in source code order, top to bottom|
