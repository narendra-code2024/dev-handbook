## What are Annotations

**Metadata about code** — doesn't change behavior directly, but can be read by the compiler, tools, or frameworks (sometimes at runtime via reflection).

```java
@Override
public String toString() { return "..."; }
```

**Common real-world uses:**

- `@Override`, `@Deprecated` — compile-time checks
- `@Test` (JUnit) — tells test runner to execute the method
- `@Entity`, `@Table` (JPA) — mark classes/fields for persistence
- `@Autowired`, `@Component` (Spring) — dependency injection + bean discovery

Annotations start with `@`, can be applied to classes, methods, fields, parameters, local variables, and packages.

## Built-in Annotations

### `@Override`

Indicates the method overrides a parent's method. Compiler verifies it's actually overriding — catches typos.

```java
class Parent { void show() { } }

class Child extends Parent {
    @Override
    void show() { }     // ✅ compiler confirms override

    @Override
    void shwo() { }     // ❌ COMPILE ERROR — typo, no such method in Parent
}
```

Optional but strongly recommended — covered in [15. Polymorphism](<15. Polymorphism.md#`@Override` Annotation>).

### `@Deprecated`

Marks a class/method/field as outdated. Usage generates a compile-time warning.

```java
@Deprecated
void oldMethod() { }

// Java 9+ — with metadata
@Deprecated(since = "1.5", forRemoval = true)
void willBeRemoved() { }
```

- `since` — version when deprecation started
- `forRemoval = true` — upgrades to a stronger warning; signals eventual deletion

### `@SuppressWarnings`

Tells the compiler to silence specific warnings for the annotated element.

```java
@SuppressWarnings("unchecked")
List<String> list = (List<String>) obj;        // unchecked cast — warning suppressed

@SuppressWarnings({"unchecked", "deprecation"})
void legacyCode() { }
```

Common values: `unchecked`, `deprecation`, `rawtypes`, `unused`, `all`.

### `@FunctionalInterface`

Enforces that the interface has **exactly one abstract method** — enabling lambda usage. Compiler errors if the rule is broken.

```java
@FunctionalInterface
interface Calculator {
    int compute(int a, int b);    // ✅ single abstract method
}

@FunctionalInterface
interface Broken {
    void a();
    void b();    // ❌ COMPILE ERROR — more than one abstract method
}
```

`default` and `static` methods don't count toward the "one" rule. Full coverage: [24. Functional Programming](<24. Functional Programming (Java 8+).md>).

## Meta-Annotations

Annotations that apply to **other annotations** — used when you define your own.

### `@Target`

Restricts where the annotation can be applied.

```java
@Target(ElementType.METHOD)
@interface OnlyMethod { }
```

Common `ElementType` values: `TYPE` (class/interface/enum), `METHOD`, `FIELD`, `PARAMETER`, `CONSTRUCTOR`, `LOCAL_VARIABLE`, `ANNOTATION_TYPE`, `PACKAGE`, `TYPE_PARAMETER`, `TYPE_USE`.

Multiple targets: `@Target({ElementType.METHOD, ElementType.FIELD})`.

### `@Retention`

How long the annotation is kept.

|Policy|When kept|Example|
|---|---|---|
|`SOURCE`|Compiler discards after compile|`@Override`, `@SuppressWarnings`|
|`CLASS`|In `.class` file, NOT at runtime|**Default** if not specified|
|`RUNTIME`|Available at runtime via reflection|`@Deprecated`, Spring/JPA annotations|

```java
@Retention(RetentionPolicy.RUNTIME)
@interface ReadAtRuntime { }
```

### `@Inherited`

Allows subclasses to inherit the annotation from the parent class.

```java
@Inherited
@Retention(RetentionPolicy.RUNTIME)
@interface Auditable { }

@Auditable
class Parent { }

class Child extends Parent { }    // automatically inherits @Auditable
```

**Only works for class-level annotations** — not methods, fields, or interface-inherited annotations.

### `@Documented`

Includes the annotation in generated Javadoc.

```java
@Documented
@interface ApiVersion { String value(); }
```

### `@Repeatable` (Java 8+)

Allows the same annotation to be applied multiple times to the same element. Requires a **container annotation** to hold the repeats.

```java
@Repeatable(Schedules.class)
@interface Schedule {
    String day();
    String time();
}

@interface Schedules {
    Schedule[] value();    // container — array of Schedule
}

// Usage — annotation applied twice
@Schedule(day = "Monday", time = "09:00")
@Schedule(day = "Friday", time = "17:00")
void runJob() { }
```

Internally the compiler wraps the repeats into the container (`@Schedules`). You can still read them as a single array via reflection — `method.getAnnotationsByType(Schedule.class)`.

## Custom Annotations

Define your own with `@interface`.

```java
import java.lang.annotation.*;

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@interface MaxAttempts {
    int value() default 3;
    String reason() default "";
}

// Usage
class Service {
    @MaxAttempts(value = 5, reason = "flaky network")
    void sendRequest() { }

    @MaxAttempts    // uses default — value=3, reason=""
    void backup() { }
}
```

### Element Rules

- Only **method-like** elements — no fields (except `static final` constants)
- Return types restricted to: primitive, `String`, `Class`, `enum`, annotation, or array of these
- Can have `default` values
- No parameters, no `throws`
- If the only element is named `value`, you can omit the name: `@MaxAttempts(5)`

### Reading Annotations at Runtime

```java
Method m = Service.class.getMethod("sendRequest");
MaxAttempts ann = m.getAnnotation(MaxAttempts.class);
if (ann != null) {
    int retries = ann.value();      // 5
    String reason = ann.reason();   // "flaky network"
}
```

**Requires `@Retention(RUNTIME)`** — otherwise the annotation isn't present at runtime and `getAnnotation()` returns `null`.

## Commonly Asked Gotchas

1. **Default retention is `CLASS`, not `RUNTIME`** — custom annotations you want to read via reflection MUST declare `@Retention(RUNTIME)`. This is the #1 cause of silently-ignored custom annotations.

2. **Annotations do nothing by themselves** — they're just metadata. A framework, tool, or your own reflection code must read them and act on them.

3. **`value()` is special** — if the annotation has only one element named `value`, you can omit the name at the use site: `@MaxAttempts(5)` is the same as `@MaxAttempts(value = 5)`.

4. **`@Inherited` only works on class-level annotations** — not methods, fields, or parameters. Interfaces don't propagate annotations to implementing classes either.

5. **Annotation element return types are restricted** — can't return `List`, `Map`, or arbitrary objects. Use arrays (`String[]`) for multi-value.

6. **`@Deprecated(forRemoval = true)` is a stronger signal** — plain `@Deprecated` may linger; `forRemoval` warns that deletion is planned.

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Annotation|Metadata about code. Doesn't change behavior directly|
|`@Override`|Compiler verifies override. Catches method-name typos|
|`@Deprecated`|Marks as outdated. Java 9+: `since`, `forRemoval`|
|`@SuppressWarnings`|Silence specific warnings: `unchecked`, `deprecation`, etc.|
|`@FunctionalInterface`|Enforces single abstract method → enables lambdas|
|`@Target`|Where annotation can be applied (METHOD, FIELD, TYPE, etc.)|
|`@Retention`|How long kept: `SOURCE`, `CLASS` (default), `RUNTIME`|
|`@Inherited`|Subclasses inherit class-level annotation|
|`@Documented`|Include annotation in Javadoc|
|`@Repeatable` (Java 8+)|Apply same annotation multiple times. Needs container annotation|
|Custom annotation|`@interface`. Elements look like methods. `value()` is special|
|Read at runtime|Requires `@Retention(RUNTIME)` + reflection (`getAnnotation()`)|
|Element return types|Primitive, `String`, `Class`, enum, annotation, or array of these|
