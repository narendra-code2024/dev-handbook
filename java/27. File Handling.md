## What & Why

Java I/O is about **reading from and writing to** files, console, network, etc. Java provides two main packages:

- **`java.io`** — original I/O (Java 1.0+), stream-based, blocking
- **`java.nio`** (New I/O) — modern I/O (Java 1.4+, enhanced in 7+), buffer-based, non-blocking support

---

## Streams (I/O Streams, NOT Java 8 Streams)

I/O streams are pipelines for reading/writing data. Completely different from Java 8 Streams API.

### Two categories

||Byte Streams|Character Streams|
|---|---|---|
|Unit|1 byte (8 bits)|Characters (encoding-dependent, e.g., UTF-8)|
|Base classes|`InputStream` / `OutputStream`|`Reader` / `Writer`|
|Use for|Binary data (images, PDFs, audio)|Text data (files, logs, CSV)|
|Examples|`FileInputStream`, `BufferedInputStream`|`FileReader`, `BufferedReader`|

**Rule of thumb:** Text files → Character streams. Everything else → Byte streams.

## Reading & Writing Files

### Character Streams (Text files)

```java
// Reading — FileReader + BufferedReader
try (BufferedReader br = new BufferedReader(new FileReader("input.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}

// Writing — FileWriter + BufferedWriter
try (BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"))) {
    bw.write("Hello World");
    bw.newLine();                // platform-independent newline
    bw.write("Second line");
}

// Append mode (don't overwrite existing content)
try (FileWriter fw = new FileWriter("output.txt", true)) {  // true = append
    fw.write("Appended line\n");
}
```

### Byte Streams (Binary files)

```java
// Reading
try (FileInputStream fis = new FileInputStream("image.png")) {
    byte[] buffer = new byte[1024];
    int bytesRead;
    while ((bytesRead = fis.read(buffer)) != -1) {
        // process buffer[0..bytesRead-1]
    }
}

// Writing
try (FileOutputStream fos = new FileOutputStream("copy.png")) {
    fos.write(byteArray);
}

// Copy file using byte streams
try (FileInputStream fis = new FileInputStream("source.png");
     FileOutputStream fos = new FileOutputStream("dest.png")) {
    byte[] buffer = new byte[1024];
    int bytesRead;
    while ((bytesRead = fis.read(buffer)) != -1) {
        fos.write(buffer, 0, bytesRead);
    }
}
```

### PrintWriter (Convenient text writing)

```java
try (PrintWriter pw = new PrintWriter(new FileWriter("output.txt"))) {
    pw.println("Line 1");           // auto newline
    pw.printf("Name: %s, Age: %d%n", "Alice", 25);  // formatted
    pw.print("No newline");
}
```

## Buffered Streams (Performance)

Buffering reduces disk access by reading/writing in chunks instead of byte-by-byte.

```java
// Without buffer — slow (each read/write = disk access)
FileReader fr = new FileReader("file.txt");

// With buffer — fast (reads chunk into memory, then serves from buffer)
BufferedReader br = new BufferedReader(new FileReader("file.txt"));

// Same for output
BufferedWriter bw = new BufferedWriter(new FileWriter("file.txt"));
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("file.bin"));
BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("file.bin"));
```

**Prefer buffered streams for large or frequent I/O operations** — significant performance difference on large files.

## java.nio.file (Modern File API — Java 7+)

The `Files` and `Path` classes are the modern way to handle files. Cleaner, more powerful, and preferred over `java.io.File`.

### Path

```java
// Creating paths
Path path = Path.of("data/file.txt");           // Java 11+
Path path = Paths.get("data/file.txt");          // Java 7+
Path path = Paths.get("data", "subdir", "file.txt");  // joins with separator

// Path operations
path.getFileName();       // file.txt
path.getParent();         // data
path.toAbsolutePath();    // /home/user/data/file.txt
path.resolve("sub");      // data/file.txt/sub (append)
path.getNameCount();      // number of path elements
```

### Files utility class

```java
// Read entire file
String content = Files.readString(Path.of("file.txt"));           // Java 11+
List<String> lines = Files.readAllLines(Path.of("file.txt"));     // Java 7+
byte[] bytes = Files.readAllBytes(Path.of("file.bin"));           // Java 7+

// Write
Files.writeString(Path.of("file.txt"), "Hello World");            // Java 11+
Files.write(Path.of("file.txt"), List.of("line1", "line2"));      // Java 7+
Files.write(Path.of("file.txt"), bytes);                          // Java 7+

// Append
Files.writeString(Path.of("file.txt"), "appended", StandardOpenOption.APPEND);
Files.write(Path.of("file.txt"), List.of("new line"), StandardOpenOption.APPEND);

// Copy, Move, Delete
Files.copy(source, target, StandardCopyOption.REPLACE_EXISTING);
Files.move(source, target, StandardCopyOption.REPLACE_EXISTING);
Files.delete(path);              // throws if not found
Files.deleteIfExists(path);      // no exception if not found

// Check
Files.exists(path);
Files.isDirectory(path);
Files.isRegularFile(path);
Files.size(path);                // file size in bytes
Files.getLastModifiedTime(path);

// Create
Files.createFile(path);
Files.createDirectory(path);
Files.createDirectories(path);   // creates parent dirs too (like mkdir -p)
```

### Stream-based file reading (Java 8+)

```java
// Lines as a stream — lazy, memory efficient for large files
try (Stream<String> lines = Files.lines(Path.of("large.txt"))) {
    lines.filter(line -> line.contains("ERROR"))
         .forEach(System.out::println);
}

// Walk directory tree
// ⚠️ Can be expensive on large directory trees — always filter early
try (Stream<Path> paths = Files.walk(Path.of("src"))) {
    paths.filter(Files::isRegularFile)
         .filter(p -> p.toString().endsWith(".java"))
         .forEach(System.out::println);
}

// Limit walk depth
try (Stream<Path> paths = Files.walk(Path.of("src"), 2)) {  // max 2 levels deep
    paths.forEach(System.out::println);
}

// List directory contents (non-recursive)
try (Stream<Path> entries = Files.list(Path.of("data"))) {
    entries.forEach(System.out::println);
}

// Watch directory for changes (NIO)
WatchService watchService = FileSystems.getDefault().newWatchService();
Path dir = Path.of("data");
dir.register(watchService,
    StandardWatchEventKinds.ENTRY_CREATE,
    StandardWatchEventKinds.ENTRY_DELETE,
    StandardWatchEventKinds.ENTRY_MODIFY);
```

### Modern Buffered Reading/Writing (Preferred)

Cleanest way — handles buffering + encoding in one call:

```java
// Reading — auto buffered + explicit charset
try (BufferedReader br = Files.newBufferedReader(Path.of("file.txt"), StandardCharsets.UTF_8)) {
    String line = br.readLine();
}

// Writing — auto buffered + explicit charset
try (BufferedWriter bw = Files.newBufferedWriter(Path.of("file.txt"), StandardCharsets.UTF_8)) {
    bw.write("Hello");
}
```

## File class (Legacy — java.io)

Still widely used but `Path` + `Files` is preferred for new code.

```java
File file = new File("data/file.txt");

file.exists();
file.isFile();
file.isDirectory();
file.getName();            // file.txt
file.getAbsolutePath();
file.length();             // size in bytes
file.delete();
file.mkdir();              // single directory
file.mkdirs();             // with parents
file.listFiles();          // returns File[] of directory contents
file.createNewFile();      // creates if not exists, returns boolean

// Convert between File and Path
Path path = file.toPath();
File file = path.toFile();
```

## try-with-resources for I/O

All I/O streams implement `AutoCloseable` — always use try-with-resources for automatic cleanup.

```java
// ✅ Clean — auto-closes
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line = br.readLine();
}

// ❌ Manual close — error-prone, verbose
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("file.txt"));
    String line = br.readLine();
} finally {
    if (br != null) br.close();
}
```

## InputStreamReader / OutputStreamWriter (Bridge)

Bridges between byte streams and character streams. Used when you have a byte stream but need to read/write characters (e.g., reading from network socket, System.in).

```java
// Reading from System.in (byte stream) as characters
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
String line = br.readLine();

// Specifying encoding
BufferedReader br = new BufferedReader(
    new InputStreamReader(new FileInputStream("file.txt"), StandardCharsets.UTF_8)
);

// Writing with specific encoding
BufferedWriter bw = new BufferedWriter(
    new OutputStreamWriter(new FileOutputStream("file.txt"), StandardCharsets.UTF_8)
);
```

## Common I/O Patterns

### Read file into String

```java
// Modern (Java 11+)
String content = Files.readString(Path.of("file.txt"));

// Java 7+
String content = new String(Files.readAllBytes(Path.of("file.txt")));

// Using BufferedReader
StringBuilder sb = new StringBuilder();
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        sb.append(line).append("\n");
    }
}
```

### Read file line by line (large files)

```java
// Best for large files — lazy, doesn't load entire file into memory
try (Stream<String> lines = Files.lines(Path.of("huge.txt"))) {
    lines.forEach(System.out::println);
}
```

### Write list of strings to file

```java
List<String> lines = List.of("line1", "line2", "line3");
Files.write(Path.of("output.txt"), lines);
```

### Copy file

```java
// Modern
Files.copy(Path.of("src.txt"), Path.of("dest.txt"), StandardCopyOption.REPLACE_EXISTING);

// InputStream → file (common in Spring Boot file uploads)
// ⚠️ Does NOT close the inputStream — use try-with-resources for the stream
Files.copy(inputStream, Path.of("uploaded.txt"), StandardCopyOption.REPLACE_EXISTING);

// File → OutputStream (common in Spring Boot file downloads)
Files.copy(Path.of("file.txt"), outputStream);

// ⚠️ Files.copy() copies files only — does NOT copy directories recursively
// For directories, walk and copy each file manually

// With streams (for custom processing during copy)
try (InputStream in = new FileInputStream("src.bin");
     OutputStream out = new FileOutputStream("dest.bin")) {
    in.transferTo(out);    // Java 9+ — efficient native transfer with optimized internal buffering
}
```

## java.io.File vs java.nio.file (Path + Files)

||java.io.File|java.nio.file (Path + Files)|
|---|---|---|
|Introduced|Java 1.0|Java 7|
|Error handling|Returns boolean (silent failure)|Throws IOException (clear errors)|
|Symbolic links|Poor support|Full support|
|File attributes|Limited|Rich (permissions, owner, timestamps)|
|Directory walking|Recursive manual code|`Files.walk()`, `Files.list()`|
|Atomic operations|No|Yes (`Files.move()` can be atomic)|
|Watch service|No|Yes (`WatchService` for file changes)|

**Prefer `Path` + `Files` for all new code.** Use `File` only when working with legacy APIs that require it.

## Commonly Asked Gotchas

1. **`FileReader`/`FileWriter` use platform default encoding** — can cause issues across OS. Use `InputStreamReader`/`OutputStreamWriter` with explicit charset for reliability.

```java
// ❌ Platform-dependent encoding
new FileReader("file.txt");

// ✅ Explicit encoding
new InputStreamReader(new FileInputStream("file.txt"), StandardCharsets.UTF_8);
```

2. **`readLine()` returns `null` at end of file** — not empty string. Always check for null.

3. **`FileWriter` / `Files.write()` overwrite by default** — creates file if not exists, overwrites if exists. For append: `new FileWriter("f.txt", true)` or `StandardOpenOption.APPEND`.

4. **`Files.readAllLines()` loads entire file into memory** — don't use for huge files. Use `Files.lines()` (lazy stream) instead.

5. **Buffered streams flush on close** — but if you need to ensure data is written before close, call `flush()` explicitly.

6. **`File.delete()` returns boolean silently** — `Files.delete()` throws IOException with reason. Prefer `Files.delete()`.

7. **Path separator differs across OS** — use `Path.of()` or `File.separator` instead of hardcoding `/` or `\\`.

8. **`Files.lines()` must be closed** — it holds an open file handle. Always use try-with-resources.

9. **`Files.write()` with APPEND on new file** — `APPEND` alone may fail if file doesn't exist. Use both `CREATE` and `APPEND`:

```java
Files.write(path, data, StandardOpenOption.CREATE, StandardOpenOption.APPEND);
```

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|Byte streams|InputStream/OutputStream — binary data (images, PDFs)|
|Character streams|Reader/Writer — text data|
|Buffered|Always wrap in Buffered — reduces disk access|
|Files (nio)|Modern, preferred. `Files.readString()`, `Files.write()`, `Files.copy()`|
|Path (nio)|`Path.of()` or `Paths.get()` — replaces `new File()`|
|File (io)|Legacy but still used. Returns boolean, no exceptions|
|try-with-resources|Always use for I/O — auto-closes resources|
|Bridge classes|InputStreamReader/OutputStreamWriter — byte ↔ character|
|Large files|`Files.lines()` — lazy stream, memory efficient|
|Encoding|Always specify UTF-8 explicitly — don't rely on platform default|
|Append mode|`new FileWriter("f.txt", true)` or `StandardOpenOption.APPEND`|
