## What & Why

Java networking enables communication over TCP/IP. The `java.net` package (Java 1.0+) provides low-level building blocks; `java.net.http` (Java 11+) offers a modern HTTP client. Most real-world code today uses Spring's `RestTemplate`/`WebClient`, OkHttp, or `HttpClient` rather than raw sockets — but the socket APIs remain interview territory and the foundation for everything built on top.

|Protocol|Reliability|Ordering|Use case|Java API|
|---|---|---|---|---|
|**TCP**|Guaranteed delivery|Ordered|HTTP, databases, most apps|`Socket`, `ServerSocket`|
|**UDP**|Best-effort (no guarantee)|Unordered|Streaming, DNS, VoIP, games|`DatagramSocket`, `DatagramPacket`|

---

## `java.net` Package Overview

Main classes:

- **`InetAddress`** — represents an IP address (v4 or v6)
- **`Socket` / `ServerSocket`** — TCP client / server endpoints
- **`DatagramSocket` / `DatagramPacket`** — UDP send/receive
- **`URL` / `HttpURLConnection`** — legacy HTTP client (rarely used in new code)
- **`java.net.http.HttpClient`** (Java 11+) — modern HTTP client (preferred)

---

## `InetAddress`

Represents an IP address; handles DNS lookups.

```java
InetAddress local = InetAddress.getLocalHost();          // your machine
InetAddress google = InetAddress.getByName("google.com"); // DNS lookup
InetAddress[] all = InetAddress.getAllByName("google.com"); // all resolved IPs

local.getHostName();       // "my-laptop"
local.getHostAddress();    // "192.168.1.5"
local.isReachable(2000);   // ping with 2s timeout
```

`InetAddress` objects are immutable and cached — JVM caches DNS lookups (configurable via `networkaddress.cache.ttl`).

## TCP Sockets — `Socket` and `ServerSocket`

### Server

```java
try (ServerSocket server = new ServerSocket(8080)) {
    System.out.println("Listening on port 8080...");
    while (true) {
        Socket client = server.accept();              // blocks until client connects
        new Thread(() -> handle(client)).start();     // handle in new thread
    }
}

void handle(Socket client) {
    try (var in = new BufferedReader(new InputStreamReader(client.getInputStream()));
         var out = new PrintWriter(client.getOutputStream(), true)) {      // true = autoflush
        String msg = in.readLine();
        out.println("Echo: " + msg);
    } catch (IOException e) { e.printStackTrace(); }
}
```

### Client

```java
try (Socket socket = new Socket("localhost", 8080);
     var out = new PrintWriter(socket.getOutputStream(), true);
     var in = new BufferedReader(new InputStreamReader(socket.getInputStream()))) {
    out.println("Hello");
    System.out.println(in.readLine());    // "Echo: Hello"
}
```

**Key points:**

- `accept()` **blocks** until a client connects — multi-client servers need threads, a thread pool, or NIO.
- `Socket` and `ServerSocket` implement `AutoCloseable` — always use try-with-resources.
- Use `socket.setSoTimeout(ms)` to avoid indefinite reads.

## UDP Sockets — `DatagramSocket` and `DatagramPacket`

No connection, no guarantees — each packet is independent.

### Sender

```java
try (DatagramSocket socket = new DatagramSocket()) {
    byte[] data = "Hello UDP".getBytes();
    InetAddress addr = InetAddress.getByName("localhost");
    DatagramPacket packet = new DatagramPacket(data, data.length, addr, 9090);
    socket.send(packet);
}
```

### Receiver

```java
try (DatagramSocket socket = new DatagramSocket(9090)) {
    byte[] buffer = new byte[1024];
    DatagramPacket packet = new DatagramPacket(buffer, buffer.length);
    socket.receive(packet);    // blocks until a packet arrives
    String msg = new String(packet.getData(), 0, packet.getLength());
    System.out.println("From " + packet.getAddress() + ": " + msg);
}
```

UDP packet size is capped by the IP MTU (typically 1500 bytes, minus overhead ≈ 1472 safe). For larger payloads, split into chunks or use TCP.

## `URL` and `HttpURLConnection` (legacy)

Original HTTP client. Clunky API, but still seen in old codebases.

```java
URL url = new URL("https://api.example.com/users");
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestMethod("GET");
conn.setRequestProperty("Accept", "application/json");

int status = conn.getResponseCode();    // 200
try (var in = new BufferedReader(new InputStreamReader(conn.getInputStream()))) {
    String body = in.lines().collect(Collectors.joining("\n"));
}
```

> **Prefer `java.net.http.HttpClient` (Java 11+)** for new code — cleaner API, HTTP/2, async support.

## Modern HTTP Client (Java 11+)

Fluent, feature-rich, built into the JDK. No external dependency.

```java
HttpClient client = HttpClient.newBuilder()
    .connectTimeout(Duration.ofSeconds(5))
    .build();

// Synchronous
HttpRequest req = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users"))
    .header("Accept", "application/json")
    .GET()
    .build();

HttpResponse<String> resp = client.send(req, HttpResponse.BodyHandlers.ofString());
System.out.println(resp.statusCode());    // 200
System.out.println(resp.body());

// Asynchronous
CompletableFuture<HttpResponse<String>> future =
    client.sendAsync(req, HttpResponse.BodyHandlers.ofString());
future.thenAccept(r -> System.out.println(r.body()));

// POST with JSON body
HttpRequest post = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users"))
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString("{\"name\":\"Alice\"}"))
    .build();
```

**When to use what:**

- Simple script / legacy — `HttpURLConnection`
- Modern plain Java — `HttpClient` (Java 11+)
- Spring app — `RestTemplate` (sync) or `WebClient` (reactive / async)

## Commonly Asked Gotchas

1. **`accept()` blocks — multi-client server needs threads** — a single-threaded server can only serve one client at a time. Use a thread pool (`ExecutorService`) or NIO `Selector` for real servers.

2. **Always set timeouts** — `socket.setSoTimeout(ms)` for reads, `HttpClient.connectTimeout(...)` for connections. Otherwise a stalled remote hangs your thread forever.

3. **UDP has no reliability guarantees** — packets can be lost, duplicated, or reordered. If you need those, use TCP or add your own sequence numbers / acknowledgements.

4. **`HttpURLConnection` doesn't close streams implicitly** — you must close input and error streams to free the underlying connection for reuse.

5. **DNS caching** — JVM caches DNS results (`networkaddress.cache.ttl`). If a hostname's IP changes (cloud load balancers), stale lookups can hit the wrong server until the cache expires.

6. **Port < 1024 needs privileges** — binding to ports 1-1023 usually requires root/admin on Unix-like systems.

## Quick Revision Cheat Sheet

|Concept|Remember|
|---|---|
|TCP vs UDP|TCP reliable + ordered; UDP best-effort, lightweight|
|`InetAddress`|IP address + DNS lookup. Immutable, cached|
|`ServerSocket.accept()`|Blocks until client connects — needs thread per client|
|`Socket`|TCP client endpoint. Use with streams (`getInputStream`/`getOutputStream`)|
|`DatagramSocket`|UDP — each `DatagramPacket` is independent|
|UDP packet size|~1472 bytes safe (1500 MTU − overhead). Larger = fragmentation|
|`HttpURLConnection`|Legacy HTTP client. Clunky. Prefer modern `HttpClient`|
|`java.net.http.HttpClient` (Java 11+)|Fluent API, HTTP/2, sync + async (`send` / `sendAsync`)|
|Timeouts|Always set — otherwise stalled remotes hang your thread|
|try-with-resources|All sockets implement `AutoCloseable` — always use it|
|In Spring apps|`RestTemplate` (sync) / `WebClient` (reactive) — not raw sockets|
