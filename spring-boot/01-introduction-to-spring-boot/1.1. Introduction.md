### Problems with Traditional Java Enterprise Development

- **Heavy application servers**: Required deploying to external servers like WebLogic, JBoss, GlassFish
- **Complex configuration**: Verbose XML-based configuration for wiring components
- **Slow development cycle**: Build → Package → Deploy → Test cycle was time-consuming
- **Inefficient JDBC and ORM integration**: Lots of manual setup and boilerplate
- **Boilerplate code**: Verbose, repetitive setup across projects
- **Tight coupling without DI**: Before DI adoption, classes created their own dependencies using `new`, making code hard to test and tightly coupled

### Spring Framework

- Introduced **Inversion of Control (IoC)** and **Dependency Injection (DI)** to solve coupling and testability problems
- Still required significant manual configuration (XML or Java config)

### Spring Boot

- **Auto-configuration**: Convention over configuration
- **Starter dependencies**: Curated dependency bundles with managed versions
- **Embedded servers**: Tomcat, Jetty, Undertow — no external server needed
- **Production-ready features**: Health checks, metrics, monitoring via Actuator
- **Supports in-memory databases**: H2, HSQLDB for development and testing

### Prerequisites: Core Spring Concepts

Before learning Spring Boot, understand these Spring fundamentals:

- **IoC and DI**: `@Component`, `@Autowired`, Constructor injection
- **Bean Management**: `@Configuration`, `@Bean`
- **ApplicationContext**: The IoC container that manages all beans
- **Spring MVC**: `@Controller`, `@RestController`, `@RequestMapping`

### Spring Boot Learning Path

1. Auto-configuration
2. Starter dependencies
3. Properties and YAML config (including profiles)
4. Spring Boot DevTools and Actuator
5. Spring Boot CLI
