### Build Tools

#### Maven

- Build and dependency management tool for Java projects
- Uses `pom.xml` to download dependencies, manage versions, and build the project

#### Gradle

- Alternative build tool using `build.gradle` (Groovy) or `build.gradle.kts` (Kotlin DSL)

---

### pom.xml

- Stands for **Project Object Model**
- Configuration file for Maven-based projects
- Defines the entire build lifecycle, plugins, profiles, project metadata, and dependencies (more comprehensive than `package.json` in the JS ecosystem)

#### Parent: `spring-boot-starter-parent`

- Provides default configurations
- Manages: Dependency versions, Plugin versions, Java version
- Internally uses `spring-boot-dependencies`

**Dependency Hierarchy:**
```
pom.xml
  ↓
spring-boot-starter-parent
  ↓
spring-boot-dependencies
```

#### Dependencies

- Define what features your app needs
- Each starter is a collection of related libraries
- Examples:
  - `spring-boot-starter-web` → Tomcat, Spring MVC, Jackson
  - `spring-boot-starter-data-jpa` → Hibernate, JPA, JDBC

#### Plugins: `spring-boot-maven-plugin`

- Package app as executable JAR
- Run app using `mvn spring-boot:run`
- Handles embedded server startup
