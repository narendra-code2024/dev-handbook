### What is a Bean?

- A Java object that is instantiated, configured, and managed by the Spring IoC container

### IoC Container

- Also known as **ApplicationContext** (preferred) or **BeanFactory** (lightweight, rarely used directly)
- Responsible for creating, wiring, and managing bean lifecycles

### How to Create Beans

1. **Using `@Component`** (and stereotype annotations: `@Service`, `@Repository`, `@Controller`)
   - Detected via component scanning
2. **Using `@Bean` in a `@Configuration` class**
   - Explicit bean definition, useful for third-party classes you can't annotate

### Bean Lifecycle

> Bean Created → Dependencies Injected → Bean Initialized → Bean Used → Bean Destroyed

### Bean Lifecycle Methods

- `@PostConstruct`: Invoked after all dependencies are injected. Use for initialization logic.
- `@PreDestroy`: Invoked just before the bean is destroyed by the container. Use for cleanup logic.

### Detailed Bean Lifecycle Order

1. Constructor called
2. Dependencies injected (`@Autowired`)
3. Aware interfaces called (`BeanNameAware`, `ApplicationContextAware`, etc.)
4. `BeanPostProcessor.postProcessBeforeInitialization()`
5. `@PostConstruct`
6. `InitializingBean.afterPropertiesSet()`
7. Custom `init()` method
8. `BeanPostProcessor.postProcessAfterInitialization()`

### Bean Scopes

| Scope | Description | Use Case |
|-------|-------------|----------|
| `singleton` | One instance per ApplicationContext (default) | Services, repositories |
| `prototype` | New instance every time it's requested | Stateful objects, request-scoped data |
| `request` | One instance per HTTP request | Web apps only |
| `session` | One instance per user HTTP session | Web apps only |
| `application` | One instance per ServletContext | Shared global config in web apps |
| `websocket` | One instance per WebSocket session | WebSocket apps |

- By default, beans are **singleton**
- Use `@Scope("prototype")` to override
