### What is DI?

- DI is a design pattern where a class receives its dependencies from the outside, instead of creating them itself.

### Types of DI

| Type | Annotation | Recommended? |
|------|-----------|--------------|
| Constructor | `@Autowired` optional if single constructor (Spring 4.3+). Required only when multiple constructors exist. | Yes, best practice |
| Setter | `@Autowired` on setter method | Okay for optional dependencies |
| Field | `@Autowired` on field | Avoid — hard to test, hides dependencies |

### Resolving Multiple Beans of the Same Type

#### Using `@Qualifier`
```java
@Autowired
public MyService(@Qualifier("s3Storage") StorageService storageService) {
    this.storageService = storageService;
}
```

- Bean name can be specified in stereotype annotations: `@Service("s3Storage")`
- If not specified: bean name = class name in camelCase
- For `@Bean` methods: bean name = method name by default

#### Using `@Primary`

- Mark one bean as `@Primary` — it will be chosen by default when no `@Qualifier` is specified

#### Injecting a Collection of Beans

**Using List** (all beans of that type):
```java
private final List<StorageService> services;

@Autowired
public StorageManager(List<StorageService> services) {
    this.services = services;
}
```

**Using Map** (bean name → bean instance):
```java
private final Map<String, StorageService> serviceMap;

@Autowired
public MappedStorageManager(Map<String, StorageService> serviceMap) {
    this.serviceMap = serviceMap;
}
```
