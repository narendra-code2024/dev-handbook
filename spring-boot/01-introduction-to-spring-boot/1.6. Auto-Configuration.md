### What Is It?

A mechanism that automatically configures beans based on:
- Dependencies present on the **classpath**
- **Properties** defined in configuration files
- **Conditional annotations** that evaluate at startup

Enabled by `@EnableAutoConfiguration`, which is included in `@SpringBootApplication`.

### Why It Exists

In traditional Spring, you had to define many beans manually and write XML or Java config for MVC, DataSource, JPA, etc. Spring Boot removes most of this setup and lets you focus on business logic.

### How It Works

#### 1. Classpath Scanning

Identifies project dependencies (from `pom.xml` or `build.gradle`)

#### 2. Locating Configuration Classes

- `spring-boot-autoconfigure.jar` contains all auto-configuration logic
- Uses metadata file: `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` (modern versions)
- Legacy: `META-INF/spring.factories`
- Spring Boot reads this metadata via `@EnableAutoConfiguration` during startup

#### 3. Conditional Configuration

- `@ConditionalOnClass`: Only if a specific class exists on the classpath
- `@ConditionalOnMissingBean`: Only if the user has NOT already defined that bean
- `@ConditionalOnProperty`: Only if a specific property is set
- `@ConditionalOnWebApplication`: Only for web applications

#### 4. Bean Registration

If all conditions are satisfied, Spring Boot instantiates the configuration class, registers beans with sensible defaults, and adds them to the ApplicationContext.

### @ConditionalOnProperty

Conditionally enables or disables beans and configuration classes based on property values.

Key attributes: `name` (or `value`), `prefix`, `havingValue`, `matchIfMissing`
```java
@Configuration
@ConditionalOnProperty(name = "feature.notifications.enabled", havingValue = "true")
public class NotificationConfig {
    // Only loaded if feature.notifications.enabled=true
}
```

### Examples

If classpath contains `spring-boot-starter-web`:
- Auto-configures: Tomcat, DispatcherServlet, Spring MVC beans, Jackson (JSON serialization)

If classpath contains `spring-boot-starter-data-jpa`:
- Auto-configures: DataSource, EntityManager, TransactionManager

### Core Features Summary

- Classpath-based configuration
- Conditional configuration
- Sensible default values
- Property-driven customization
- Starter-based
- Easy to override with your own beans
- Opt-out support via `exclude`:
```java
@SpringBootApplication(exclude = DataSourceAutoConfiguration.class)
```

- Debugging: Set `debug=true` in properties to see auto-configuration report

### What Is the Classpath?

The classpath is a list of locations (directories, JAR files) where the Java compiler and JVM look for `.class` files. It usually includes:
1. Your compiled application classes
2. All dependency JARs
