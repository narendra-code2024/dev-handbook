### 1. `main()` Method

- Executes `SpringApplication.run()`

### 2. Spring Boot Pre-Setup

- Detects application type: **Servlet** (Tomcat) / **Reactive** (Netty) / **Non-web**
- Loads `ApplicationContextInitializers` and `ApplicationListeners`
- Prepares Environment:
  - `application.properties` / `application.yml`
  - Profile-specific configs (`application-dev.properties`)
  - OS environment variables
  - JVM arguments
  - Profiles resolved (`spring.profiles.active`)

### 3. Banner Printed

- Default Spring banner, custom `banner.txt`, or disabled via config

### 4. ApplicationContext Created

Depending on app type:
- `AnnotationConfigServletWebServerApplicationContext` (most common — web apps)
- `ReactiveWebServerApplicationContext` (reactive apps)
- `AnnotationConfigApplicationContext` (non-web apps)

### 5. Configuration Processing

`@SpringBootApplication` internally enables:
- `@Configuration` — marks the class as a source of bean definitions
- `@EnableAutoConfiguration` — triggers auto-configuration
- `@ComponentScan` — scans for `@Component`, `@Service`, `@Repository`, `@Controller`, `@RestController`

### 6. Auto-Configuration

- Reads `AutoConfiguration.imports` (current) or `spring.factories` (legacy)
- Applies conditional annotations (`@ConditionalOnClass`, `@ConditionalOnMissingBean`, `@ConditionalOnProperty`)

### 7. Bean Definitions Created

- Spring registers **BeanDefinitions** (metadata only): bean name, scope, dependencies, init methods
- Beans are NOT fully instantiated yet at this stage

### 8. Bean Instantiation (IoC in Action)

- Creates singleton beans
- Resolves and injects dependencies (constructor → setter → field)
- Full bean lifecycle executes (see Bean Lifecycle Order in Beans page)

### 9. Embedded Web Server Starts

For web apps:
- Tomcat / Jetty / Undertow is created and started
- Port binding (e.g., 8080)
- Servlet context initialized

> **Note**: The server is created during context refresh. Bean instantiation and server startup are interleaved during this phase — they're not strictly sequential separate steps.

### 10. Startup Runners Execute

After context is fully ready:
```java
@Component
class StartupRunner implements CommandLineRunner {
    public void run(String... args) {
        // executes here
    }
}
```

Use for: DB seeding, cache warm-up, startup validation checks

### 11. Application Lifecycle Events Published

Key events in order:
- `ApplicationStartingEvent`
- `ApplicationEnvironmentPreparedEvent`
- `ApplicationPreparedEvent`
- `ContextRefreshedEvent`
- `ApplicationStartedEvent`
- `ApplicationReadyEvent` — app is fully ready

### 12. App Starts Serving Requests

> Client → DispatcherServlet → Controller → Service → Repository → DB

### One-Line Summary

> Spring Boot startup = Environment setup → Context creation → Auto-config + Bean lifecycle → Embedded server start → App ready

### Properties and Profiles

#### Configuration Files

- `application.properties` or `application.yml` — main configuration
- Profile-specific: `application-dev.properties`, `application-prod.properties`
- Activate profiles: `spring.profiles.active=dev`

#### Property Precedence (highest to lowest)

1. Command-line arguments
2. JVM system properties (`-D`)
3. OS environment variables
4. Profile-specific properties (`application-{profile}.properties`)
5. `application.properties` / `application.yml`
6. Default values in code
