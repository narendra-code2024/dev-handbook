### Maven Lifecycles

Maven has three independent lifecycles:

1. **Clean lifecycle**: `clean` — deletes `target/` directory
2. **Default lifecycle**: `validate → compile → test → package → verify → install → deploy`
3. **Site lifecycle**: `site` — generates project documentation

> `clean` is a separate lifecycle. Running `mvn clean install` invokes two lifecycles together.

### Common Maven Commands

| Command | Purpose |
|---------|---------|
| `mvn clean` | Deletes `target/` directory |
| `mvn compile` | Compiles source code → `target/classes` |
| `mvn test` | Runs unit tests (JUnit / TestNG) |
| `mvn package` | Creates JAR/WAR in `target/` |
| `mvn install` | Installs JAR/WAR into local repo (`~/.m2/repository`) |
| `mvn deploy` | Builds, tests, packages, and uploads artifact to remote repo (CI/CD) |
| `mvn spring-boot:run` | Runs the app directly without packaging a JAR/WAR. Ideal for local dev. Use with DevTools for auto-restart on changes. |
| `mvn spring-boot:build-image` | Builds a Docker image for the app (no Dockerfile required) |

### How Maven Works in Spring Boot

1. Reads `pom.xml`
2. Resolves dependencies from Maven Central Repository
3. Downloads and stores them in `~/.m2/repository`
4. Adds all resolved dependencies to the classpath

### External Libraries

- All downloaded dependencies appear as **External Libraries** in the IDE
- Key library to know: `spring-boot-autoconfigure`
