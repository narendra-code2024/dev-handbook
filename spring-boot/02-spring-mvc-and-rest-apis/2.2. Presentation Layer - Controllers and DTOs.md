### Presentation Layer

The Presentation Layer is responsible for:

- Handling HTTP requests
- Validating input
- Returning HTTP responses (JSON / XML / View)

### Spring Boot Web Project Structure (Real-world)

| Package | Responsibility |
|---------|---------------|
| `controller` | Request/Response handling (Presentation Layer) |
| `service` | Business logic |
| `repository` | Database access (JPA/Hibernate) |
| `dto` | Data Transfer Objects (API request/response) |
| `entity` | JPA Entities (Database table mappings) |
| `config` | Security, Swagger, etc. |

### Controllers in Spring MVC

Spring MVC provides an annotation-based programming model where controllers use annotations to define request mappings, input binding, validation, and exception handling.

#### @Controller vs @RestController

*@Controller*
- Used for MVC with views
- Returns a **view name** (String) by default
- Can also return `ModelAndView`
- Use `@ResponseBody` on individual methods to return data directly

*@RestController*
- Combines `@Controller` + `@ResponseBody`
- Automatically serializes return values to JSON/XML
- ViewResolver is not involved
- Used for REST APIs

### Request Mapping Annotations

#### @RequestMapping

Used to map HTTP requests to controller methods. It can match by:
- URL path
- HTTP method
- Request parameters
- Headers
- Media types (Consumes / Produces)
```java
@RequestMapping(value = "/products", method = RequestMethod.GET)
```

#### HTTP Method-Specific Shortcuts (Recommended)

Instead of `@RequestMapping`, use: `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`, `@PatchMapping`
```java
@GetMapping("/products")
public List<ProductDto> getProducts() {
    return productService.getAll();
}
```

### Dynamic URL Paths

Spring MVC supports dynamic request data using these annotations:

#### @RequestParam (Query Parameters)

Example URL: `/employees?id=123`
```java
@GetMapping("/employees")
public Employee getEmployee(@RequestParam Long id) {}
```

Key points:
- **Required by default** (`required = true`). Set `@RequestParam(required = false)` to make optional.
- Used for filtering, sorting, pagination, search
- Common in list/search APIs

#### @PathVariable (Path Parameters)

Example URL: `/employees/123`
```java
@GetMapping("/employees/{id}")
public Employee getEmployee(@PathVariable Long id) {}
```

Key points:
- Identifies a specific resource
- Always mandatory (part of the URL path)
- RESTful and preferred for resource identification

#### @RequestBody

Binds the HTTP request body to a Java object.
```java
@PostMapping("/employees")
public Employee createEmployee(@RequestBody EmployeeDto dto) {}
```

How it works:
- Client sends JSON (or XML)
- Spring uses HttpMessageConverter
- Jackson converts JSON â†’ Java object
- **Required by default** â€” if body is missing, Spring throws `HttpMessageNotReadableException`

Use cases:
- Typically used when client sends complex/structured data (POST, PUT, PATCH)

### DTO vs Entity (Important Best Practice)

**Don't expose Entity directly:**
- Tightly coupled to database schema
- Security risk (exposes internal structure)
- Changes to DB affect API contract

**Use DTOs:**
- Clean API contract
- Independent of DB structure
- Control what data is exposed
