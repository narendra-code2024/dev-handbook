
---

### Overview

Input Validation ensures that incoming request data is correct, complete, and safe.

It prevents:
- Invalid data persistence
- Unexpected errors
- Security issues (injection, overflow)

### Dependency
```xml
<groupId>org.springframework.boot</groupId>
<artifactId>spring-boot-starter-validation</artifactId>
```

---

### How Validation Works (Flow)

> Client Request → DTO with Validation Annotations → `@Controller` + `@Valid` → Spring Validation Engine → Exception if validation fails (before controller logic executes)

### @Valid in Controller

Triggers validation. If validation fails → `MethodArgumentNotValidException` is thrown automatically.
```java
@PostMapping("/users")
public ResponseEntity<UserDto> createUser(@Valid @RequestBody UserDto dto) {
    return ResponseEntity.ok(service.create(dto));
}
```

### Validation Annotations

#### Null & Empty Checks

| Annotation | Description |
|-----------|-------------|
| `@NotNull` | Field must not be null |
| `@NotEmpty` | Not null + size > 0 (String, Collection, Array) |
| `@NotBlank` | Not null + trimmed length > 0 (String only) |

#### Size & Length

| Annotation | Description |
|-----------|-------------|
| `@Size(min, max)` | Validates size or length |

#### Numeric Validations

| Annotation | Description |
|-----------|-------------|
| `@Min` | Minimum allowed value |
| `@Max` | Maximum allowed value |
| `@DecimalMin` | Min value (supports decimals) |
| `@DecimalMax` | Max value (supports decimals) |
| `@Positive` | > 0 |
| `@PositiveOrZero` | >= 0 |
| `@Negative` | < 0 |
| `@NegativeOrZero` | <= 0 |
| `@Digits` | Limits integer & fraction digits |

#### Pattern & Format

| Annotation | Description |
|-----------|-------------|
| `@Email` | Valid email format |
| `@Pattern` | Regex-based validation |

#### Date & Time

| Annotation | Description |
|-----------|-------------|
| `@Past` | Date must be in past |
| `@PastOrPresent` | Past or present |
| `@Future` | Date must be in future |
| `@FutureOrPresent` | Future or present |

#### Boolean Checks

| Annotation | Description |
|-----------|-------------|
| `@AssertTrue` | Must be true |
| `@AssertFalse` | Must be false |

### @Valid for Nested Object Validation
```java
@Valid
private AddressDto address;
```

- Triggers validation on nested objects recursively
- Applies all validation annotations inside `AddressDto`

---

### Handling Validation Exceptions

Spring throws `MethodArgumentNotValidException` on validation failure. This is thrown **before** controller logic executes.

Accessing validation errors inside exception handler:
```java
ex.getBindingResult().getFieldErrors();
```

You can extract field names, error messages, and return meaningful API responses.

### Typical Validation Error Response
```json
{
    "errors": {
        "email": "Invalid email format",
        "age": "Must be positive"
    }
}
```

---

### Best Practices

- Validate DTOs, not entities
- Always use `@Valid` with `@RequestBody`
- Centralize validation error handling using `@RestControllerAdvice`
- Return meaningful error messages

> **One-line Interview Answer:** Spring Boot input validation uses Bean Validation annotations on DTOs and the `@Valid` annotation in controllers to automatically validate request data and throw exceptions for invalid input.
