### Why Exception Handling?

- Java exception handling basics are a prerequisite
- Proper exception handling sends meaningful error messages to clients, returns appropriate HTTP status codes, simplifies API usage and debugging, and centralizes error handling logic

---

### @ResponseStatus (Simple Cases)

For simple exception-to-status mapping:
```java
@ResponseStatus(HttpStatus.NOT_FOUND)
public class ResourceNotFoundException extends RuntimeException {
    public ResourceNotFoundException(String message) {
        super(message);
    }
}
```

When this exception is thrown, Spring automatically returns a `404` response.

### Global Exception Handling with @RestControllerAdvice

- `@RestControllerAdvice` = `@ControllerAdvice` + `@ResponseBody`
- Applies globally to all controllers
- Intercepts exceptions thrown in any controller
- Returns custom responses as JSON

### Exception Handler Precedence

More **specific** exception handlers are matched first. If you have handlers for both `ResourceNotFoundException` and `Exception`, the specific one is used when a `ResourceNotFoundException` is thrown.

### How to Use @RestControllerAdvice
```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // Most specific — handles validation errors
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ValidationErrorResponse> handleValidationException(
        MethodArgumentNotValidException ex) {

        ValidationErrorResponse errorResponse = new ValidationErrorResponse();

        ex.getBindingResult().getFieldErrors().forEach(error -> {
            errorResponse.addError(error.getField(), error.getDefaultMessage());
        });

        return ResponseEntity.badRequest().body(errorResponse);
    }
    
    // Specific — handles resource not found
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleNotFound(ResourceNotFoundException ex) {
        ErrorResponse error = new ErrorResponse("NOT_FOUND", ex.getMessage());
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }

    // Catch-all — handles everything else
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleAll(Exception ex) {
        ErrorResponse error = new ErrorResponse("INTERNAL_ERROR", "Something went wrong");
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(error);
    }
}
```

| Annotation | Purpose |
|-----------|---------|
| `@RestControllerAdvice` | Defines global exception handler class |
| `@ExceptionHandler(ExceptionClass.class)` | Maps specific exception type to handler method |

### Custom Error Response Structure
```json
{
    "message": "Validation Failed",
    "status": "400 BAD_REQUEST",
    "timestamp": "2026-02-09T15:30:00Z",
    "subErrors": [
        {"field": "email", "message": "must be a valid email"},
        {"field": "age", "message": "must be positive"}
    ]
}
```

### Benefits of Global Exception Handling

- Avoids duplicated try-catch in every controller
- Centralizes error message formatting
- Provides consistent API error responses
- Single place to update error structure

> **One-line Interview Answer:** `@RestControllerAdvice` enables centralized exception handling in Spring Boot REST APIs, returning consistent and meaningful error responses for exceptions thrown in controllers.
