### What is Spring Data JPA?

- A framework that simplifies database access in Spring Boot by providing an abstraction layer over JPA
- Built on top of JPA (Specification) and Hibernate (Default implementation)
- Provides: Repository abstraction, Automatic query generation, Pagination & sorting, Reduced boilerplate code
- Main goal: Eliminate manual DAO implementation

Instead of writing:
```java
EntityManager.persist()
EntityManager.createQuery()
```

You write:
```java
interface UserRepository extends JpaRepository<User, Long> {
    List<User> findByName(String name);
}
```

And Spring generates the implementation automatically.

---

### Repository Hierarchy

> `Repository` (marker, no methods)
> ↓
> `CrudRepository` (basic CRUD)
> ↓
> `PagingAndSortingRepository` (pagination & sorting)
> ↓
> `JpaRepository` (batch ops, flush, JPA-specific)

---

### Key Features

#### 1. Repository Abstraction

- No need to implement DAO manually
- Spring auto-generates implementation at runtime

#### 2. Custom Query Methods
```java
List<User> findByName(String name);
```

Spring converts this to JPQL automatically.

#### 3. Pagination and Sorting
```java
Page<User> findAll(Pageable pageable);
```

Supports page number, page size, sorting direction.

#### 4. Query Derivation (Dynamic Query Methods)

Spring parses method names and builds queries automatically.
```java
List<User> findByEmail(String email);
```

Spring converts internally to JPQL:

> `SELECT u FROM User u WHERE u.email = :email`

Then Hibernate converts JPQL → SQL.

---

### Structure of a Query Method

A derived query method has 4 parts:

> `ReturnType` `QuerySubject` `By` `QueryPredicate(InputParams)`

Example:
```java
List<User> findByNameAndStatus(String name, String status);
```

---

### Rules for Creating Query Methods

#### 1. Method Prefix (Query Subject)

Common prefixes: `find…By`, `read…By`, `get…By`, `query…By`, `count…By`, `exists…By`, `delete…By`, `remove…By`
```java
findByName()
readByEmail()
queryByStatus()
getById()
```

#### 2. Limiting Results (First / Top)
```java
findFirstByName(String name);
findTop5ByStatus(String status);
findFirst3ByOrderByCreatedAtDesc();
```

#### 3. Distinct Results
```java
findDistinctByName(String name);
findNameDistinctByStatus(String status);
```

#### 4. Combine Conditions (And, Or)
```java
findByNameAndStatus(String name, String status);
findByEmailOrPhone(String email, String phone);
```

#### 5. Query Keywords

Spring supports many operators. See the [official docs](https://docs.spring.io/spring-data/jpa/reference/repositories/query-keywords-reference.html) for the full list.

---

### Nested Property Queries

If entity has a relationship:
```java
class Order {
    User user;
}
```

You can write:
```java
findByUserName(String name);
```

Spring automatically performs JOIN internally.

---

### When Query Method Is Not Enough

If method name becomes too long:
```java
findByNameAndStatusAndTypeAndCreatedDateBetween(...)
```

Use `@Query` annotation instead:
```java
@Query("SELECT u FROM User u WHERE u.name = :name")
List<User> findUserByName(@Param("name") String name);
```

Supports JPQL and Native SQL.

---

### How It Works Internally

When application starts:
1. Spring scans repository interfaces
2. Creates proxy implementation
3. Parses method names
4. Generates JPQL
5. Uses EntityManager internally
6. Hibernate executes SQL

You never implement DAO manually.

---

### Important Clarifications

- Spring Data JPA still uses JPA, EntityManager, Hibernate, and JDBC internally
- It just hides the complexity

---

### Why Use Spring Data JPA Instead of Plain JPA?

| JPA | Spring Data JPA |
|-----|----------------|
| Requires manual DAO | No DAO implementation |
| Uses EntityManager directly | Uses Repository abstraction |
| Manual query writing | Query derivation support |
| More boilerplate | Less boilerplate |

### When NOT to Use Spring Data JPA?

- Extremely complex dynamic queries → use Criteria API / QueryDSL
- Heavy performance-tuned systems → sometimes use plain Hibernate
- Bulk updates → use native queries

> **Interview Summary:** Spring Data JPA reduces boilerplate, eliminates manual DAO classes, supports query derivation from method names, pagination & sorting, uses proxy pattern internally, and still depends on JPA + Hibernate underneath.
