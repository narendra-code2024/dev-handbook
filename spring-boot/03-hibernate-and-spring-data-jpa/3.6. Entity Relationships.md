### Entity Relations

In relational databases, tables are connected using foreign keys. In JPA, we represent those using annotations.

| Relationship | Annotation |
|-------------|-----------|
| One to One | `@OneToOne` |
| One to Many | `@OneToMany` |
| Many to One | `@ManyToOne` |
| Many to Many | `@ManyToMany` |

---

### ManyToOne (Most Common)

Example: Many Employees → One Department

Database:
- `employee` → id, name, department_id (FK)
- `department` → id, name
```java
@Entity
class Employee {
    @Id
    private Long id;
    private String name;

    @ManyToOne
    @JoinColumn(name = "department_id") // Owning side: specifies FK column
    private Department department;
}

@Entity
class Department {
    @Id
    private Long id;
    private String name;
}
```

- `@ManyToOne` is always the owning side
- `@JoinColumn` is used on the owning side (entity that defines and manages the foreign key column)
- `@JoinColumn` also accepts `nullable` parameter (`true` or `false`)

---

### OneToMany

Example: One Department → Many Employees
```java
@Entity
class Department {
    @Id
    private Long id;
    private String name;

    @OneToMany(mappedBy = "department") // Inverse side: points to 'department' field in Employee
    private List<Employee> employees;
}
```

- `mappedBy` indicates this is the inverse (non-owning) side
- The owning side (`Employee.department`) manages the foreign key

---

### OneToOne

Example: One User → One Profile

Database:
- `user` → id
- `profile` → id, user_id (FK)
```java
@Entity
class Profile {
    @Id
    private Long id;

    @OneToOne
    @JoinColumn(name = "user_id") // Owning side
    private User user;
}

@Entity
class User {
    @Id
    private Long id;

    @OneToOne(mappedBy = "user") // Inverse side
    private Profile profile;
}
```

---

### ManyToMany

Example: Students ↔ Courses

Database: `student`, `course`, `student_course` (join table)
```java
@Entity
class Student {
    @Id
    private Long id;

    @ManyToMany
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private Set<Course> courses;
}

@Entity
class Course {
    @Id
    private Long id;

    @ManyToMany(mappedBy = "courses") // Inverse side
    private Set<Student> students;
}
```

- One side defines `@JoinTable` (owning side)
- Other side uses `mappedBy` (inverse side)
- Use `Set` instead of `List` to avoid Hibernate duplicate issues with ManyToMany

### Fetch Types

| Relation | Default Fetch |
|----------|--------------|
| `@ManyToOne` | EAGER |
| `@OneToOne` | EAGER |
| `@OneToMany` | LAZY |
| `@ManyToMany` | LAZY |

Override example:
```java
@ManyToOne(fetch = FetchType.LAZY)
```

### EAGER vs LAZY

- **EAGER**: Loads related entity immediately with the main entity
- **LAZY**: Loads related entity only when accessed
```java
Department dept = repository.findById(1L).get();
dept.getEmployees(); // Lazy load triggers query here
```

> **Best practice**: Default to LAZY and use fetch joins when you need related data. Avoid EAGER on collections.
