### What is Cascade?

Cascade defines what should happen to related entities when a parent entity operation is performed.
```java
@Entity
public class Order {
    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL)
    private List<OrderItem> items;
}
```

When you save/delete/update Order, operations can automatically propagate to OrderItem.

---

### Available Cascade Types

| Type | Behavior |
|------|----------|
| `CascadeType.PERSIST` | Save parent → child saved automatically |
| `CascadeType.MERGE` | Update parent → child updated automatically |
| `CascadeType.REMOVE` | Delete parent → child deleted. Be careful with this. |
| `CascadeType.REFRESH` | Reload parent from DB → child also refreshed |
| `CascadeType.DETACH` | Parent detached from persistence context → child also detached |
| `CascadeType.ALL` | Includes all of the above |

---

### Real Example
```java
@OneToMany(
    mappedBy = "order",
    cascade = CascadeType.ALL,
    orphanRemoval = true
)
private List<OrderItem> items;
```

What happens:
- Save order → items saved
- Delete order → items deleted
- Remove item from list → item deleted (because `orphanRemoval = true`)

---

### JPA Cascade vs Database Cascade

- **JPA Cascade**: Handled by Hibernate at application level
- **Database Cascade** (`ON DELETE CASCADE`): Handled by database engine

They are different layers. Best practice: Use JPA cascade for object-level control.

---

### What is orphanRemoval?

`orphanRemoval = true` means: If a child entity is removed from the parent's collection, it should also be **deleted from the database**.

Works only with `@OneToMany` and `@OneToOne`.

#### Why is it called "Orphan"?

An orphan = a child entity that no longer has a parent.

> Order ├── Item1 ├── Item2
> Remove Item1 from collection:
> Order ├── Item2
> Item1 has no parent → it becomes an orphan.
> With orphanRemoval = true → Hibernate DELETEs it from DB.
> Without it → it stays in DB with a null foreign key.

---

### Cascade REMOVE vs orphanRemoval

| Action | cascade = REMOVE | orphanRemoval = true |
|--------|-----------------|---------------------|
| Delete parent | Child deleted | Child deleted |
| Remove child from collection | Not deleted | Deleted |

---

### orphanRemoval Requirements

- Must be within an **active transaction** (persistence context must be open)
- Must maintain **both sides** of bidirectional relationship correctly
- If bidirectional mapping is not maintained → unexpected behavior

### When to Use orphanRemoval

**Use when:**
- Child cannot exist without parent
- Lifecycle of child depends on parent
- Examples: Order → OrderItems, Blog → Comments, Invoice → InvoiceLines

**Do NOT use when:**
- Child can exist independently
- Examples: Student → Course (ManyToMany), Employee → Department
