### What is a Transaction?

A transaction ensures **ACID** properties:
- **Atomicity** — all operations succeed or all are rolled back
- **Consistency** — data moves from one valid state to another
- **Isolation** — concurrent transactions don't interfere
- **Durability** — committed data survives system failures

Example: If payment fails → order insert is rolled back.
```java
@Transactional
public void createOrder(Order order) {
    orderRepository.save(order);
    paymentService.processPayment(order);
}
```

---

### How Spring Manages Transactions

Spring uses:
- **AOP Proxy** — wraps your service methods
- **PlatformTransactionManager** — manages transaction lifecycle
- JDBC / Hibernate underneath

You don't manually open/commit transactions.

---

### Default Behavior of @Transactional

| Feature | Default |
|---------|---------|
| Propagation | `REQUIRED` |
| Isolation | DB Default |
| Rollback | RuntimeException only |
| Read-only | `false` |

---

### Propagation Types

| Type | Behavior |
|------|----------|
| `REQUIRED` (default) | Join existing transaction or create new one |
| `REQUIRES_NEW` | Always create new transaction (suspends existing) |
| `SUPPORTS` | Use transaction if exists, else run without |
| `MANDATORY` | Must have existing transaction, throws exception if none |
| `NOT_SUPPORTED` | Suspends existing transaction, runs without one |
| `NEVER` | Must NOT have existing transaction, throws exception if one exists |
| `NESTED` | Creates a savepoint within existing transaction |

---

### Isolation Levels

Controls visibility of changes between concurrent transactions.

| Level | Dirty Read | Non-Repeatable Read | Phantom Read |
|-------|-----------|-------------------|-------------|
| `READ_UNCOMMITTED` | Possible | Possible | Possible |
| `READ_COMMITTED` | No | Possible | Possible |
| `REPEATABLE_READ` | No | No | Possible |
| `SERIALIZABLE` | No | No | No |
```java
@Transactional(isolation = Isolation.REPEATABLE_READ)
```

---

### Rollback Behavior

- Rolls back on **unchecked exceptions** (`RuntimeException` and subclasses) by default
- Does **NOT** roll back on **checked exceptions** unless specified:
```java
@Transactional(rollbackFor = Exception.class)  // rolls back on all exceptions
```

---

### Read-only Transactions

Improves performance and prevents accidental updates:
```java
@Transactional(readOnly = true)
public List<Order> getOrders() {
    return orderRepository.findAll();
}
```

---

### Important Gotchas

**Self-invocation problem:** Calling a `@Transactional` method from within the same class will NOT trigger the transaction, because the AOP proxy is bypassed.
```java
@Service
public class OrderService {
    public void process() {
        this.placeOrder(order); // Transaction NOT applied!
    }

    @Transactional
    public void placeOrder(Order order) { ... }
}
```

Always call `@Transactional` methods from another bean.

**Private methods:** `@Transactional` on private methods won't work (proxy limitation).

**Missing @Transactional in service layer:** Leads to `LazyInitializationException` because persistence context closes immediately.

---

### Common Mistakes

- Using `cascade = ALL` everywhere → leads to accidental mass deletion
- Missing `@Transactional` in service layer → `LazyInitializationException`
- Putting `@Transactional` on private methods → won't work
- Not specifying `rollbackFor` → checked exceptions silently commit
