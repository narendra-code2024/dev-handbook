### What is Auditing?

Auditing in Spring Boot allows you to automatically populate certain fields such as:
- Creation and modification **timestamps**
- The **user** who created or modified the entity

---

### Steps to Add Auditing

#### 1. Create Auditable Base Entity
```java
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class Auditable {

    @CreatedBy
    @Column(updatable = false)
    private String createdBy;

    @CreatedDate
    @Column(updatable = false)
    private LocalDateTime createdDate;

    @LastModifiedBy
    private String lastModifiedBy;

    @LastModifiedDate
    private LocalDateTime lastModifiedDate;

    // getters and setters
}
```

- `@MappedSuperclass` — makes this a base class (not a separate table)
- `@EntityListeners(AuditingEntityListener.class)` — enables audit event listening
- `@CreatedBy` / `@CreatedDate` — populated on first save
- `@LastModifiedBy` / `@LastModifiedDate` — populated on every update

#### 2. Extend Entities from the Base Class
```java
@Entity
public class Employee extends Auditable {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
}
```

Now `Employee` automatically gets `createdBy`, `createdDate`, `lastModifiedBy`, `lastModifiedDate` columns.

#### 3. Implement AuditorAware

Provides the current authenticated user from Spring Security.
```java
@Component
public class AuditorAwareImpl implements AuditorAware<String> {

    @Override
    public Optional<String> getCurrentAuditor() {
        // Get current user from Spring Security
        return Optional.ofNullable(SecurityContextHolder.getContext())
            .map(SecurityContext::getAuthentication)
            .filter(Authentication::isAuthenticated)
            .map(Authentication::getName);
    }
}
```

If not using Spring Security, you can return a hardcoded value for development:
```java
return Optional.of("system");
```

#### 4. Enable JPA Auditing
```java
@Configuration
@EnableJpaAuditing(auditorAwareRef = "auditorAwareImpl")
public class AuditConfig {
}
```

- `auditorAwareRef` — references the bean name of your `AuditorAware` implementation

---

### Internal Working of Auditing

When an entity is persisted or updated, `AuditingEntityListener` triggers:

- **@PrePersist** — fills `@CreatedBy`, `@CreatedDate`, `@LastModifiedBy`, `@LastModifiedDate`
- **@PreUpdate** — fills `@LastModifiedBy`, `@LastModifiedDate` only

The `AuditorAware` interface provides the current user information to populate `@CreatedBy` and `@LastModifiedBy`.

### Hibernate Envers (Advanced Auditing)

Spring Boot auditing tracks **who** changed data and **when**. But it doesn't track **what** was changed (previous values). For that, use **Hibernate Envers**.

#### What is Envers?

- A Hibernate module that provides **full entity versioning and audit logging**
- Automatically creates audit tables that store every historical version of an entity
- Every insert, update, and delete is recorded

#### Dependency
```xml
<dependency>
    <groupId>org.hibernate.orm</groupId>
    <artifactId>hibernate-envers</artifactId>
</dependency>
```

#### Usage

Simply add `@Audited` to your entity:
```java
@Entity
@Audited
public class Employee {
    @Id
    private Long id;
    private String name;
    private Double salary;
}
```

Hibernate automatically creates an `employee_aud` table and a `revinfo` (revision info) table.

#### What Gets Stored

When you update an employee's salary from 50000 to 60000:

| id | name | salary | rev | revtype |
|----|------|--------|-----|---------|
| 1 | John | 50000 | 1 | 0 (INSERT) |
| 1 | John | 60000 | 2 | 1 (UPDATE) |

- `rev` — revision number
- `revtype` — 0 = INSERT, 1 = UPDATE, 2 = DELETE

#### Real-World Use Cases

- **Compliance & Regulatory** — financial apps, healthcare systems that must track every data change
- **Undo/Revert** — allow users to roll back to a previous version
- **Dispute Resolution** — "who changed this order and when?"
- **Data Recovery** — restore accidentally modified/deleted records

---

### Spring Boot Auditing vs Hibernate Envers

| Feature                       | Spring Boot Auditing | Hibernate Envers |
| ----------------------------- | -------------------- | ---------------- |
| Tracks who & when             | Yes                  | Yes              |
| Tracks what changed (history) | No                   | Yes              |
| Stores previous values        | No                   | Yes              |
| Separate audit tables         | No                   | Yes              |
| Setup complexity              | Simple               | Moderate         |
| Use case                      | Basic audit fields   | Full audit trail |
