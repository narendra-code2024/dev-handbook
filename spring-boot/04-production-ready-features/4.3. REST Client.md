### What is RestClient?

- A **synchronous** HTTP client introduced in Spring Framework 6.1 / Spring Boot 3.2
- Provides a modern, fluent API for making HTTP requests
- Abstracts over HTTP libraries (Apache HttpClient, JDK HttpClient, etc.)
- Converts Java objects to HTTP requests and HTTP responses to Java objects

---

### Evolution of HTTP Clients in Spring

| Client | Type | Status |
|--------|------|--------|
| `RestTemplate` | Synchronous, blocking | Legacy — still works but no new features |
| `WebClient` | Asynchronous, non-blocking (Project Reactor) | Use for reactive apps or when you need non-blocking I/O |
| `RestClient` | Synchronous, blocking | **Recommended** — modern replacement for RestTemplate |

- Use **RestClient** for standard synchronous REST calls (most common)
- Use **WebClient** only if your app is reactive or you need non-blocking behavior

### Real-World Example: Employee Service Client

#### application.properties
```properties
employee-service.base-url=http://localhost:8081/api
employee-service.username=admin
employee-service.password=secret
```

#### Configuration Properties
```java
@ConfigurationProperties(prefix = "employee-service")
public class EmployeeServiceProperties {
    private String baseUrl;
    private String username;
    private String password;
    // getters and setters
}
```

#### Client Interface
```java
public interface EmployeeClient {
    EmployeeResponse getEmployee(Long id);
    List<EmployeeResponse> getAllEmployees();
    EmployeeResponse createEmployee(EmployeeRequest request);
    void deleteEmployee(Long id);
}
```

#### Client Implementation
```java
@Component
public class EmployeeClientImpl implements EmployeeClient {

    private final RestClient restClient;

    public EmployeeClientImpl(RestClient restClient) {
        this.restClient = restClient;
    }

    @Override
    public EmployeeResponse getEmployee(Long id) {
        return restClient.get()
            .uri("/employees/{id}", id)
            .accept(MediaType.APPLICATION_JSON)
            .retrieve()
            .body(EmployeeResponse.class);
    }

    @Override
    public List<EmployeeResponse> getAllEmployees() {
        return restClient.get()
            .uri("/employees")
            .accept(MediaType.APPLICATION_JSON)
            .retrieve()
            .body(new ParameterizedTypeReference<>() {});
    }

    @Override
    public EmployeeResponse createEmployee(EmployeeRequest request) {
        return restClient.post()
            .uri("/employees")
            .contentType(MediaType.APPLICATION_JSON)
            .body(request)
            .retrieve()
            .body(EmployeeResponse.class);
    }

    @Override
    public void deleteEmployee(Long id) {
        restClient.delete()
            .uri("/employees/{id}", id)
            .retrieve()
            .onStatus(HttpStatusCode::is4xxClientError, (req, res) -> {
                throw new RuntimeException("Employee not found: " + res.getStatusText());
            })
            .toBodilessEntity();
    }
}
```

#### REST Client Configuration
```java
@Configuration
@EnableConfigurationProperties(EmployeeServiceProperties.class)
public class RestClientConfig {

    @Bean
    public RestClient restClient(EmployeeServiceProperties properties) {
        return RestClient.builder()
            .baseUrl(properties.getBaseUrl())
            .defaultHeader(HttpHeaders.AUTHORIZATION,
                encodeBasic(properties.getUsername(), properties.getPassword()))
            .build();
    }

    private String encodeBasic(String username, String password) {
        String credentials = username + ":" + password;
        return "Basic " + Base64.getEncoder()
            .encodeToString(credentials.getBytes());
    }
}
```

### Available HTTP Methods

- `restClient.get()` — GET request
- `restClient.post()` — POST request
- `restClient.put()` — PUT request
- `restClient.patch()` — PATCH request
- `restClient.delete()` — DELETE request

---

### Handling Errors
```java
restClient.get()
    .uri("/employees/{id}", id)
    .retrieve()
    .onStatus(HttpStatusCode::is4xxClientError, (req, res) -> {
        throw new EmployeeNotFoundException("Not found: " + res.getStatusText());
    })
    .onStatus(HttpStatusCode::is5xxServerError, (req, res) -> {
        throw new ServiceUnavailableException("Employee service down");
    })
    .body(EmployeeResponse.class);
```

- `onStatus()` — intercepts specific HTTP status codes
- Chain multiple `onStatus()` for different error types
- If no `onStatus()` is defined, non-2xx responses throw `RestClientResponseException` by default

### Multiple RestClient Beans (Multiple Services)

When calling multiple external services, create a separate `RestClient` bean for each since they have different base URLs, auth, and configurations.

#### Configuration
```java
@Configuration
public class RestClientConfig {

    @Bean
    public RestClient employeeRestClient(EmployeeServiceProperties props) {
        return RestClient.builder()
            .baseUrl(props.getBaseUrl())
            .defaultHeader(HttpHeaders.AUTHORIZATION, encodeBasic(props))
            .build();
    }

    @Bean
    public RestClient paymentRestClient(PaymentServiceProperties props) {
        return RestClient.builder()
            .baseUrl(props.getBaseUrl())
            .defaultHeader("X-API-KEY", props.getApiKey())
            .build();
    }
}
```

#### Inject Using @Qualifier
```java
public class EmployeeClientImpl implements EmployeeClient {
    public EmployeeClientImpl(@Qualifier("employeeRestClient") RestClient restClient) {
        this.restClient = restClient;
    }
}

public class PaymentClientImpl implements PaymentClient {
    public PaymentClientImpl(@Qualifier("paymentRestClient") RestClient restClient) {
        this.restClient = restClient;
    }
}
```

Each client gets its own `RestClient` with service-specific configuration.

### Package Structure for REST Clients

| Package | Class | Purpose |
|---------|-------|---------|
| `config` | `RestClientConfig.java` | REST client bean configuration |
| `config` | `EmployeeServiceProperties.java` | Configuration properties class |
| `client` | `EmployeeClient.java` | Client interface |
| `client/impl` | `EmployeeClientImpl.java` | Client implementation |
| `dto/external` | `EmployeeRequest.java` | Request DTO to external service |
| `dto/external` | `EmployeeResponse.java` | Response DTO from external service |

Keep `client` separate from `service` — services contain your business logic, clients are for calling external APIs.
