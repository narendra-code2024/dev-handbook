### What is OpenAPI?

- **OpenAPI** is a specification for describing REST APIs in a standard, language-agnostic format
- **Swagger UI** is a tool that renders OpenAPI specs as interactive, browsable API documentation
- Developers and testers can explore and test endpoints directly from the browser

### Springdoc OpenAPI

A library that automatically generates OpenAPI 3 documentation from your Spring MVC endpoints and provides Swagger UI out of the box.

### Dependency
```xml
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.5.0</version>
</dependency>
```

---

### Access URLs

| URL | Purpose |
|-----|---------|
| `/swagger-ui.html` | Interactive Swagger UI |
| `/v3/api-docs` | Raw OpenAPI JSON spec |

No additional configuration needed — just add the dependency and these URLs are available.

---

### Optional Configuration
```properties
# Custom API docs path
springdoc.api-docs.path=/api-docs

# Custom Swagger UI path
springdoc.swagger-ui.path=/swagger-ui.html

# Disable in production
springdoc.api-docs.enabled=false
springdoc.swagger-ui.enabled=false
```

### Common Annotations (Optional — for enriching docs)

| Annotation | Purpose |
|-----------|---------|
| `@Tag(name = "Employees")` | Groups endpoints under a label (on controller class) |
| `@Operation(summary = "Get employee by ID")` | Describes a specific endpoint (on method) |
| `@Parameter(description = "Employee ID")` | Describes a method parameter |
| `@ApiResponse(responseCode = "200")` | Documents possible responses |
| `@Schema(description = "Employee name")` | Describes DTO fields |

These are optional — Springdoc auto-generates docs from your controller methods even without them.
