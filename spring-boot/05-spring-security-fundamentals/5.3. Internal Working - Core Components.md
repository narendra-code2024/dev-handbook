### SecurityContext

- Holds the currently authenticated user
- Stored in `ThreadLocal` — each request thread has its own
- Access it anywhere:

```java
Authentication auth = SecurityContextHolder.getContext().getAuthentication();
String email = auth.getName();
```

---

### Authentication Object

Represents the authenticated user. Contains:

- **Principal** — who (email/username)
- **Credentials** — password (usually erased after authentication)
- **Authorities** — roles/permissions (`ROLE_USER`, `ROLE_ADMIN`)

---

### UserDetails

- Spring Security's interface for representing a user
- Your database User entity is NOT a UserDetails — you must convert it
- Methods: `getUsername()`, `getPassword()`, `getAuthorities()`

---

### UserDetailsService

- Interface with one method: `loadUserByUsername(String username)`
- **The bridge between your database and Spring Security**
- Spring calls this during authentication
- You implement it, fetch from DB, return UserDetails
- Two built-in implementations:
    - `InMemoryUserDetailsManager` — stores users in memory (testing/demos)
    - Your custom `MyUserDetailsServiceImpl` — fetches from database

```java
@Service
@RequiredArgsConstructor
public class CustomUserDetailsService implements UserDetailsService {

    private final UserRepository userRepository;

    @Override
    public UserDetails loadUserByUsername(String email)
            throws UsernameNotFoundException {

        User user = userRepository.findByEmail(email)
            .orElseThrow(() ->
                new UsernameNotFoundException("Invalid credentials"));

        return org.springframework.security.core.userdetails.User.builder()
            .username(user.getEmail())
            .password(user.getPassword())
            .authorities(
                user.getRoles().stream()
                    .map(SimpleGrantedAuthority::new)
                    .collect(Collectors.toList())
            )
            .build();
    }
}
```

> Method is called `loadUserByUsername` but we pass email — Spring doesn't care what the value represents, it just passes whatever the user provides during login.

### PasswordEncoder

- Encodes and verifies passwords
- **NEVER store plain text passwords**
- Methods: `encode()` — hash a raw password, `matches()` — compare raw vs encoded
- Always use `BCryptPasswordEncoder`

```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```

### AuthenticationManager and AuthenticationProvider

```
AuthenticationManager
  │  Authentication authenticate(Authentication obj);
  │
  ▼  implemented by
ProviderManager
  │  holds a list: List<AuthenticationProvider> providers;
  │  loops through each provider
  │
  ▼  one of many AuthenticationProviders
AuthenticationProvider
  │  boolean supports();
  │  Authentication authenticate(Authentication obj);
  │
  ├──► DaoAuthenticationProvider (username/password from DB)
  ├──► InMemoryAuthenticationProvider (in-memory users)
  ├──► OAuth2AuthenticationProvider (OAuth2 login)
  └──► ...
```

- **AuthenticationManager** — interface that orchestrates authentication
- **ProviderManager** — the default implementation, holds a list of `AuthenticationProvider`s and loops through them
- **AuthenticationProvider** — does the actual authentication. Each type supports a different authentication mechanism
- **DaoAuthenticationProvider** — the one we use. It calls `UserDetailsService` to fetch user details and `PasswordEncoder` to verify the password

```java
@Bean
public AuthenticationManager authenticationManager(
        AuthenticationConfiguration config) throws Exception {
    return config.getAuthenticationManager();
}
```

### Full Authentication Flow (DaoAuthenticationProvider)

```
authenticationManager.authenticate(email, password)
       │
       ▼
ProviderManager loops through providers
       │
       ▼
DaoAuthenticationProvider selected (supports username/password)
       │
       ├──► UserDetailsService.loadUserByUsername(email)
       │           │
       │           ▼
       │    UserRepository.findByEmail(email)
       │           │
       │           ▼
       │    User Entity → converted to → UserDetails
       │
       ├──► PasswordEncoder.matches(rawPassword, encodedPassword)
       │
       ▼
 Match?
  ├── Yes → returns Authentication object (with principal + authorities)
  └── No  → throws AuthenticationException
```
