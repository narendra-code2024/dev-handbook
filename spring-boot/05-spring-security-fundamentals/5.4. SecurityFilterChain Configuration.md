### Basic Setup

```java
@Configuration
@EnableWebSecurity
@RequiredArgsConstructor
public class SecurityConfig {

    private final JwtAuthFilter jwtAuthFilter;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http)
            throws Exception {

        http
            .csrf(csrf -> csrf.disable())

            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))

            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/register").permitAll()
                .requestMatchers("/api/auth/login").permitAll()
                .anyRequest().authenticated()
            )

            .addFilterBefore(jwtAuthFilter,
                UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }
}
```

---

### Key Configurations Explained

#### CSRF

> `.csrf(csrf -> csrf.disable())`

Disabled because we use JWT in `Authorization: Bearer` header — browser doesn't auto-send headers, so CSRF isn't possible. If your SPA uses HttpOnly cookies instead, add `SameSite=Lax` to the cookie or enable CSRF protection. See Page 1 (CSRF Prevention) for details.

#### Session Management

> `SessionCreationPolicy.STATELESS` — no sessions, no JSESSIONID cookie (for JWT) `SessionCreationPolicy.IF_REQUIRED` — default, creates session when needed (for session auth)

#### URL Authorization Rules

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/register").permitAll()
    .requestMatchers("/api/auth/login").permitAll()
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    .anyRequest().authenticated()
)
```

> `.anyRequest().authenticated()` is the catch-all — anything not explicitly listed above requires login.

#### Filter Order

> `.addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)`

Your JwtAuthFilter runs BEFORE Spring's default UsernamePasswordAuthenticationFilter.

### Session Auth vs JWT Config Comparison

|Config|Session Auth|JWT Auth|
|---|---|---|
|CSRF|Enable|Disable|
|Session|Default (IF_REQUIRED)|STATELESS|
|Custom Filter|Not needed|`addFilterBefore(jwtAuthFilter)`|

> **What about `.formLogin()` and `.httpBasic()`?** These are Spring's **built-in login mechanisms** — `.formLogin()` generates a login page at `/login`, `.httpBasic()` shows a browser popup asking for credentials. In real applications, **you don't need either**. Your frontend has its own login page that sends a POST request to your `/api/auth/login` endpoint, and your `AuthController` handles it. This is true for both session-based and JWT-based auth — Spring's built-in login UI is only useful for quick prototyping or testing without a frontend.
