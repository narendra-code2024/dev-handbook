### The Problem

- `JwtAuthFilter` runs **before** DispatcherServlet
- `@RestControllerAdvice` (GlobalExceptionHandler) only catches exceptions **inside** DispatcherServlet
- JWT exceptions (expired, malformed, invalid signature) happen in the filter → never reach GlobalExceptionHandler

```
Request
   │
   ▼
Filters (JWT exceptions happen here)
   │  ✗ Can't use @RestControllerAdvice
   │
   ▼
DispatcherServlet
   │
   ▼
Controller (auth exceptions happen here)
      ✓ @RestControllerAdvice works here
```

---

### Solution: Handle JWT Exceptions in Filter

Use `ObjectMapper` to manually write JSON response in your `ApiResponse` format:

```java
private void sendError(HttpServletResponse response,
        HttpStatus status, String message) throws IOException {
    response.setStatus(status.value());
    response.setContentType(MediaType.APPLICATION_JSON_VALUE);
    ApiResponse<?> error = ApiResponse.builder()
        .status(status.value())
        .message(message)
        .build();
    objectMapper.writeValue(response.getWriter(), error);
}
```

### JWT Exceptions Handled in Filter

|Exception|Cause|HTTP Status|
|---|---|---|
|`ExpiredJwtException`|Token past expiry time|401|
|`SignatureException`|Token tampered with|401|
|`MalformedJwtException`|Not a valid JWT format|401|
|`UnsupportedJwtException`|Unsupported JWT type|401|
|Generic `Exception`|Any other issue|401|

### Controller-Level Exceptions (GlobalExceptionHandler)

These ARE caught by `@RestControllerAdvice` since they happen inside DispatcherServlet:

|Exception|Cause|HTTP Status|
|---|---|---|
|`AuthenticationException`|Invalid email or password (login)|401|
|`BadCredentialsException`|Wrong password|401|
|`AccountExpiredException`|Account expired|401|
|`CredentialsExpiredException`|Password expired|401|
|`IllegalArgumentException`|Email already taken (register)|400|

```java
@ExceptionHandler(AuthenticationException.class)
public ResponseEntity<ApiResponse<Void>> handleAuthException(
        AuthenticationException ex) {
    return buildResponse(HttpStatus.UNAUTHORIZED,
        "Invalid email or password", null);
}
```

### Consistent Error Response (everywhere)

```json
{
    "timestamp": "2026-03-01T...",
    "status": 401,
    "message": "Token has expired"
}
```

Both filter-level and controller-level errors use the same `ApiResponse` format.
