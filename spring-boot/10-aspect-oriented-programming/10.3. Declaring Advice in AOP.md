### What is Advice?

The actual code that runs at a matched join point. You've seen `@Before` — but Spring AOP provides five advice types that let you run code at different points around a method execution.

```
@Around (before proceed)
  │
  ▼
@Before
  │
  ▼
Method executes
  │
  ▼
@AfterReturning (if success) or @AfterThrowing (if exception)
  │
  ▼
@After (always — like finally block)
  │
  ▼
@Around (after proceed)
```

---

### 1. @Before

Runs **before** the method executes. Can access method info via `JoinPoint` but **cannot** stop the method from running.

```java
@Before("serviceMethods()")
public void logBefore(JoinPoint joinPoint) {
    log.info("→ Entering: {}.{}",
        joinPoint.getTarget().getClass().getSimpleName(),
        joinPoint.getSignature().getName());
}
```

```
→ Entering: EmployeeServiceImpl.getEmployeeById
```

---

### 2. @After

Runs **after** the method finishes — whether it returned normally or threw an exception. Like a `finally` block.

```java
@After("serviceMethods()")
public void logAfter(JoinPoint joinPoint) {
    log.info("← Finished: {}", joinPoint.getSignature().getName());
}
```

```
← Finished: getEmployeeById    (runs regardless of success or failure)
```

---

### 3. @AfterReturning

Runs **only if** the method returned successfully. Can access the **return value**.

```java
@AfterReturning(pointcut = "serviceMethods()", returning = "result")
public void logAfterReturning(JoinPoint joinPoint, Object result) {
    log.info("✓ {} returned: {}",
        joinPoint.getSignature().getName(), result);
}
```

```
✓ getEmployeeById returned: EmployeeDto(id=1, name=John)
```

> `returning = "result"` binds the method's return value to the `result` parameter.

---

### 4. @AfterThrowing

Runs **only if** the method threw an exception. Can access the **exception**.

```java
@AfterThrowing(pointcut = "serviceMethods()", throwing = "ex")
public void logAfterThrowing(JoinPoint joinPoint, Exception ex) {
    log.error("✗ {} threw: {}",
        joinPoint.getSignature().getName(), ex.getMessage());
}
```

```
✗ getEmployeeById threw: Employee not found with id: 999
```

> `throwing = "ex"` binds the thrown exception to the `ex` parameter.

---

### 5. @Around (Most Powerful)

Wraps the entire method — runs code **before and after**, controls whether the method executes, can modify the return value or catch exceptions.

Uses `ProceedingJoinPoint` instead of `JoinPoint` — the `proceed()` method actually calls the target method.

```java
@Around("serviceMethods()")
public Object logAround(ProceedingJoinPoint joinPoint) throws Throwable {
    String methodName = joinPoint.getSignature().getName();

    log.info("→ Before: {}", methodName);

    Object result = joinPoint.proceed();  // actual method runs here

    log.info("← After: {} returned: {}", methodName, result);

    return result;  // must return the result
}
```

```
→ Before: getEmployeeById
← After: getEmployeeById returned: EmployeeDto(id=1, name=John)
```

> **Important:** You must call `joinPoint.proceed()` — otherwise the actual method never runs. You must also return the result — otherwise the caller (e.g., controller that called the service method) gets null instead of the actual return value.

> **Void methods:** `proceed()` returns `null` for void methods. Same code works — Spring ignores the return value for void methods.

### Practical Example — Measure Execution Time

`@Around` is perfect for this — you need code before _and_ after the method:

```java
@Aspect
@Component
public class PerformanceAspect {

    private static final Logger log =
        LoggerFactory.getLogger(PerformanceAspect.class);

    @Around("@annotation(com.app.annotation.LogExecutionTime)")
    public Object measureTime(ProceedingJoinPoint joinPoint)
            throws Throwable {

        long start = System.currentTimeMillis();

        Object result = joinPoint.proceed();

        long duration = System.currentTimeMillis() - start;

        log.info("{}.{} executed in {}ms",
            joinPoint.getTarget().getClass().getSimpleName(),
            joinPoint.getSignature().getName(),
            duration);

        return result;
    }
}
```

```java
// Custom annotation (from Page 10.2)
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface LogExecutionTime {
}
```

```java
// Usage — just add the annotation to any method
@Service
public class EmployeeServiceImpl {

    @LogExecutionTime
    public EmployeeDto getEmployeeById(Long id) {
        // business logic
    }
}
```

```
Output: EmployeeServiceImpl.getEmployeeById executed in 45ms
```

> Add `@LogExecutionTime` to any method you want to measure — no logging code inside the method itself.

### Advice Summary

| Advice            | When It Runs                  | Access To                     | Use For                                             |
| ----------------- | ----------------------------- | ----------------------------- | --------------------------------------------------- |
| `@Before`         | Before method                 | JoinPoint (method info, args) | Logging entry, validation, security checks          |
| `@After`          | After method (always)         | JoinPoint                     | Cleanup, logging exit                               |
| `@AfterReturning` | After successful return       | JoinPoint + return value      | Logging result, post-processing                     |
| `@AfterThrowing`  | After exception thrown        | JoinPoint + exception         | Error logging, alerting                             |
| `@Around`         | Before + after (wraps method) | ProceedingJoinPoint           | Execution time, caching, transactions, full control |
