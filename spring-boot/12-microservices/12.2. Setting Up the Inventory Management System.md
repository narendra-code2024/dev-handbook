### Project Setup

#### Step 1: Create Services from Spring Initializr

Go to [start.spring.io](https://start.spring.io/) and create two projects:

**Inventory Service:**

- Group: `com.example.ecommerce`
- Artifact: `inventory-service`
- Dependencies: Spring Web, Spring Data JPA, PostgreSQL Driver, Lombok

**Order Service:**

- Group: `com.example.ecommerce`
- Artifact: `order-service`
- Dependencies: Spring Web, Spring Data JPA, PostgreSQL Driver, Lombok

#### Step 2: Organize into One Folder

```bash
mkdir ecommerce
# Download both zips into ecommerce/
cd ecommerce
unzip inventory-service.zip
unzip order-service.zip
rm *.zip
```

```
ecommerce/
  ├── inventory-service/
  │     └── src/main/java/com/example/ecommerce/inventory_service/
  └── order-service/
        └── src/main/java/com/example/ecommerce/order_service/
```

> Open the `ecommerce` folder in IntelliJ — it will detect both projects.

### Inventory Service — Entities and Structure

```
inventory-service/src/main/java/com/example/ecommerce/inventory_service/
  ├── entity/
  │     └── Product.java
  ├── repository/
  │     └── ProductRepository.java
  ├── service/
  │     └── ProductService.java
  ├── controller/
  │     └── ProductController.java
  └── dto/
        └── ProductDto.java
```

```java
@Entity
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class Product {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String title;

    private String description;

    @Column(nullable = false)
    private BigDecimal price;

    @Column(nullable = false)
    private Integer stock;
}
```

```properties
# inventory-service application.properties
server.port=8082
spring.datasource.url=jdbc:postgresql://localhost:5432/inventory_db
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.jpa.hibernate.ddl-auto=update
```

### Order Service — Entities and Structure

```
order-service/src/main/java/com/example/ecommerce/order_service/
  ├── entity/
  │     ├── Order.java
  │     ├── OrderItem.java
  │     └── OrderStatus.java
  ├── repository/
  │     └── OrderRepository.java
  ├── service/
  │     └── OrderService.java
  ├── controller/
  │     └── OrderController.java
  └── dto/
        └── OrderDto.java
```

```java
@Entity
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    private BigDecimal totalAmount;

    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL)
    private List<OrderItem> items;
}
```

```java
public enum OrderStatus {
    PENDING, CONFIRMED, CANCELLED
}
```

```java
@Entity
@Getter @Setter @NoArgsConstructor @AllArgsConstructor @Builder
public class OrderItem {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long productId;  // references Product in Inventory Service

    private Integer quantity;

    private BigDecimal price;

    @ManyToOne
    @JoinColumn(name = "order_id")
    private Order order;
}
```

```properties
# order-service application.properties
server.port=8081
spring.datasource.url=jdbc:postgresql://localhost:5432/order_db
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.jpa.hibernate.ddl-auto=update
```

> Each service has its own database — `inventory_db` and `order_db`. This is the **database per service** pattern in microservices.

---

### Key Points

- **Separate databases** — services don't share tables. Order Service stores `productId` as a reference, not a foreign key to Inventory Service's Product table.
- **Separate ports** — Order on 8081, Inventory on 8082. Both run independently.
- **No direct communication yet** — Order Service can't call Inventory Service. That's covered in Page 12.5 (OpenFeign).
- **No service discovery yet** — services don't know about each other. Covered in Page 12.3 (Eureka).
