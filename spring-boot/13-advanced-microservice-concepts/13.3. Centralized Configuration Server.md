### The Problem

Each microservice has its own `application.properties` — to change a config, you must go into each service, modify the file, and restart. With 10+ services, this is slow and error-prone.

```
Without Config Server:
  Order Service     → application.properties (local)
  Inventory Service → application.properties (local)
  API Gateway       → application.yml (local)
  Change DB URL? → edit each file → restart each service

With Config Server:
  All services → pull config from → Config Server → pulls from → GitHub
  Change DB URL? → edit one file in GitHub
```

---

### What is a Config Server?

A **Spring Cloud Config Server** is a dedicated service that acts as a **centralized repository**, allowing all microservices to retrieve their configurations **dynamically at runtime**.

- Manages different environments — `dev`, `test`, `prod` — using **Spring profiles**
- Each environment can have its own set of properties (e.g., different DB URLs, API keys)

---

### How It Works

```
┌──────────────────┐        ┌───────────────────────┐        ┌─────────────┐
│  Order Service   │        │  Spring Cloud         │        │   GitHub    │
│  (Config Client) │◄───────│  Config Server        │◄───────│   Repo      │
├──────────────────┤        │  port: 8888           │        │             │
│  Inventory Svc   │◄───────│                       │        │ application │
│  (Config Client) │        │  Pulls configs from   │        │ .yml        │
├──────────────────┤        │  GitHub on startup    │        │ order-svc   │
│  API Gateway     │◄───────│                       │        │ .properties │
│  (Config Client) │        │                       │        │ inv-svc     │
└──────────────────┘        └───────────────────────┘        │ .properties │
                                                             └─────────────┘
```

1. **Config Server starts** → clones the GitHub repo
2. **Microservices start** → ask Config Server for their config (by service name + profile)
3. **Config Server returns** the matching properties
4. **Config changes in GitHub** → picked up on next restart

---

### GitHub Repo Structure

```
ecommerce-config-repo/
  ├── application.yml                 ← global/shared config (all services inherit)
  ├── order-service.properties        ← order service, default profile
  ├── order-service-dev.properties    ← order service, dev profile
  ├── inventory-service.properties    ← inventory service, default profile
  └── api-gateway.yml                 ← API gateway config
```

> **Important:** File names must exactly match `spring.application.name` of each service.  
> The global `application.yml` is inherited by **all** services — put shared config here (e.g., Eureka URL).

#### Config Loading Order (Profile Hierarchy)

```
1. application.yml                    (global — shared by all services)
         │
         ▼
2. order-service.properties           (service default profile)
         │
         ▼
3. order-service-dev.properties       (dev profile — overrides defaults)
```

### Setting Up the Config Server

#### Step 1 — Create Project

From [start.spring.io](https://start.spring.io/):

- **Artifact:** `config-server`
- **Dependencies:** `Config Server`, `Eureka Discovery Client`

#### Step 2 — Enable Config Server

```java
@SpringBootApplication
@EnableConfigServer
public class ConfigServerApplication {
    public static void main(String[] args) {
        SpringApplication.run(ConfigServerApplication.class, args);
    }
}
```

#### Step 3 — Configure `application.yml`

```yaml
spring:
  application:
    name: config-server
  cloud:
    config:
      server:
        git:
          uri: https://github.com/your-username/ecommerce-config-repo
          username: your-github-username
          password: your-github-personal-access-token
          default-label: master                         # or main
          clone-on-start: true

server:
  port: 8888

eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka
```

> **Private Repo?** Generate a **Personal Access Token** in GitHub:  
> `Settings → Developer Settings → Personal Access Tokens → Fine-grained tokens`  
> Grant **Contents: Read** and **Administration: Read** permissions for the config repo.  
> ⚠️ Copy the token immediately — it won't be shown again.

#### Step 4 — Verify Config Server

```
GET http://localhost:8888/order-service/default
GET http://localhost:8888/order-service/dev
GET http://localhost:8888/inventory-service/default
```

### Setting Up Config Clients

Add to **every microservice** that should pull config from the server.

#### Step 1 — Add Dependency

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-config</artifactId>
</dependency>
```

#### Step 2 — Minimal `application.properties`

```properties
spring.application.name=order-service
spring.config.import=configserver:http://localhost:8888
```

> The local `application.properties` only needs the **service name** and **config server location**. Everything else comes from the GitHub repo.

---

### Shared / Global Config — `application.yml`

Properties common across all services go in the root `application.yml` in the GitHub repo:

```yaml
# inherited by ALL services
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka
```

> If a specific service needs a different value, it can override in its own file.

---

### Profile-Specific Config

```
order-service.properties           ← default profile
order-service-dev.properties       ← dev profile
order-service-prod.properties      ← prod profile
```

```properties
# order-service.properties (default)
my.variable=order-github-default

# order-service-dev.properties (dev)
my.variable=order-github-dev        ← overrides the default value
```

#### Accessing in Code

```java
@Value("${my.variable}")
private String myVariable;

@GetMapping("/hello")
public String hello() {
    return "Hello from Order Service. my.variable = " + myVariable;
}
```

#### Activating a Profile

```properties
# application.properties (or via environment variable / JVM flag)
spring.profiles.active=dev
```

### Startup Order

```
1. Discovery Service  (Eureka — port 8761)
2. Config Server      (port 8888 — registers with Eureka, clones GitHub repo)
3. All other services (pull config from Config Server, register with Eureka)
```

---

### Final Architecture

```
ecommerce/
  ├── discovery-service/    port 8761  (Eureka)
  ├── config-server/        port 8888  (pulls from GitHub)
  ├── api-gateway/          port 8080  (config client)
  ├── order-service/        port 8081  (config client)
  └── inventory-service/    port 8082  (config client)
```
